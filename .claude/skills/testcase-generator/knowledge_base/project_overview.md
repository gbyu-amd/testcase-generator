# 项目整体介绍

## 项目背景

本系统主要服务于制药企业的商业化生产阶段，覆盖原料药、制剂（如片剂、注射液）及生物制品等多种产品类型。系统管理对象涵盖产品全生命周期数据，包括产品基础信息（名称、剂型、编码）、生产操作、关键物料属性（CMA）及中间控制标准（IPC）。其业务逻辑根植于产品-工艺矩阵，明确关联特定产品、其生产工艺路线以及对应的CPP与CQA库，形成结构化数据模型。该系统需满足不同产品在不同阶段（初始阶段与稳定阶段）的差异化监控频率要求，并确保所有分析活动均符合产品特定工艺规程与质量标准。

## 核心价值主张

| 价值主张 | 说明 |
|---|---|
| 确保安全 | 通过对工艺的持续监控，能够更早、更有效地发现潜在的质量风险，防止不合格产品流入市场，从根本上保障药品的安全、有效和质量可控。 |
| 降低合规风险 | 建立一个符合 FDA 21 CFR Part 11（电子记录电子签名）、EU Annex 11 等法规要求的数字化系统，轻松应对监管审计，减少 483 警告信或数据完整性缺陷项。 |
| 提升运营效率 | 自动化数据收集、分析和报告生成，将工艺工程师从繁琐的手工劳动中解放出来，使其能专注于高价值的工艺分析和问题解决，大幅提升工作效率。 |
| 驱动持续改进 | 通过对工艺数据的深度分析（如多变量分析、趋势分析），可以发现工艺优化的机会，实现更高的产率、更低的能耗和废品率，从而降低生产成本。 |
| 增强决策支持 | 为企业的质量决策（如产品放行、变更控制、年度产品回顾）提供基于数据的、客观的、及时的洞察和支持。 |

持续工艺确认（CPV）的产品背景是一个由监管变革驱动、行业痛点催生、业务价值导向、并由技术发展赋能的必然结果。它代表了药品质量保证从“静态、事后、基于合规”到“动态、实时、基于科学与风险”的根本性转变。因此，CPV 系统不仅仅是一个 IT 工具，而是企业质量体系的核心组成部分，是连接生产操作、质量管理和持续改进的数字化枢纽，其最终目标是确保每一批投放市场的药品都具有稳定可靠的质量。

## 系统概述

CPV（持续工艺确认）系统是一个面向制药行业工艺验证的闭环管理平台。它通过年度计划驱动、任务执行跟踪、方案模版标准化、监控项目数据分析和报告自动生成的全流程数字化管理，将传统的纸质化、离散式工艺监控转变为线上化、自动化、智能化的合规管理体系。

## 主要功能模块

### 自定义站点主要功能模块

| 功能模块 | 功能名称 | 描述 |
|---|---|---|
| 首页/工作台 | 待办与看板 | 个性化展示待办事项、任务状态分布、核心模块快捷入口。 |
| 产品管理 | 产品增删改查 | 维护 CPV 监控的对象，供后续所有模块关联，支持导入导出。 |
| 年度计划设置 | 计划全生命周期管理 | 定义周期性 CPV 任务的生成规则，支持草稿、审批、生效、修订、失效。生效时自动生成任务。 |
| 任务管理 | 任务执行与状态跟踪 | 聚合展示所有执行项（周期/单次），提供日历视图，关联方案与报告，跟踪进度条。 |
| 方案模版 | 方案模版相关功能 | 创建标准方案格式，支持附件（Word/PDF）上传与在线编辑，作为方案编制的依据。 |
| 方案编制 | 方案设计与审批 | 基于模板和具体任务，编制包含监控项目列表的详细方案，管理其审核与生效流程。 |
| 监控项目 | 监控项目 | 记录具体分析项及其分析结果，状态跟随方案审批和数据分析流转（草稿 -> 已分析 -> 已确认）。 |
| 数据分析 | 数据分析 | 提供数据源连接、图表配置与渲染功能，支持监控项目的单个及批量数据分析，保存分析结果。 |
| 报告模版 | 报告模版相关功能 | 创建报告模版。 |
| 报告编制 | 报告编制与审批相关 | 引用报告模板，基于报告监控数据和分析结果，填充报告内容，完成最终报告的审核与输出。 |
| 用户管理 | 用户账号管理 | 维护系统用户账号信息，包括新增、编辑、启用/禁用、密码重置，支持批量导入和审计追踪。 |
| 角色管理 | 角色权限分配 | 基于 RBAC 模型管理角色，为角色分配菜单、按钮、数据权限，并管理角色与用户的关联关系。 |
| 部门管理 | 组织架构管理 | 以树形结构维护企业组织架构，支持部门的新增、编辑、删除及层级关系调整。 |
| 岗位管理 | 岗位信息维护 | 维护企业岗位信息，包括岗位名称、所属部门、岗位职级等，支持岗位的新增、编辑、删除，用于用户信息补充及审批流程中的人员筛选。 |
| 工作流配置 | 审批流程定义 | 支持自定义各模块的审批流（多级、加签、退回），驱动状态变更。 |
| 异步任务处理 | 异步任务调度 | 处理耗时操作（如导入导出、报告生成），支持任务提交、状态查询、结果下载及失败重试。 |
| 审计追踪 | 操作记录 | 记录所有关键操作的日志，满足合规要求（21 CFR Part 11）。 |
| 电子签名 | 关键动作确认 | 对审核、批准等重要操作进行电子签名认证。 |
| 通知管理 | 消息提醒 | 任务执行、审批待办等场景的站内信、邮件通知。 |

### 公共管理站点主要功能模块

| 功能模块 | 功能名称 | 描述 |
|---|---|---|
| 统一门户管理 | 统一门户与站点入口 | 用户登录公共管理系统后进入统一门户页面，展示欢迎信息、当前时间，以及所有可访问的业务站点入口；公共管理站点在此展示入口。页面提供退出登录功能。 |
| 用户管理 | 全局用户账号维护 | 在公共管理系统中统一维护所有用户账号，账号全局唯一。用户可被授权访问多个业务站点，每个站点下可独立配置角色和部门，并通过后续权限分配实现。用户信息创建后自动同步至其关联的业务站点。 |
| 站点管理 | 多站点基础信息与状态管理 | 集中维护系统中的所有业务站点（如分公司、项目、部门等独立单元），支持站点新增、删除、编辑、查询及启用/禁用状态控制。禁用后，该站点及其关联数据（如用户、权限）在业务中不可见或不可操作。支持为每个站点指定多个站点管理员，负责站点下用户、配置等日常管理；支持上传站点图片（如 Logo 或封面）并在卡片中展示，用于提升站点辨识度。该模块是多站点架构基础，用于定义独立业务单元并支撑多租户或分级管理。 |
| 权限管理 | 角色权限配置与用户授权 | 用于管理系统中的权限（角色或权限实体），支持权限新增、删除、编辑、查询、批量删除及分配用户等操作。每个权限可配置菜单权限和按钮权限，分别控制用户可访问的菜单范围与可操作的按钮范围；可将一个权限分配给多个用户，实现统一授权。 |
| 系统界面配置 | 公共管理界面个性化配置 | 允许公共管理站点管理员对系统界面进行个性化定制，包括 Logo 配置。仅拥有系统界面配置权限的管理员可进行操作。所有配置仅对公共管理站点生效，不影响各业务站点；未配置的业务站点默认使用公共管理的界面配置。 |
| 审计追踪 | 公共管理操作留痕与审计检索 | 记录公共管理中所有关键业务操作，满足 FDA 21 CFR Part 11 等法规对电子记录可追溯性的要求。支持按模块、操作类型、受影响对象等多维度检索，确保所有操作可追溯、可审计。 |
| 用户登录日志 | 登录行为记录与安全分析 | 记录系统用户的登录行为，包括登录时间、登出时间、登录状态、客户端信息等，用于安全审计、异常行为追溯及用户活跃度分析。 |
| 系统服务管理 | 公共管理级全局系统配置 | 提供公共管理级别的系统全局配置，包括登录配置、登录密码策略、电子签名密码策略和水印配置。配置对所有业务站点生效，仅可在公共管理站点中操作。 |

## 公共管理站点与自定义站点关系

| 项目 | 关系说明 |
|---|---|
| 系统定位 | 公共管理站点负责全局基础数据管理，包括站点、用户、权限等，不处理具体业务数据，如 CPV 计划、任务、方案、报告等。自定义站点负责各站点内的具体业务功能，包括首页、计划管理、任务管理、方案管理、监控项目、报告管理、配置管理等，业务数据存储在站点本地。 |
| 数据同步 | 公共管理站点向自定义站点同步基础数据，包括用户信息、站点信息、角色权限分配等。自定义站点创建的业务本地用户如需同步至公共管理，需要同步至公共管理系统的用户管理，账号全局唯一。 |
| 页面跳转与站点切换 | 公共管理站点和各自定义站点的页面顶部导航栏均展示当前用户可访问的站点列表，包括公共管理站点和所有有权限的自定义站点。用户点击任意站点名称即可跳转至该站点首页，无需重新登录；当前所在站点在列表中高亮显示。 |
| 统一认证 | 所有站点共用同一套用户认证体系，基于 JWT 或 Session 的认证 Token 在站点间通用。用户会话信息中包含用户 ID、用户名、可访问站点列表等，用于控制站点入口展示和跨站点访问。 |
| 业务功能边界 | 公共管理站点只包含全局管理能力，如站点管理、用户管理、权限管理、系统界面配置、审计追踪、登录日志、系统服务管理等；不包含计划、任务、方案、报告等业务菜单。自定义站点独立运行，承载 CPV 业务功能，各站点业务数据相互隔离。 |
| 站点管理员权限 | 自定义站点管理员在所属站点内拥有最高权限，可管理本站点的业务配置、本地用户与菜单权限。公共管理站点的站点管理员拥有管理所有站点、用户、权限的权限。 |
| 数据分析插件化平台 | 各自定义站点共用同一套数据分析平台能力，站点与站点之间的数据相互隔离，支持数据管理、数据处理和数据分析等能力复用。 |
| 公共管理站点特性 | 公共管理站点在系统初始化时自动创建，站点编号为 `PUBLIC_ADMIN`，站点名称为“公共管理”。该站点不可删除、不可停用，不参与 CPV 业务数据处理，仅提供全局配置能力。 |

## 核心业务关系

1. 年度计划管理生效时，生成周期性任务进入任务管理。
2. 任务管理可被方案管理和报告管理关联。
3. 方案管理引用方案模板，并定义或生成监控项目。
4. 报告管理引用报告模板，并引用监控项目的分析结果。
5. 报告管理生效后，推动关联任务状态更新，如从进行中更新为已完成。
