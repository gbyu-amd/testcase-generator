# 用户角色说明

## 用户角色与职责

CPV（持续工艺确认）系统的用户角色应围绕方案创建、数据分析、合规审核、生产数据查看、检验结果确认和系统权限管理进行划分。测试用例生成时，应重点关注不同角色的可操作范围、审批权限、数据隔离、审计追踪和电子签名要求。

| 角色 | 核心职责 |
|---|---|
| 工艺工程师 | 创建和维护 CPV 方案；执行或审核数据分析；起草报告；调查工艺异常。 |
| QA 专员/经理 | 审核与批准 CPV 方案和报告；监督整个 CPV 流程的合规性；处理与 CPV 相关的偏差和变更。 |
| 生产操作员 | 查看执行任务；录入无法自动采集的线下工艺数据。 |
| QC 分析员 | 确认与 CPV 相关的 CQA 检验结果。 |
| 系统管理员 | 管理系统用户、角色、权限；配置系统参数、工作流和外部数据源接口。 |

## 工艺工程师

### 可操作范围

- 创建、编辑和维护 CPV 方案
- 维护产品、工艺路线、CPP、CQA、CMA、IPC 等分析对象
- 执行或审核数据分析，查看趋势分析、统计过程控制和异常信号
- 起草 CPV 报告，填写工艺分析结论和改进建议
- 对工艺异常进行调查、说明和跟踪

### 权限边界

- 不应批准自己起草的最终 CPV 报告
- 不应修改已批准、已归档或已锁定的原始生产和检验数据
- 不应访问未授权产品、工厂、部门或批次的数据

### 测试关注点

- 工艺工程师只能创建或维护授权产品范围内的 CPV 方案
- CPP/CQA/CMA 关联关系变更应保留历史版本和变更原因
- 报告起草后应进入 QA 审核流程，不能直接发布
- 数据分析结果应与来源数据保持一致
- 未授权产品或批次数据不可查看、导出或分析

## QA 专员/经理

### 可操作范围

- 审核和批准 CPV 方案、监控计划和 CPV 报告
- 监督 CPV 流程是否符合 GMP、数据完整性和企业质量体系要求
- 审核偏差、OOS、OOT、CAPA 和变更相关记录
- 查看审计追踪、电子签名、审批记录和历史版本
- 驳回不符合要求的方案、任务或报告

### 权限边界

- 不应直接篡改原始生产数据、检验数据或审计追踪
- 不应绕过审批流程直接发布报告
- 不应在前置审核或必要数据缺失时批准关键记录

### 测试关注点

- 未授权用户不能审核或批准 CPV 方案和报告
- 审核通过后记录状态应正确流转并锁定关键内容
- 驳回后应回退到正确节点，并保留驳回原因
- 审批动作应记录操作者、时间、意见和电子签名
- QA 应能追溯完整数据链路和历史版本

## 生产操作员

### 可操作范围

- 查看与本人相关的生产执行任务、产品、批次和工艺步骤
- 录入无法自动采集的线下工艺数据
- 补充生产过程中的关键操作信息、设备信息或过程参数
- 查看本人任务状态、待办事项和提交结果
- 按流程提交数据更正申请或说明

### 权限边界

- 不应查看或修改无关产品、无关批次或其他部门数据
- 不应修改已复核、已审批、已锁定或已归档的数据
- 不应审核 CPV 报告、批准异常结论或关闭 CAPA

### 测试关注点

- 生产操作员只能看到分配给自己的任务和授权批次
- 必填线下工艺数据缺失时不能提交
- 超出工艺规程范围的数据应触发提示或异常流程
- 提交后任务状态应正确流转到复核或审核节点
- 数据更正应记录原因、时间、操作者和前后值

## QC 分析员

### 可操作范围

- 查看与本人检验任务相关的产品、批次、样品和质量标准
- 确认与 CPV 相关的 CQA 检验结果
- 录入或复核实验室检测结果、检验结论和异常标识
- 标记或提交疑似 OOS/OOT 结果
- 查看检验任务状态和数据同步结果

### 权限边界

- 不应修改已批准或已归档的检验结果
- 不应批准最终 OOS/OOT 调查结论
- 不应绕过复核流程直接将异常数据纳入最终 CPV 报告

### 测试关注点

- CQA 检验结果应按产品特定质量标准校验
- OOS/OOT 数据应触发异常标识和后续调查流程
- 数据录入或导入应校验单位、范围、样品和批次匹配关系
- 检验结果修改应记录审计追踪
- 未确认或未复核的检验结果不应作为最终 CPV 结论依据

## 系统管理员

### 可操作范围

- 管理系统用户、角色、权限和组织范围
- 配置系统参数、工作流、审批节点和通知规则
- 配置外部数据源接口，如生产系统、实验室系统或数据仓库接口
- 维护基础字典，如产品类型、剂型、工艺阶段、监控频率和状态枚举
- 查看系统运行日志和接口同步状态

### 权限边界

- 不应替业务用户完成 CPV 方案审核、报告批准或异常关闭
- 不应直接修改业务审批后的 CPV 数据、报告结论或审计记录
- 不应删除电子签名、审计追踪或关键历史记录

### 测试关注点

- 非管理员不能访问用户、角色、权限和系统配置页面
- 权限变更后用户重新登录或刷新后应立即生效
- 工作流配置变更应影响后续新流程，不应破坏已进行中的流程
- 外部数据源接口配置错误时应有明确失败提示和日志
- 管理员关键操作应记录审计追踪

## 跨角色通用测试关注点

- 不同角色只能访问授权产品、工厂、部门、批次和任务范围内的数据
- 登录态过期后，保存、提交、审核、批准、导出等动作应被拦截
- 关键业务动作应记录操作者、时间、动作、原因、前后值和电子签名
- 已批准、已归档或已锁定的数据不可被普通编辑动作修改
- 审批流应支持提交、审核、驳回、重新提交、批准和归档等状态流转
- 同一用户是否允许同时承担创建、审核、批准角色，应按系统权限和质量规程控制
- 批量导入、批量审批、批量导出等操作应校验角色权限、数据范围和数据状态
