# 用户角色说明

## 未登录用户

未登录用户可以访问首页、搜索页和部分商品详情页，但不能直接加入购物车、查看购物车、提交订单或使用账号相关能力。

测试关注点：

- 未登录访问购物车时应跳转登录页。
- 未登录点击立即购买时应跳转登录页。
- 登录成功后应返回原始操作页面或继续原始操作。

## 普通登录用户

普通登录用户可以搜索商品、查看详情、加入购物车、编辑购物车、选择地址、使用优惠券和提交订单。

测试关注点：

- 登录态有效期。
- 购物车和订单数据归属当前用户。
- 不同账号之间数据隔离。

## 异常账号用户

异常账号包括被冻结、被限制交易、风险验证未通过或需要二次验证的账号。

测试关注点：

- 账号被冻结时禁止登录或禁止交易。
- 风险账号提交订单时应触发校验。
- 需要二次验证时页面应有明确引导。

## 商家或后台运营

商家或运营可以改变商品状态、库存、价格和活动信息。前端测试中通常把这些变化作为外部数据状态。

测试关注点：

- 商品下架后前端不可继续购买。
- 库存变化后购物车和结算页应刷新。
- 价格变化后订单金额应重新计算。
