# 购物车流程

## 流程目标

验证用户可以管理购物车商品，包括查看、勾选、修改数量、删除、批量操作和跳转结算。

## 主流程

1. 用户登录后进入购物车。
2. 页面展示当前账号购物车商品。
3. 用户勾选商品。
4. 用户修改商品数量或删除商品。
5. 页面实时刷新商品小计、店铺合计和总金额。
6. 用户点击结算进入结算页。

## 分支流程

- 商品库存不足时提示库存不足。
- 商品失效时不可勾选结算。
- 商品价格变化时提示价格已更新。
- 全选和取消全选应影响所有可结算商品。
- 多页签修改购物车后刷新应展示最新状态。

## 测试重点

- 勾选状态。
- 数量边界。
- 金额计算。
- 失效商品处理。
- 与结算页数据一致。
