# 数据分析业务流程

## 流程目标

描述 CPV 系统中数据分析从进入分析列表、上传 Excel 文件、文件校验、选择分析算法、插件状态检查、分析配置到结果生成与结果处理的完整流程。测试用例生成时，应重点关注文件校验、插件依赖、算法配置、字段联动、结果生成、结果保存和外部系统回传等场景。

## 主流程

1. 用户进入数据分析列表。
2. 用户点击数据分析按钮。
3. 用户上传 Excel 文件。
4. 系统执行文件校验。
5. 文件校验成功后，进入数据分析详情页面。
6. 用户选择分析算法。
7. 系统执行插件状态检查。
8. 插件已安装时，用户选择具体分析项目。
9. 用户进入分析图配置。
10. 用户配置维度字段配置区。
11. 系统联动显示动态配置表单。
12. 用户点击开始分析。
13. 系统生成分析结果。
14. 点击“推送至其他业务系统”，将数据、分析图、结论回传至CPV系统，对应监控项目的分析任务状态更新为“已分析”，分析结果名称字段填充为生成的结果名称。
15. 流程结束。

## 分支流程

### 文件校验失败

1. 用户上传 Excel 文件。
2. 系统校验文件格式、字段、必填项、数据类型、数据范围和数据完整性。
3. 文件校验失败时，系统提示失败原因。
4. 用户需重新上传符合要求的 Excel 文件后，才能进入数据分析详情页面。

### 插件未安装

1. 用户选择分析算法后，系统检查对应插件状态。
2. 如果插件未安装，系统提示用户下载并安装插件。
3. 用户完成插件安装后，继续选择具体分析项目。
4. 插件仍未安装或安装失败时，应阻断后续分析流程。

### 结果回传其他系统

1. 系统生成分析结果后，用户选择回传其他系统。
2. 系统通过接口回调通知结果。
3. 回传成功后流程结束。
4. 回传失败时，应记录失败原因，并支持重试或人工处理。

### 结果保存到当前系统

1. 系统生成分析结果后，用户选择保存到当前系统。
2. 系统将分析结果保存至分析列表。
3. 保存成功后，用户可在数据分析列表查看历史分析记录。

## 关键业务规则

- 只有通过文件校验的 Excel 文件才能进入数据分析详情页面。
- 文件校验应覆盖模板格式、字段完整性、数据类型、数据范围、必填项和数据完整性。
- 分析算法选择后，系统必须检查对应插件是否已安装。
- 插件未安装时，不允许继续执行具体分析项目配置和开始分析。
- 分析图配置和维度字段配置应与所选算法、分析项目动态联动。
- 动态配置表单应根据字段配置变化实时更新。
- 点击开始分析前，必须校验算法、分析项目、字段配置和动态配置表单是否完整。
- 分析结果生成后，必须明确选择结果处理方式。
- 保存到当前系统时，应生成可追溯的分析记录。
- 回传其他系统时，应记录接口调用状态、返回结果、失败原因和重试记录。

## 测试重点

- Excel 文件上传格式、大小、空文件、损坏文件、字段缺失和非法字段校验。
- 文件校验失败时的错误提示、重传路径和状态保持。
- 不同分析算法对应插件状态检查是否准确。
- 插件未安装、下载插件、安装成功、安装失败后的流程控制。
- 具体分析项目列表是否随算法正确变化。
- 分析图配置和维度字段配置区是否正确联动。
- 动态配置表单是否根据字段配置实时刷新。
- 必填配置缺失时是否禁止开始分析。
- 分析结果是否正确生成、展示、保存或回传。
- 接口回调失败时是否支持失败提示、日志记录和重试。
- 分析列表中是否能查看保存后的分析结果和执行状态。
- 分析记录是否包含数据来源、算法、配置、执行人、执行时间、结果状态和处理方式。
