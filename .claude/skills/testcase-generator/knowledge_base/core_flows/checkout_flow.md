# 结算下单流程

## 流程目标

验证用户在结算页确认商品、地址、优惠、金额后，可以正确提交订单，并处理库存、价格、优惠和地址异常。

## 主流程

1. 用户从购物车或商品详情页进入结算页。
2. 页面展示商品信息、收货地址、配送方式、优惠信息和订单金额。
3. 用户选择或新增收货地址。
4. 用户选择可用优惠券。
5. 系统计算应付金额。
6. 用户点击提交订单。
7. 系统生成订单并进入支付前页面。

## 分支流程

- 未设置收货地址时提示新增地址。
- 地址不可配送时禁止提交。
- 优惠券失效时重新计算金额。
- 库存不足时阻断提交订单。
- 重复点击提交订单时只能生成一个订单。
- 提交订单接口失败时应提示失败，并允许用户重试。

## 测试重点

- 订单金额准确。
- 库存和价格提交前二次校验。
- 重复提交防护。
- 失败后的状态回滚。
