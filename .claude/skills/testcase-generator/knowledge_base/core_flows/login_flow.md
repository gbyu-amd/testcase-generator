# 登录流程

## 流程目标

验证用户可以通过账号密码、手机号验证码等方式完成登录，并在登录成功后获得正确的页面访问权限和业务状态。

## 主流程

1. 用户打开登录页。
2. 输入账号、密码或验证码。
3. 点击登录按钮。
4. 系统校验账号状态、登录凭证和风险状态。
5. 登录成功后进入首页或返回原访问页面。
6. 页面展示用户昵称、购物车入口和账号相关入口。

## 分支流程

- 密码错误时展示错误提示。
- 账号不存在时提示账号未注册。
- 验证码错误或过期时提示重新获取。
- 账号被冻结时阻断登录。
- 登录成功后返回原本访问的购物车或结算页。

## 测试重点

- 登录态写入和保持。
- 登录成功后的页面跳转。
- 未登录访问受限页面时的拦截。
- 多页签退出登录后的状态同步。
