# 登录与站点选择流程

## 流程目标

描述用户从登录页输入账号密码、选择可访问站点、完成账号与站点权限校验，到跳转至公共管理站点或自定义业务站点的完整流程。测试用例生成时，应重点关注账号密码校验、站点列表展示、站点访问权限、公共管理与自定义站点跳转差异、会话信息记录和异常登录场景。

## 主流程

1. 用户进入系统登录页。
2. 用户输入账号和密码。
3. 系统根据用户账号查询该用户有权限访问的所有站点。
4. 登录页展示站点下拉列表，站点范围包括“公共管理”以及用户有权限访问的自定义站点。
5. 用户从站点下拉列表中选择目标站点。
6. 系统验证账号密码是否正确。
7. 系统校验该账号是否有权限登录所选站点。
8. 校验通过后，系统根据所选站点跳转至对应系统：
   - 如果选择“公共管理”，跳转至公共管理系统的统一门户页面。
   - 如果选择其他自定义业务站点，直接跳转至该站点首页或该站点的统一门户。
9. 登录成功后，系统在会话中记录 `user_id`、`username`、站点 ID、站点类型等信息。
10. 流程结束。

## 分支流程

### 账号密码错误

1. 用户输入账号和密码。
2. 用户选择目标站点并提交登录。
3. 系统校验账号密码失败。
4. 系统拒绝登录，并提示账号或密码错误。
5. 用户停留在登录页，可重新输入账号密码后再次登录。

### 账号无所选站点权限

1. 用户输入正确账号密码。
2. 用户选择一个无访问权限的站点并提交登录。
3. 系统校验账号与所选站点的授权关系。
4. 校验失败时，系统拒绝登录并提示无站点访问权限。
5. 用户需重新选择有权限的站点后才能登录。

### 用户无任何可访问站点

1. 用户输入账号。
2. 系统查询该用户可访问站点列表。
3. 如果用户无任何可访问站点，站点下拉列表为空或不可选。
4. 用户无法完成登录，系统提示未分配可访问站点或联系管理员。

### 选择公共管理站点

1. 用户账号拥有公共管理访问权限。
2. 用户在站点下拉列表中选择“公共管理”。
3. 系统完成账号密码和站点权限校验。
4. 登录成功后跳转至公共管理系统统一门户页面。
5. 统一门户展示欢迎信息、当前时间以及用户可访问的站点入口。

### 选择自定义业务站点

1. 用户账号拥有某自定义业务站点访问权限。
2. 用户在站点下拉列表中选择该自定义站点。
3. 系统完成账号密码和站点权限校验。
4. 登录成功后跳转至该站点首页或该站点的统一门户。
5. 当前站点在页面顶部站点列表中高亮展示。

### 站点被禁用

1. 用户选择的目标站点处于禁用状态。
2. 系统校验站点可用性。
3. 校验失败时，系统拒绝登录该站点。
4. 用户不可进入该站点首页，也不可操作该站点关联业务数据。

## 业务规则

1. 用户登录必须同时通过账号密码校验和站点访问权限校验。
2. 站点下拉列表只展示当前用户有权限访问的站点，包括公共管理站点和自定义业务站点。
3. 公共管理站点与自定义业务站点共用统一认证体系，登录成功后无需在站点间重复登录。
4. 登录成功后的会话信息应包含用户标识、用户名、站点 ID、站点类型和可访问站点列表等关键字段。
5. 选择公共管理站点时进入公共管理系统统一门户；选择自定义业务站点时进入对应业务站点首页或统一门户。
6. 禁用站点不可被登录访问，其关联数据在业务中不可见或不可操作。

## 测试关注点

1. 正确账号密码登录公共管理站点成功。
2. 正确账号密码登录自定义业务站点成功。
3. 站点下拉列表仅展示当前用户有权限访问的站点。
4. 账号密码错误时不得登录任何站点。
5. 用户无所选站点权限时不得登录该站点。
6. 用户无任何可访问站点时无法完成登录。
7. 选择禁用站点时应拦截登录并给出明确提示。
8. 登录成功后会话中记录的用户信息、站点 ID、站点类型与实际选择一致。
9. 公共管理站点和自定义业务站点之间跳转时不应要求重复登录。
10. 当前所在站点应在站点列表中高亮展示。
