# 一键分析业务流程

## 流程目标

描述数据分析详情页中“替换数据源”和“一键分析”的完整业务流程。用户可在不退出分析流程的情况下更换底层数据源，系统在替换数据源后自动匹配已有分析项并尝试批量生成分析结果。测试用例生成时，应重点关注数据源选择、字段匹配、变量类型与数据分布校验、数据处理规则同步、全部失败/部分失败/全部成功结果提示、未分析原因展示和异常恢复。

## 主流程

1. 用户进入数据分析详情页面。
2. 页面第一列顶部展示“按流程展示 / 按列表展示”切换页签。
3. 系统在两个展示页签下方提供“替换数据源”和“数据处理”按钮。
4. 用户点击“替换数据源”按钮。
5. 系统弹出替换数据源弹窗。
6. 用户在弹窗左侧分类树中选择或搜索数据分类。
7. 用户在右上侧选择“数据管理”或“数据处理”页签；默认进入“数据管理”页签。
8. 如果选择“数据管理”，用户可继续在“系统集成”和“本地数据”页签间切换；默认进入“系统集成”页签。
9. 用户在右下侧列表中单选一个数据表或文件。
10. 用户点击“确定”。
11. 系统执行数据源替换流程。
12. 替换成功后，系统自动遍历当前所有已分析配置项。
13. 系统匹配新数据源中的字段、变量类型和数据分布要求。
14. 系统按匹配结果执行一键分析并批量生成可分析项结果。
15. 系统根据一键分析执行结果展示成功、失败或部分失败提示。
16. 用户确认提示后，查看分析结果或进入配置并分析页面处理未分析项。
17. 流程结束。

## 分支流程

### 数据处理入口

1. 用户在数据分析详情页面点击“数据处理”按钮。
2. 系统跳转至数据处理详情页面。
3. 数据处理详情页面展示“返回数据分析”按钮，并去掉原有“数据分析”按钮。
4. 用户点击“返回数据分析”。
5. 系统跳转回原数据分析详情页面。

### 选择数据管理表

1. 用户在替换数据源弹窗中选择“数据管理”下的数据表。
2. 用户点击“确定”。
3. 系统替换数据源。
4. 系统自动清空数据分析详情页面的数据处理规则，包括排序、筛选、转换、异常数据剔除。
5. 系统继续执行一键分析匹配和结果生成。

### 选择数据处理表

1. 用户在替换数据源弹窗中选择“数据处理”下的数据处理表。
2. 用户点击“确定”。
3. 系统替换数据源。
4. 系统保留数据处理规则，例如排序、筛选条件、转换。
5. 系统清空异常数据剔除规则。
6. 系统继续执行一键分析匹配和结果生成。

### 再次替换同一数据源

1. 用户已经选择过某个数据源。
2. 用户再次点击“替换数据源”。
3. 系统打开替换数据源弹窗。
4. 弹窗自动定位到上一次选择的数据源，并保留上一次选择内容。

### 全部分析失败

1. 系统完成新数据源与已有分析项匹配。
2. 所有分析项都无法匹配到任一检验项，无任何分析项可生成。
3. 系统弹窗提示：`当前分析项与数据源不匹配，或全部数据分析失败。系统将清空您已设置的参数，是否继续？`
4. 用户确认继续后，系统清空已设置参数和无法复用的分析结果。
5. 用户取消时，系统应保持当前页面状态，不继续清空配置。

### 部分成功部分失败

1. 系统完成一键分析。
2. 部分分析项生成结果成功，部分分析项未能分析。
3. 系统提示：`当前分析包含 [总数] 个子分析结果，已完成快速分析 [成功数] 个，未分析 [失败数] 个。未分析原因请至页面查看。`
4. 用户点击“确定”。
5. 系统进入配置并分析页面。
6. 未分析项在列表中显示“未分析”状态标签。
7. 用户鼠标悬停“未分析”标签时，系统展示具体未分析原因。

### 全部成功

1. 系统完成一键分析。
2. 所有分析项均成功生成分析结果。
3. 系统提示：`当前分析包含 [总数] 个子分析结果，已完成快速分析 [成功数] 个。`
4. 用户确认后查看已生成的分析结果。

### 字段删减导致未分析

1. 系统发现分析项依赖字段在新数据源中不存在。
2. 对应分析结果状态变为“未分析”。
3. 鼠标悬停提示：`XX、XXX 字段有删减，请手动分析。`
4. 系统清空配置区域中涉及的失效字段，不显示 `NULL`，避免残留引用。
5. 用户可手动重新配置并分析。

### 正态性变化导致未分析

1. 系统发现数据源替换或数据变换导致检验项正态性发生变化。
2. 对应分析结果状态变为“未分析”。
3. 鼠标悬停提示：`XXX 正态性发生改变，请手动分析。`
4. 用户可手动进入该分析项配置并重新分析。

### 数据源失效

1. 用户用于分析的数据表被中台侧停用或失效。
2. 系统在数据分析区提示：`当前数据表状态已失效，暂不支持分析。`
3. 所有分析结果无法继续分析时，系统统一提示：`数据源不可用。`
4. 用户需重新选择有效数据源后才能继续分析。

### 网络或服务器异常

1. 系统执行一键分析过程中发生网络中断或服务器异常。
2. 系统中断本次一键分析。
3. 页面提示：`分析失败，请稍后重试。`
4. 系统应保留可恢复的页面状态，避免出现部分配置丢失且无提示的情况。

## 关键业务规则

- 替换数据源弹窗左侧分类树支持分类名称搜索。
- “数据管理”页签下包含“系统集成”和“本地数据”页签，默认显示“系统集成”。
- 系统集成列表展示集成数据表名称、编号、分类、备注、状态、更新人、更新时间，支持名称和编号搜索。
- 本地数据列表展示文件名称、分类、状态、更新人、更新时间，支持文件名称搜索。
- 数据处理列表展示数据处理名称、数据管理名称、分类、更新人、更新时间，支持数据名称搜索。
- 替换数据源只能单选一项数据表或文件。
- 一键分析在替换数据源确定后自动触发。
- 分析项配置中使用的变量（横轴、纵轴、分组等）必须在新数据源中存在同名字段。
- 字段匹配还需满足分析方法对变量类型（定量 / 定类）和数据分布（正态 / 非正态）的要求。
- 未分析原因按优先级展示，每个未分析状态只显示命中的最高优先级原因。
- 当数据分析已产生结果，用户执行数据处理操作（排序、筛选、转换等）并保存后，再次进入页面时应自动重新分析并根据当前数据状态更新结果。
- 替换数据源后一键分析的结果可推送至其他业务系统。
- 数据分析插件化平台和 iBatchInsight 业务系统中的数据均需要支持替换数据源功能。

## 未分析原因优先级

| 优先级 | 原因类型 | 提示文案 |
|---|---|---|
| 1 | 定类变量暂不支持分析 | `XXX 变量为定类变量，暂不支持分析。` |
| 2 | 字段有删减 | `XXX、XXX 字段有删减，请手动分析。` |
| 3 | 正态性发生变化 | `XXX 正态性发生改变，请手动分析。` |
| 4 | 用此变量定义阶段下的内容不完全相同 | `用此变量定义阶段下的内容不完全相同，暂不支持分析。` |
| 5 | 其他单项分析错误 | 沿用单个分析时开始分析报错逻辑，例如“分析列中存在无效数值”“没有足够的数据”“阶段数超过 5 个”等。 |

## 测试重点

- 数据分析详情页“按流程展示 / 按列表展示”页签下是否均展示“替换数据源”和“数据处理”按钮。
- “数据处理”跳转和“返回数据分析”返回路径是否正确。
- 替换数据源弹窗的分类树、页签默认值、列表字段、搜索能力、单选和取消操作是否正确。
- 选择系统集成、本地数据、数据处理表时，列表字段和搜索条件是否符合对应类型。
- 选择数据管理表后，排序、筛选、转换、异常数据剔除规则是否全部清空。
- 选择数据处理表后，排序、筛选、转换是否保留，异常数据剔除是否清空。
- 再次打开替换数据源弹窗时，是否自动定位并保留上一次选择。
- 一键分析字段同名匹配、变量类型匹配、数据分布匹配是否准确。
- 全部失败、部分成功部分失败、全部成功三类执行结果提示是否准确。
- 未分析标签、悬停原因、原因优先级展示是否正确。
- 字段删减后配置区域是否清空失效字段，且不显示 `NULL`。
- 正态性变化后是否转为“未分析”并提示用户手动分析。
- 数据表停用、字段失效、无公共字段、网络中断、服务器异常等异常场景是否有明确提示和状态保护。
- 用户手动修复配置并重新分析成功后，是否清除“未分析”标记。
- 一键分析结果推送至其他业务系统时，是否记录推送状态和失败原因。
