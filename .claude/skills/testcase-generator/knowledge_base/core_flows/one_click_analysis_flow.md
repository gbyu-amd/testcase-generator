# 一键分析业务流程

## 流程目标

描述数据分析详情页中“数据替换”和“数据处理”触发一键分析的核心流程。用户可在不退出分析流程的情况下更换底层数据源或调整数据处理规则，系统自动匹配已有分析项并尝试批量生成分析结果。测试用例生成时，应重点关注入口范围、触发方式、字段匹配、变量类型与数据分布校验、结果提示、未分析原因和异常恢复。

## 适用入口

一键分析可在以下数据分析详情页触发：

1. 数据分析工作台模块数据分析详情页。
2. 监控项目模块数据分析详情页。
3. 相关性分析模块数据分析详情页。

## 触发方式

### 数据替换触发

1. 用户在数据分析详情页点击数据替换或替换数据源。
2. 用户选择新数据源，可选择中台数据对接同步表、MES 集成同步表、本地上传数据表或数据处理后的表。
3. 用户确认替换后，系统自动触发一键分析。

### 数据处理触发

1. 用户进入数据处理流程。
2. 用户对当前数据执行筛选、排序、转换或剔除。
3. 用户保存数据处理结果并返回数据分析详情页。
4. 系统基于处理后的数据自动触发一键分析。

## 主流程

1. 数据替换或数据处理完成后，系统遍历当前页面已有分析配置项。
2. 系统按字段名称匹配新数据源中的字段。
3. 系统校验变量类型和数据分布是否满足对应分析图要求。
4. 系统对可分析项执行一键分析并批量生成分析结果。
5. 系统根据执行结果展示全部成功、部分成功部分失败或全部失败提示。
6. 用户确认提示后查看分析结果；未分析项保留在页面中，供用户手动修复配置后重新分析。

## 结果提示

### 全部成功

1. 所有分析项均成功生成分析结果。
2. 系统提示：`当前分析包含 [总数] 个子分析结果，已完成快速分析 [成功数] 个。`
3. 用户确认后查看已生成的分析结果。

### 部分成功部分失败

1. 部分分析项生成结果成功，部分分析项未能分析。
2. 系统提示：`当前分析包含 [总数] 个子分析结果，已完成快速分析 [成功数] 个，未分析 [失败数] 个。未分析原因请至页面查看。`
3. 用户确认后进入配置并分析页面。
4. 未分析项显示“未分析”状态标签。
5. 鼠标悬停“未分析”标签时，系统展示具体未分析原因。

### 全部分析失败

1. 所有分析项都无法匹配到任一可分析项，无分析结果可生成。
2. 系统弹窗提示：`当前分析项与数据源不匹配，或全部数据分析失败。系统将清空您已设置的参数，是否继续？`
3. 用户确认继续后，系统清空已设置参数。
4. 用户取消时，系统保持当前页面状态，不继续清空配置。

## 一键分析失败原因

### 定类变量导致未分析

1. 系统发现分析项依赖字段在新数据源中为定类变量，即变量下数据全为空或存在非数值。
2. 对应分析结果状态变为“未分析”。
3. 鼠标悬停提示：`XXX 变量为定类变量，暂不支持分析。`
4. 系统清空配置区域中涉及的失效字段，不显示 `NULL`。
5. 用户可手动重新配置并分析。

### 字段删减导致未分析

1. 系统发现分析项依赖字段在新数据源中不存在。
2. 对应分析结果状态变为“未分析”。
3. 鼠标悬停提示：`XX、XXX 字段有删减，请手动分析。`
4. 系统清空配置区域中涉及的失效字段，不显示 `NULL`。
5. 用户可手动重新配置并分析。

### 正态性变化导致未分析

1. 系统发现数据源替换或数据处理导致检验项正态性发生变化。
2. 当前仅过程能力分析（正态）和过程能力分析（非正态）存在该类场景。
3. 对应分析结果状态变为“未分析”。
4. 鼠标悬停提示：`XXX 正态性发生改变，请手动分析。`
5. 用户可手动进入该分析项配置并重新分析。

### 阶段内容不一致导致未分析

1. 系统发现分析项依赖的阶段字段在新数据源中不存在，或已配置参考线的阶段在新数据源中不存在。
2. 对应分析结果状态变为“未分析”。
3. 鼠标悬停提示：`用此变量定义阶段下的内容不完全相同，暂不支持分析。`
4. 用户可手动重新配置并分析。

### 其他原因导致未分析

1. 系统发现数据源替换或数据处理后，当前分析图触发单项分析错误。
2. 对应分析结果状态变为“未分析”。
3. 鼠标悬停提示沿用点击开始分析时报错逻辑，例如“分析列中存在无效数值”“没有足够的数据”“阶段数超过 5 个”等。
4. 用户可手动重新配置并分析。

## 未分析原因优先级

| 优先级 | 原因类型 | 提示文案 |
|---|---|---|
| 1 | 定类变量暂不支持分析 | `XXX 变量为定类变量，暂不支持分析。` |
| 2 | 字段有删减 | `XXX、XXX 字段有删减，请手动分析。` |
| 3 | 正态性发生变化 | `XXX 正态性发生改变，请手动分析。` |
| 4 | 用此变量定义阶段下的内容不完全相同 | `用此变量定义阶段下的内容不完全相同，暂不支持分析。` |
| 5 | 其他原因分析错误 | 沿用点击开始分析时报错逻辑，例如“分析列中存在无效数值”“没有足够的数据”“阶段数超过 5 个”等。 |

## 异常场景

### 数据源失效

1. 用户用于分析的数据表被停用或失效。
2. 进入数据分析详情页时提示：`当前数据表状态已失效，暂不支持分析。`
3. 用户需重新选择有效数据源后才能继续分析。

### 网络或服务器异常

1. 系统一键分析过程中发生网络中断或服务器异常。
2. 系统中断本次一键分析并提示失败原因。
3. 页面应保留可恢复状态，避免配置丢失且无提示。

## 关键业务规则

- 一键分析仅在数据替换或数据处理后自动触发。
- 字段匹配优先按字段名称匹配，再校验变量类型和数据分布要求。
- 每个未分析项只展示命中的最高优先级原因。
- 定类变量、字段删减等导致配置字段失效时，配置区域应清空失效字段，不显示 `NULL`。
- 全部失败且用户取消时，不应清空已有配置或分析结果。
- 用户手动修复未分析项配置并重新分析成功后，应清除“未分析”状态。

## 测试重点

- 三类入口是否都能通过数据替换和数据处理触发一键分析。
- 数据替换是否支持中台数据对接同步表、MES 集成同步表、本地上传数据表和数据处理后的表。
- 数据处理后的筛选、排序、转换、剔除结果是否参与一键分析。
- 字段名称、变量类型和数据分布匹配是否准确。
- 全部成功、部分成功部分失败、全部失败三类提示是否准确。
- 全部失败时确认和取消的处理结果是否正确。
- 未分析标签、悬停原因和原因优先级是否正确。
- 字段删减或定类变量导致配置失效时，配置区域是否清空且不显示 `NULL`。
- 数据源失效、网络中断和服务器异常时是否有明确提示并保留可恢复状态。
