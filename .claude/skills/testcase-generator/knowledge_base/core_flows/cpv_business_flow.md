# CPV 主业务流程

## 流程目标

描述 CPV（持续工艺确认）系统从年度计划、周期性任务、方案模板、方案管理、监控项目、数据分析到报告管理的核心业务链路。测试用例生成时，应重点关注各模块之间的关联关系、状态流转、审批控制、版本管理、数据引用和报告输出的一致性。

## 核心业务链路

1. 年度计划管理中创建 CPV 年度计划。
2. 年度计划审批并生效后，系统按计划规则生成周期性任务。
3. 任务进入任务管理模块，跟踪待执行、进行中、完成、关闭等状态。
4. 方案模板提供标准方案结构，被方案管理模块引用。
5. 方案管理基于任务和方案模板创建具体方案，并在方案生效后定义或生成监控项目。
6. 监控项目或相关性分析完成数据分析后，分析结果可被报告管理模块引用。
7. 报告模板提供标准报告结构，被报告管理模块引用。
8. 报告管理基于任务、报告模板、监控数据和分析结果生成最终 CPV 报告。
9. 报告生效后，回推或推进任务状态，形成任务闭环。

## 模块状态流转

| 模块 | 状态流转 |
|---|---|
| 年度计划管理 | 草稿 -> 审批中 -> 已生效 -> 失效；支持退回、修订 |
| 任务管理 | 草稿 -> 审批 -> 待执行 -> 进行中 -> 完成 -> 关闭；支持修订 |
| 方案模板 | 草稿 -> 已生效 -> 失效；支持升版 |
| 方案管理 | 草稿 -> 审批 -> 已生效 -> 失效；支持退回、升版 |
| 监控项目 | 草稿 -> 未分析 -> 已分析 -> 已确认 -> 关闭；支持修订 |
| 相关性分析 | 草稿 -> 未分析 -> 已分析 -> 已确认 -> 关闭；支持修订 |
| 报告模板 | 草稿 -> 已生效 -> 失效；支持升版 |
| 报告管理 | 草稿 -> 审批 -> 已生效 -> 失效；支持退回、升版 |

## 模块关系

| 起点模块 | 关系 | 目标模块 | 说明 |
|---|---|---|---|
| 年度计划管理 | 生成 | 任务管理 | 年度计划生效后，根据计划规则生成周期性任务。 |
| 任务管理 | 关联 | 方案管理 | 任务关联具体 CPV 方案，用于跟踪任务执行和方案状态。 |
| 方案模板 | 引用 | 方案管理 | 方案管理引用已生效的方案模板创建标准化方案。 |
| 方案管理 | 定义/生成 | 监控项目 | 方案生效后生成或定义对应监控项目。 |
| 监控项目 | 分析结果引用 | 报告管理 | 监控项目的数据分析结果进入报告编制。 |
| 相关性分析 | 分析结果引用 | 报告管理 | 相关性分析结果进入报告编制。 |
| 报告模板 | 引用 | 报告管理 | 报告管理引用已生效的报告模板生成报告结构。 |
| 任务管理 | 关联 | 报告管理 | 报告与任务关联，报告生效后推进任务状态。 |

## 关键业务规则

- 年度计划未生效前，不应生成正式周期性任务。
- 已失效的年度计划、方案模板或报告模板不应被新任务、方案或报告引用。
- 方案未生效前，不应生成正式监控项目或进入数据分析。
- 监控项目或相关性分析未完成分析或未确认前，不应作为最终报告结论依据。
- 报告编制应基于已确认的监控项目数据、监控项目分析结果或相关性分析结果。
- 审批退回后，记录应回到可修订状态，并保留退回原因。
- 升版应保留历史版本，已生效历史版本不应被覆盖。
- 报告生效后，应推进相关任务状态，并锁定关键报告内容。

## 测试重点

- 各模块状态流转是否符合审批和生效规则。
- 年度计划生效后是否正确生成周期性任务。
- 任务、方案、监控项目、相关性分析、报告之间的关联是否完整且可追溯。
- 模板失效或升版后，历史方案和报告是否保持版本一致性。
- 方案生效后是否正确生成监控项目。
- 监控项目或相关性分析结果是否能被报告正确引用。
- 报告生效后任务状态是否正确推进。
- 退回、修订、升版、失效等分支流程是否保留操作原因和审计追踪。
