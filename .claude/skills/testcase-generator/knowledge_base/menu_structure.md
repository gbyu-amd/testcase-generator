# 系统菜单结构索引

## 维护目标

本文件用于统一 CPV 系统菜单层级和标准模块名。生成测试用例时，用于判断需求所属站点、菜单路径、模块命名和参考用例匹配。

本文件只维护菜单结构和索引信息，不维护详细业务流程、状态流转和测试规则；详细业务规则以 `core_flows/`、`business_glossary.md` 和 `user_roles.md` 为准。

## 维护规则

- 菜单名称以系统界面实际显示为准。
- `标准模块名` 用于生成用例标题、输出文件名和跨文档术语统一。
- `三级菜单` 不存在或暂未明确时填写 `-`。
- 产品数据权限适用范围只在 `user_roles.md` 中维护，本文件不重复维护。
- 若菜单名称和历史参考用例索引不一致，在备注中记录别名或历史名称，不直接改业务标准名。

## 使用提示

- 生成权限类用例时，应优先读取本文件和 `user_roles.md`。
- 生成跨模块链路用例时，应结合本文件和 `core_flows/cpv_business_flow.md`。
- 生成数据分析、一键分析相关用例时，应结合本文件、`core_flows/data_analysis_flow.md` 和 `core_flows/one_click_analysis_flow.md`。
