# 购物车历史缺陷

## 缺陷一：失效商品仍可参与结算

- 影响模块：购物车、结算下单
- 严重程度：高
- 触发条件：商品加入购物车后被商家下架，用户刷新购物车并勾选该商品。
- 历史问题：失效商品仍被计入结算金额。
- 回归关注：失效商品不可勾选，不可进入结算，页面提示商品已失效。

## 缺陷二：修改数量后小计金额错误

- 影响模块：购物车
- 严重程度：高
- 触发条件：用户在购物车中连续增加商品数量。
- 历史问题：数量已更新，但商品小计仍使用旧数量计算。
- 回归关注：数量、单价、小计、总金额必须同步更新。

## 缺陷三：多页签购物车数量不一致

- 影响模块：购物车、多页签操作
- 严重程度：中
- 触发条件：用户在两个页签同时打开购物车，在一个页签删除商品，另一个页签刷新。
- 历史问题：另一个页签仍展示已删除商品。
- 回归关注：刷新后必须展示服务端最新购物车数据。
