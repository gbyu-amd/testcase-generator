# 登录历史缺陷

## 缺陷一：登录态过期后仍可访问购物车

- 影响模块：登录、购物车
- 严重程度：高
- 触发条件：用户登录后长时间停留在购物车页面，登录态过期后继续点击结算。
- 历史问题：前端未及时识别登录态过期，仍允许进入结算页。
- 回归关注：登录态过期后应跳转登录页，不能继续进入结算链路。

## 缺陷二：多页签退出登录状态不同步

- 影响模块：登录、多页签操作
- 严重程度：高
- 触发条件：用户打开两个淘宝页面，在其中一个页面退出登录，另一个页面继续操作购物车。
- 历史问题：另一个页面仍展示登录状态，并允许点击结算。
- 回归关注：退出登录后其他页签需要在关键操作时识别登录状态变化。

## 缺陷三：密码错误提示不清晰

- 影响模块：登录
- 严重程度：中
- 触发条件：用户输入错误密码。
- 历史问题：页面只提示登录失败，未说明账号或密码错误。
- 回归关注：错误提示应明确且不泄露敏感信息。
