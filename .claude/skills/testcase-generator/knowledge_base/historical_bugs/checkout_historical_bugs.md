# 结算下单历史缺陷

## 缺陷一：重复点击提交订单生成多笔订单

- 影响模块：结算下单
- 严重程度：高
- 触发条件：用户在弱网环境下连续点击提交订单按钮。
- 历史问题：系统生成两笔相同商品、相同金额的订单。
- 回归关注：提交后按钮应置灰或加锁，同一请求只能生成一笔订单。

## 缺陷二：优惠券失效后金额未刷新

- 影响模块：结算下单
- 严重程度：高
- 触发条件：用户进入结算页后，优惠券被后台置为失效，再点击提交订单。
- 历史问题：页面仍按旧优惠金额提交订单。
- 回归关注：提交订单前需要重新校验优惠券可用性，并刷新应付金额。

## 缺陷三：不可配送地址仍允许提交订单

- 影响模块：结算下单
- 严重程度：高
- 触发条件：用户选择偏远地区或商家不配送地区地址。
- 历史问题：页面未拦截，订单提交成功。
- 回归关注：不可配送地址应展示提示，并禁用提交订单能力。
