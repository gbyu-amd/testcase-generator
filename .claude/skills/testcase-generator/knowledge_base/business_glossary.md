# 业务术语表

| 术语 | 含义 | 测试关注点 |
|---|---|---|
| 登录态 | 用户完成登录后在前端和服务端保持的身份状态 | 过期、退出、多页签同步 |
| 商品 | 平台售卖的基础对象 | 名称、价格、库存、状态 |
| 规格 | 商品的颜色、尺码、版本等可选属性 | 组合选择、库存刷新、价格刷新 |
| 库存 | 商品当前可购买数量 | 库存不足、库存为零、并发扣减 |
| 购物车 | 用户临时保存待购买商品的容器 | 数量、勾选、金额、失效商品 |
| 结算页 | 用户提交订单前确认商品、地址、优惠和金额的页面 | 金额一致性、地址可配送、优惠可用性 |
| 优惠券 | 可抵扣订单金额的营销权益 | 可用、不可用、过期、门槛不满足 |
| 收货地址 | 用户下单时选择的收货信息 | 默认地址、不可配送、地址为空 |
| 订单 | 用户提交购买请求后生成的交易记录 | 重复提交、状态生成、失败回滚 |
| 商品失效 | 商品被下架、删除或不可购买 | 禁止加购、禁止结算、提示明确 |
| 多页签 | 同一浏览器中打开多个淘宝页面 | 登录态、购物车、商品状态同步 |
