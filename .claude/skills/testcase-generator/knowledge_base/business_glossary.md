# 业务术语表

| 术语 | 含义 | 测试关注点 |
|---|---|---|
| CPV | Continued Process Verification，持续工艺确认，用于在商业化生产阶段持续监控工艺表现和产品质量稳定性 | 计划、任务、方案、监控项目、分析结果和报告之间的数据链路是否完整 |
| CPP | Critical Process Parameter，关键工艺参数，对产品质量有显著影响的生产过程参数 | 参数上下限、趋势变化、异常值、与监控项目和分析结果的关联 |
| CQA | Critical Quality Attribute，关键质量属性，影响产品安全性、有效性和质量可控性的质量指标 | 指标结果、规格限、OOS/OOT 判断、报告引用一致性 |
| CMA | Critical Material Attribute，关键物料属性，可能影响工艺和产品质量的物料属性 | 物料属性维护、批次关联、字段完整性、异常数据影响 |
| IPC | In-Process Control，中间过程控制，生产过程中对关键步骤或中间产品的控制与检测 | 过程检测数据、控制限、数据采集完整性、异常提示 |
| OOS | Out of Specification，检测结果超出既定质量标准或规格限 | 超标识别、告警、审批、CAPA 关联、报告展示 |
| OOT | Out of Trend，检测结果未超规格但偏离历史趋势或预期趋势 | 趋势判断、统计规则、预警、人工确认 |
| CAPA | Corrective and Preventive Action，纠正与预防措施 | 异常触发、措施跟踪、关闭状态、审计追踪 |
| SPC | Statistical Process Control，统计过程控制，用统计方法监控过程稳定性 | 控制图、控制限、趋势规则、异常点识别 |
| 产品 | CPV 监控对象，包含产品名称、剂型、编码和对应工艺路线等信息 | 产品新增、编辑、导入导出、与计划/方案/监控项目的引用关系 |
| 年度计划 | 周期性 CPV 工作的计划定义，用于按规则生成任务 | 草稿、审批、生效、修订、失效、自动生成任务 |
| 任务 | 由年度计划或人工创建的 CPV 执行项 | 状态流转、进度展示、日历视图、方案和报告关联 |
| 方案模板 | 方案编制使用的标准格式或附件模板 | 模板上传、在线编辑、版本变更、被方案引用后的影响 |
| 方案编制 | 基于任务和模板生成具体 CPV 方案，定义监控项目和审批流程 | 方案保存、提交、审批、生效、退回、监控项目生成 |
| 监控项目 | CPV 分析的具体对象或分析项，承载数据源、分析配置、分析结果和确认状态 | 草稿、已分析、已确认状态流转，数据源替换、未分析原因、结果回写 |
| 数据分析 | 对监控项目数据执行统计、趋势、图表或插件化算法分析的功能 | 文件校验、插件状态、字段配置、动态表单、结果保存和回传 |
| 一键分析 | 替换数据源后，系统自动匹配已有分析配置并批量生成分析结果的能力 | 全部成功、部分失败、全部失败、字段匹配、变量类型、正态性、未分析原因优先级 |
| 替换数据源 | 在数据分析详情页不退出分析流程的情况下更换底层数据集 | 弹窗页签、分类树、单选数据源、数据处理规则同步、上次选择回显 |
| 数据管理表 | 来源于数据管理模块的数据表，可作为分析数据源 | 选择后是否清空排序、筛选、转换、异常数据剔除等处理规则 |
| 数据处理表 | 已经过排序、筛选、转换等处理后的数据表，可作为分析数据源 | 选择后是否保留排序、筛选、转换并清空异常数据剔除 |
| 系统集成数据 | 从外部系统集成进入平台的数据表 | 名称/编号搜索、状态校验、字段完整性、失效提示 |
| 本地数据 | 用户上传或维护在平台内的本地文件数据 | 文件名称搜索、分类、状态、字段匹配、文件失效 |
| 数据处理规则 | 对分析数据执行排序、筛选、转换、异常数据剔除等处理的规则 | 替换数据源后的保留或清空逻辑、重新分析触发、规则残留 |
| 分析项配置 | 单个分析项使用的横轴、纵轴、分组、变量类型、分析方法等配置 | 必填校验、字段失效、NULL 残留、配置与数据源匹配 |
| 未分析 | 分析项因字段删减、正态性变化、定类变量不支持等原因未生成结果的状态 | 状态标签、悬停原因、原因优先级、手动修复后清除标记 |
| 字段删减 | 新数据源缺少原分析配置依赖的字段 | 未分析状态、失效字段清空、不显示 NULL、提示字段有删减 |
| 正态性变化 | 替换数据源或数据变换导致检验项数据分布发生变化 | 标记未分析、提示正态性改变、手动重新分析 |
| 定类变量 | 表示分类的数据变量，如类别、分组、阶段等 | 不支持分析时的拦截、提示文案、优先级展示 |
| 定量变量 | 表示连续数值的数据变量，如温度、压力、含量等 | 数值范围、异常值、统计分析适配性 |
| 阶段 | 分析中用于区分工艺阶段、批次阶段或分组阶段的变量定义 | 阶段内容一致性、阶段数限制、阶段变化后的未分析提示 |
| 报告模板 | 报告编制引用的标准模板 | 模板创建、版本、引用关系、内容填充 |
| 报告编制 | 基于报告模板、监控数据和分析结果生成 CPV 报告 | 数据引用一致性、审批流、导出、生效后任务状态更新 |
| 公共管理站点 | 系统初始化自动创建的全局管理站点，站点编号为 PUBLIC_ADMIN | 不可删除、不可停用、只处理全局基础数据、不包含 CPV 业务菜单 |
| 自定义站点 | 承载具体 CPV 业务的业务站点，如分公司、项目、部门等独立单元 | 站点启用/禁用、数据隔离、业务菜单、站点管理员权限 |
| 站点管理 | 公共管理中维护业务站点的功能 | 增删改查、启用/禁用、管理员授权、Logo/封面展示 |
| 统一门户 | 登录公共管理后展示欢迎信息、时间和可访问站点入口的页面 | 站点入口权限、跳转、退出登录、当前站点高亮 |
| 统一认证 | 公共管理站点和自定义站点共用的用户认证体系 | 登录态共享、Token/Session、跨站点免登录、无权限拦截 |
| 全局用户 | 在公共管理系统统一维护且账号全局唯一的用户 | 多站点授权、同步到业务站点、禁用/启用、重复账号校验 |
| 权限/角色 | 控制用户可访问菜单和可操作按钮的授权实体 | 菜单权限、按钮权限、批量删除、分配用户、跨站点权限隔离 |
| 菜单权限 | 控制用户可访问哪些菜单的权限配置 | 菜单展示、URL 直访拦截、不同站点菜单差异 |
| 按钮权限 | 控制用户可执行哪些按钮操作的权限配置 | 按钮显示/置灰、接口权限校验、越权操作拦截 |
| 审计追踪 | 对关键业务操作进行留痕，满足电子记录可追溯要求 | 操作人、时间、模块、操作类型、受影响对象、检索和导出 |
| 电子签名 | 对审批、批准等关键动作进行身份确认的电子签名机制 | 密码策略、签名失败、签名记录、合规追溯 |
| 用户登录日志 | 记录用户登录、登出、登录状态和客户端信息的日志 | 安全审计、异常登录追溯、用户活跃度分析 |
| 系统服务管理 | 公共管理级全局系统配置，包括登录配置、密码策略、电子签名策略和水印配置 | 仅公共管理可操作、对所有业务站点生效、配置变更审计 |
| 系统界面配置 | 公共管理站点的 Logo 等界面个性化配置 | 权限控制、生效范围、业务站点默认配置继承 |
