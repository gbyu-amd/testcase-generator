# 淘宝登录功能 PRD

## 1. 文档信息

| 项目 | 内容 |
|---|---|
| 文档名称 | 淘宝登录功能 PRD |
| 适用端 | Web 端、移动 H5 |
| 关联模块 | 登录、账号体系、购物车、结算下单 |
| 目标用户 | 普通消费者、已注册用户、未登录访客 |
| 文档状态 | 草稿 |

## 2. 需求背景

淘宝登录功能是用户访问个人资产、购物车、收藏、订单、结算下单等核心能力的前置入口。为提升登录成功率、降低异常登录风险，并保障交易链路安全，需要明确登录入口、登录方式、字段校验、异常处理、登录态保持和跨页面跳转规则。

本需求重点覆盖账号密码登录、手机号验证码登录、登录态失效拦截、登录成功后回跳原页面，以及多 Tab 下登录状态同步。

## 3. 需求目标

- 用户可以通过账号密码或手机号验证码完成登录。
- 未登录用户访问受保护页面时，系统应引导用户登录。
- 登录成功后，用户应回到原访问页面或默认首页。
- 登录失败时，页面应给出明确、可理解的错误提示。
- 登录态应在购物车、结算下单等核心链路中保持一致。
- 多 Tab 场景下，登录和退出状态应及时同步。

## 4. 功能范围

### 4.1 本期范围

- 账号密码登录。
- 手机号验证码登录。
- 登录表单字段校验。
- 登录失败提示。
- 登录成功跳转。
- 未登录访问购物车、结算页时的登录拦截。
- 登录态过期后的重新登录。
- 多 Tab 登录态同步。

### 4.2 非本期范围

- 第三方平台授权登录。
- 人脸识别登录。
- 新用户注册流程。
- 忘记密码找回流程。
- 风控后台管理能力。

## 5. 用户角色

| 用户角色 | 说明 |
|---|---|
| 未登录访客 | 未建立有效登录态，可浏览部分商品信息，但不能访问购物车结算等受保护能力 |
| 普通登录用户 | 已完成登录，可访问购物车、订单、收藏、结算下单等功能 |
| 异常账号用户 | 账号被冻结、禁用或存在风险，需要被限制登录或触发额外校验 |

## 6. 功能入口

- 首页顶部“登录”入口。
- 商品详情页“加入购物车”“立即购买”触发登录。
- 购物车页面未登录访问触发登录。
- 结算页未登录访问触发登录。
- 登录态过期后用户继续操作触发登录。
- 用户退出登录后再次访问受保护页面触发登录。

## 7. 页面与字段规则

### 7.1 账号密码登录

| 字段 | 规则 |
|---|---|
| 账号 | 支持手机号、邮箱或淘宝会员名 |
| 手机号 | 必须为 11 位大陆手机号 |
| 邮箱 | 必须符合邮箱格式 |
| 淘宝会员名 | 允许中文、英文、数字和常见符号，长度 4-32 个字符 |
| 密码 | 长度 6-32 个字符，不能为空 |

### 7.2 手机号验证码登录

| 字段 | 规则 |
|---|---|
| 手机号 | 必须为 11 位大陆手机号 |
| 验证码 | 6 位数字，不能为空 |
| 获取验证码按钮 | 手机号格式正确后可点击 |
| 验证码有效期 | 5 分钟 |
| 重新发送间隔 | 60 秒倒计时结束后可重新获取 |

## 8. 主流程

### 8.1 账号密码登录成功

1. 用户打开登录页。
2. 用户选择账号密码登录。
3. 用户输入有效账号和正确密码。
4. 用户点击“登录”按钮。
5. 系统校验账号和密码。
6. 校验通过后建立登录态。
7. 页面跳转到原访问页面；若无原访问页面，则跳转首页。
8. 页面展示用户昵称、头像或已登录状态。

### 8.2 手机号验证码登录成功

1. 用户打开登录页。
2. 用户选择手机号验证码登录。
3. 用户输入有效手机号。
4. 用户点击“获取验证码”。
5. 系统发送验证码并启动倒计时。
6. 用户输入正确验证码。
7. 用户点击“登录”按钮。
8. 系统校验验证码。
9. 校验通过后建立登录态并跳转原访问页面或首页。

## 9. 分支流程与异常处理

| 场景 | 处理规则 |
|---|---|
| 账号为空 | 登录按钮不可提交或提示“请输入账号” |
| 密码为空 | 登录按钮不可提交或提示“请输入密码” |
| 手机号格式错误 | 提示“请输入正确的手机号” |
| 邮箱格式错误 | 提示“请输入正确的邮箱地址” |
| 密码错误 | 提示“账号或密码错误”，不建立登录态 |
| 账号不存在 | 提示“账号或密码错误”或引导注册，不暴露账号是否存在的敏感信息 |
| 账号被冻结 | 提示“账号状态异常，请联系客服” |
| 验证码错误 | 提示“验证码错误，请重新输入” |
| 验证码过期 | 提示“验证码已过期，请重新获取” |
| 登录接口失败 | 提示“登录失败，请稍后重试”，保留用户已输入账号 |
| 网络超时 | 提示网络异常，允许用户重新提交 |
| 登录态过期 | 拦截当前操作并跳转登录页，登录成功后回到原页面 |

## 10. 页面跳转规则

- 用户从首页主动点击登录，登录成功后默认跳转首页。
- 用户从商品详情页点击“加入购物车”触发登录，登录成功后回到该商品详情页并继续展示原商品信息。
- 用户从购物车页触发登录，登录成功后回到购物车页并刷新购物车数据。
- 用户从结算页触发登录，登录成功后回到结算页；若商品状态、库存、价格或优惠发生变化，需重新计算并提示用户确认。
- 登录失败时停留在登录页，不跳转。
- 登录态过期后重新登录成功，应回到触发拦截的页面。

## 11. 与其他模块联动

| 关联模块 | 联动规则 |
|---|---|
| 商品详情 | 未登录点击加入购物车或立即购买时触发登录；登录成功后保留商品上下文 |
| 购物车 | 登录成功后刷新当前用户购物车数据和购物车角标 |
| 结算下单 | 未登录或登录态过期时禁止提交订单；重新登录后需重新校验商品、库存、价格、地址和优惠 |
| 用户信息 | 登录成功后展示用户昵称、头像或账号标识 |
| 多 Tab | 一个 Tab 登录或退出后，其他 Tab 应同步更新登录状态 |

## 12. 安全与风控要求

- 密码输入框必须以密文形式展示。
- 登录失败次数过多时，应触发图形验证码或其他安全校验。
- 不应在错误提示中暴露账号是否存在等敏感信息。
- 登录接口返回异常时，不应泄露后端错误堆栈或内部错误码。
- 退出登录后，受保护页面不能继续提交订单、修改购物车或查看个人订单。

## 13. 验收标准

| 编号 | 验收标准 |
|---|---|
| AC_LOGIN_001 | 用户输入正确账号和密码后，可以登录成功，并展示已登录状态 |
| AC_LOGIN_002 | 用户输入错误密码后，登录失败，页面展示明确错误提示，系统不建立登录态 |
| AC_LOGIN_003 | 用户输入错误手机号格式时，页面应提示手机号格式不正确 |
| AC_LOGIN_004 | 用户使用正确手机号和验证码后，可以登录成功 |
| AC_LOGIN_005 | 验证码错误或过期时，系统应阻止登录并展示对应提示 |
| AC_LOGIN_006 | 未登录用户访问购物车或结算页时，应跳转登录页或弹出登录引导 |
| AC_LOGIN_007 | 登录成功后，应回到原访问页面；无原访问页面时跳转首页 |
| AC_LOGIN_008 | 登录态过期后继续提交订单时，系统应拦截操作，不生成订单 |
| AC_LOGIN_009 | 一个 Tab 退出登录后，其他 Tab 再执行受保护操作时应识别未登录状态 |
| AC_LOGIN_010 | 登录接口失败或网络异常时，页面应提示稍后重试，不清空用户已输入账号 |

## 14. 数据埋点建议

| 事件 | 触发时机 |
|---|---|
| login_page_view | 用户进入登录页 |
| login_submit | 用户点击登录按钮 |
| login_success | 用户登录成功 |
| login_fail | 用户登录失败 |
| sms_code_send | 用户点击获取验证码 |
| login_redirect_back | 登录成功后回跳原页面 |
| login_session_expired | 登录态过期触发拦截 |

## 15. 待确认问题

- 是否需要支持扫码登录。
- 登录失败达到多少次后触发二次校验。
- 手机验证码发送频控策略是否按账号、手机号、设备或 IP 限制。
- 登录态有效期和刷新机制是否由服务端统一控制。
- 多 Tab 登录态同步是否要求实时同步，还是在用户操作时被动识别。
