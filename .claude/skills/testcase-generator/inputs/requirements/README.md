# 需求文档说明

## 存放内容

本目录用于存放淘宝项目的产品需求文档、需求变更说明、版本说明和功能补充说明。

## 建议命名

- `登录流程优化需求.md`
- `购物车批量删除需求.md`
- `结算页优惠券改版需求.md`
- `商品详情规格选择需求.md`

## 阅读重点

读取需求文档时，应重点提取：

- 需求背景
- 功能入口
- 涉及用户角色
- 主流程
- 分支流程
- 字段校验规则
- 异常处理规则
- 页面跳转规则
- 与其他模块的联动关系
- 验收标准

## 用例生成要求

需求文档中的每一条验收标准至少应转化为一条测试用例。涉及核心交易链路的需求必须补充异常、边界、权限和回归场景。
