# 需求文档说明

## 存放内容

本目录用于存放 CPV 系统的产品需求文档、需求变更说明、版本说明、功能补充说明和验收标准。需求可覆盖公共管理站点、自定义业务站点、年度计划、任务、方案、监控项目、数据分析、一键分析、报告编制、审计追踪、电子签名和异步任务等模块。

## 建议命名

- 整份原始 PRD：`CPV产品规格说明书 (0615).md`
- 自动拆分索引：`requirements_index.md`
- 拆分后单需求：`<site_type>/<module_slug>/REQ-CPV-001.md`

推荐结构：

```text
requirements/
  CPV产品规格说明书 (0615).md
  requirements_index.md
  public_site/
    audit_trail/
      REQ-CPV-001.md
  business_site/
    data_analysis/
      REQ-CPV-002.md
```

整份 PRD 保留原文不改；按小章节拆出的 `REQ-*` 文件作为后续生成测试用例的主要输入。

## 自动拆分

在技能根目录执行：

```bash
python scripts/split_requirements.py --prd "inputs/requirements/CPV产品规格说明书 (0615).md"
```

脚本默认按 3/4 级标题拆分需求，并为每个需求创建对应 UI 目录 README。拆分后必须先检查 `requirements_index.md`，确认章节边界、站点分类、模块目录和 UI 归属。

## 阅读重点

读取需求文档时，应重点提取：

- 需求背景
- 功能入口
- 涉及用户角色
- 涉及站点范围（公共管理 / 自定义业务站点）
- 菜单权限和按钮权限
- 主流程
- 分支流程
- 字段校验规则
- 异常处理规则
- 状态流转规则
- 审批流和电子签名规则
- 审计追踪要求
- 数据来源、数据处理和数据分析规则
- 页面跳转规则
- 与其他模块的联动关系
- 异步任务、导入导出、结果推送和失败重试规则
- 数据隔离、版本管理和历史记录要求
- 验收标准

## 用例生成要求

需求文档中的每一条验收标准至少应转化为一条测试用例。涉及 CPV 核心业务链路、合规追溯、数据完整性、站点隔离或报告结论准确性的需求必须补充异常、边界、权限、联动和回归场景。

以下需求点必须优先覆盖：

- 年度计划生效后是否正确生成周期性任务。
- 任务、方案、监控项目、分析结果和报告之间的引用关系是否一致。
- 方案或报告审批、批准、生效是否符合工作流和电子签名要求。
- 监控项目未分析、未确认或数据源失效时，是否禁止进入最终报告结论。
- 数据分析文件校验、插件状态、字段配置、动态表单和结果保存/回传是否符合规则。
- 替换数据源和一键分析是否正确处理字段删减、变量类型、正态性变化和未分析原因优先级。
- 公共管理站点和自定义业务站点的菜单边界、权限范围和业务数据隔离是否正确。
- 审计追踪是否记录操作人、时间、模块、操作类型、受影响对象、操作前后值和失败原因。
