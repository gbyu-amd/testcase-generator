## 原始需求内容

### 相关性分析

【需求背景】

在方案编制中定义相关性分析后，方案生效需在“方案执行”下提供独立的相关性分析管理模块。用户需要执行这些分析、查看结果、并根据分析结果创建监控项目。同时需支持批量操作、状态流转、与监控项目的联动。

【需求描述】

|  |  |
| --- | --- |
| **项目** | **内容** |
| **需求描述** | **1. 页面布局与筛选** • 左侧产品列表：选择产品后，右侧联动显示该产品下的方案（下拉选择）。 • 顶部筛选：支持按分析状态（全部、未分析、已分析、已确认、已关闭、修订中）筛选；支持按工艺参数名称、编号搜索。 • 列表单页默认50条数据，支持翻页。  **2. 列表字段** 全选、编号、工艺参数名称、分析状态、参数类型、数据源属性信息、方案名称（超链接跳转）、统计分析工具、分析结果名称（点击可查看分析结果）、任务名称（超链接跳转任务详情）、操作栏（按钮随状态变化）。  **3. 页面操作按钮** • 批量删除：仅允许删除“未分析”状态记录。 • 批量分析：仅允许对“未分析”或“修订中”或“已分析”状态记录执行批量分析。  **4. 操作栏按钮（详见状态操作矩阵）** • 未分析：创建监控项目、数据分析、删除 • 已分析：创建监控项目、数据分析、确认、关闭 • 已确认：创建监控项目、变更 • 修订中：创建监控项目、数据分析 • 关闭需填写关闭原因；删除需二次确认。  **5. 创建监控项目** 点击“创建监控项目”弹出弹窗（字段见下表）。系统自动带入相关性分析结果中的变量： •  **Pearson相关性分析**：按相关系数绝对值从大到小排序，展示每个变量对，监控项目列表需要将每个等级变量对列出来，去重。 • **相关性分析**：按影响程度（例如影响系数）排序，展示每个影响因素的监控项目，这里只展示预测变量的监控项目列表） 用户可勾选需生成的监控项，配置分析方法（默认百分位数控制图），并支持批量添加/删除行。 • **去重规则**：同一方案下，监控项目名称+分析方法组合不可重复（包括本次提交内重复及与已有监控项目重复）。 • **颜色标记**：强相关颜色为红色，中等相关颜色为黄色，弱相关颜色为白色 **6、其他数据分析等一系列功能同之前监控项目的功能。** |
| **异常情况说明** | 1、出现重复的监控项目（同名+同分析方法） → 提示“监控项目已存在，无法重复添加”。 |
| **后置处理功能** | 1. 执行数据分析后，分析结果自动存储，“分析结果名称”字段可点击查看详情。 2. 创建监控项目成功后，自动刷新所属方案的监控项目列表，无需记录审计日志。 3. 确认分析结果后，状态变更为“已确认”，不可再编辑。 4. 发起变更后状态进入“修订中”，可重新分析并再次确认。 5. 删除相关性分析记录时，若已生成监控项目，仅删除分析记录，监控项目保留。 6. 批量确认功能同监控项目。 |

**分析状态操作矩阵**

|  |  |  |
| --- | --- | --- |
| **当前状态** | **操作按钮** | **说明** |
| **未分析** | 数据分析、删除 | 可手动执行分析；删除需二次确认；创建监控项目基于分析结果。未分析时，无创建监控项目的按钮。已分析和已确认可以创建。 |
| **已分析** | 创建监控项目、数据分析、确认、关闭 | 可重新分析（覆盖原结果）；确认后变已确认；关闭需填原因 |
| **已确认** | 创建监控项目、变更 | 变更后进入“变更中”；仍可基于原结果创建监控项目 |
| **变更中** | 创建监控项目、数据分析 | 重新分析后可再次确认 |
| **已关闭** | 无（仅查看） | 所有操作按钮不可见/置灰，只能查看分析结果 |

1、相关性分析模块支持权限配置，涉及到的按钮为：创建监控项目、数据分析、删除、批量分析、批量删除、确认、关闭、数据分析、批量确认。

2、批量确认功能同监控项目模块。
