## 原始需求内容

### 任务执行与状态跟踪（CPV-TM-01）

**任务管理模块详细功能说明**

|  |  |
| --- | --- |
| **项目** | **描述** |
| **模块功能概述** | 展示所有具体的执行项，区分周期性任务和单次任务。提供日历视图，管理任务状态，并与方案、报告联动。任务有两种生成方式：年度计划自动生成的周期性任务，以及本模块手动创建的单次任务。 |
| **需求描述** | **1. 页面布局** • 左侧为产品列表（取自产品管理模块），点击产品可筛选该产品下的任务。 |
| **页面交互** | • 点击列表行进入详情页，详情页顶部进度条根据当前任务状态高亮对应节点。 |
| **异常逻辑** | • 关闭任务时，若任务已关联方案且方案未完成，需提示“该任务已关联进行中的方案，请先完成方案”。 |
| **后置流程处理** | • 任务状态更新时，系统自动发送通知给提醒对象（根据提醒天数提前发送）。 |

**新增任务字段详情表（单次任务）**

**新增页面，顶部展示CPV任务状态，具体样式已设计为准**

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| **字段名称** | **输入类型** | **必填** | **默认值** | **说明** |
| 任务名称 | 文本框 | 是 | 无 | 最多200个字符，用户自定义。 |
| 所属产品 | 下拉选择 | 是 | 无 | 从产品管理模块读取，单选。若从左侧产品树进入，自动填充。 |
| 任务类型 | 只读 | 是 | 单次 | 系统自动填充，不可更改。 |
| 计划日期 | 日期选择器 | 是 | 无 | 任务计划执行的日期。 |
| 提醒天数 | 数字输入框 | 是 | 7 | 任务开始前多少天发送提醒。 |
| 提醒对象 | 用户选择器 | 是 | 无 | 可选择多个用户/角色/部门，接收任务提醒。 |
| 描述 | 文本域 | 否 | 无 | 任务描述信息，最大支持300个字符。 |
| 状态 | 只读 | 是 | 草稿 | 初始状态。 |
| 创建成员 | 只读 | 是 | 当前用户 | 系统自动记录。 |

**说明**：

实际开始时间、实际结束时间，关闭原因，关闭人在创建提交之后的预览页面才会由此信息展示。周期性任务由系统自动生成，不可手动新增，其字段继承自年度计划，且不可编辑。

**任务状态操作矩阵（无相关权限的按钮不可见）**

|  |  |  |
| --- | --- | --- |
| **当前状态** | **操作按钮** | **说明** |
| **草稿** | 编辑、删除、提交、复制 | 单次创建任务确认提交后，变为草稿状态， 单次任务提交后进入“审批中”状态，周期性任务直接进入“待执行”状态。 |
| **审批中** | 申请人：撤回、复制 审批人：同意、退回，转审 | 审批中不可编辑。审批通过后进入“待执行”，被退回后回到“已退回”。 |
| **已撤回** | 编辑、删除、提交、复制 | 已撤回状态下，用户可重新提交 |
| **已退回** | 编辑，提交、复制 | 编辑后可再次提交，重新进入“审批中” |
| **待执行** | **关闭**、**开始任务**、复制 | 关闭需填写原因，点击开始任务时，进入进行中的状态 |
| **进行中** | 关闭、复制 | 只读状态，所有操作通过关联方案、关联报告进行。关闭需要填写关闭原因。 |
| **已完成** | 复制 | 此状态为自动完成，关联的方案&报告字段均不为空，且方案和报告的状态均为“已生效”。 状态为只读。 |
| **修订中** | 复制 | 只读状态，等待关联方案/报告升版完成后自动变回“已完成”。 已完成状态下，若关联的方案或报告执行了升版操作，任务状态从“已完成”自动变为“修订中”，待新版文档全部生效后再次变回“已完成”。 **注意：** 1）任务已完成状态下，若关联的方案执行了升版操作，报告没有变化，任务进度为方案编制；（方案编制审批完成之后，方案执行，执行之后由于报告没有升版，直接跳转至已完成状态） 2）任务已完成状态下，若关联的报告执行了升版，方案没有变化，任务进度为报告编制； 3）任务已完成状态下，若关联的报告和方案都执行了升版，任务进度为方案编制； 4）所有任务关联的方案和报告都审批通过之后，任务状态变成已完成； |
| **已关闭** | 复制 | 只读状态。待执行状态下，用户点击关闭并填写关闭原因。 |

**补充说明**：

**任务状态与关联模块的联动逻辑**

|  |  |  |
| --- | --- | --- |
| **触发动作** | **任务状态变化** | **说明** |
| 年度计划生效 | 自动生成任务 -> 待执行 | 周期性任务初始状态为待执行。 |
| 单次任务提交 | 草稿 -> 审批中 | 进入审批流程。 |
| 审批通过 | 审批中 -> 待执行 | 审批通过，点击开始任务进入待执行。 |
| 关联方案开始编制 | 待执行 -> 进行中 | 用户在方案编制模块创建方案并关联本任务时触发。 |
| 关联方案生效 | 进行中 -> 进行中 | 方案生效后任务仍为进行中，等待报告。 |
| 关联监控项目分析完成 | 进行中 -> 进行中 | 监控项目分析完成之后，状态为进行中 |
| 关联报告生效 | 进行中 -> 已完成 | 方案和报告均生效后，任务自动完成。 |
| 已完成状态下方案/报告升版 | 已完成 -> 修订中 | 当已生效的方案或报告被升版时触发。 |
| 升版后的方案/报告生效 | 修订中 -> 已完成 | 升版完成且生效后，任务回到已完成。 |
| 用户点击关闭 | 待执行 -> 已关闭 | 用户主动关闭任务。 |

**注意**：任务状态的变化应通过后台事件驱动，减少人工干预，确保数据一致性。
