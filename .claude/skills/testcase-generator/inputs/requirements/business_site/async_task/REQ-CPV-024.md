## 原始需求内容

### 异步任务处理相关（CPV- ATP-01）

功能功能概述：

处理耗时操作（如批量导出监控项目、批量导入数据、报告生成等），将其转为异步任务执行，避免前端长时间等待。用户可在本模块查看任务状态、进度、执行结果，并支持失败任务的详情查看与重试。

**1、模块详细功能描述**

|  |  |
| --- | --- |
| **项目** | **描述** |
| **需求描述** | **1. 页面布局** • 顶部提供状态快速切换页签：全部、进行中、已完成、失败。 • 页面上方为搜索框，支持按**任务对象**进行模糊搜索（如按导出的监控项目名称、关联的任务名称等）。 • 下方为异步任务列表，默认每页显示50条数据。  **2. 列表属性信息** 序号、任务对象、模块（如监控项目、报告编制）、任务类型（如批量导出、批量导入、报告生成）、任务状态（进行中/已完成/失败）、创建人、创建时间、操作栏（根据状态显示不同操作）。  **3. 操作栏说明** • **进行中**：显示“/”（无操作）。 • **已完成**：显示“导出文件”按钮（若任务类型为导出），点击可下载生成的文件；对于其他类型任务，可显示“查看结果”按钮。 • **失败**：显示“查看详情”按钮，点击弹出失败详情弹窗，展示本次任务成功处理了多少个、失败了多少个，以及失败的具体原因列表。 |
| **页面交互** | • 状态切换页签支持快速筛选，默认进入时显示“全部”状态。 • 搜索框支持实时搜索或点击搜索按钮触发。 • 点击“查看详情”弹窗中，失败原因以列表形式展示，每条包含失败对象的标识（如监控项目编号）和失败原因描述，便于用户定位问题。 |
| **异常逻辑** | • 任务执行过程中系统异常崩溃，任务状态应标记为“失败”，并在详情中记录错误信息。 • 导出文件超过系统限制大小时，提示“文件过大，请分批导出”并标记失败。 |

**状态操作矩阵**

|  |  |  |
| --- | --- | --- |
| **当前状态** | **操作按钮** | **说明** |
| **进行中** | 查看（或取消） | 查看任务基本信息，可选支持取消任务。 |
| **已完成** | 导出文件 / 查看结果 | 对于导出类任务，提供文件下载； |
| **失败** | 查看详情 | 查看失败详情，包含成功/失败统计及失败原因列表。 |

**说明**：操作按钮受权限控制，仅任务创建人或有管理员权限的用户可查看详情和下载文件。

此处一步任务处理模块，只显示当前登录者的异步任务，系统总管理员可查看所有的异步任务。
