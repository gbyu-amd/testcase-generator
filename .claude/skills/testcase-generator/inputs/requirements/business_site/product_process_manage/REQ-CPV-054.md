## 原始需求内容

### 产品工艺管理模块兼容老数据

【需求背景】

原产品管理模块已支持产品基础信息（产品编号、名称）及关联的工艺参数（CPP、CQA、CMA）的维护。本次升级为产品工艺管理模块，新增了剂型、规格、工艺步骤、工艺流程、关键设备、工艺矩阵等复杂功能。为确保平滑升级，需兼容旧版本数据，使原有产品信息和工艺参数无缝迁移至新模块，同时保留原有编辑、删除、导入导出等操作能力，并支持用户后续补充新功能所需的工艺步骤等数据。

【需求描述】

|  |  |
| --- | --- |
| **名称** | **需求描述** |
| **需求说明** | **1. 数据迁移兼容** • 原有产品管理表数据迁移至新产品工艺管理表，自动生成默认值：（剂型、规格、生产商改为非必填项，对应导入导出的规则需进行调整） |
| **页面布局** | 与原产品工艺管理模块设计一致：左侧产品列表（含搜索、新增、导入导出），右侧五个TAB页签。迁移后的旧数据在各页签中展示： |
| **异常情况处理** | 1. 数据迁移时若质量标准无法解析为LSL/USL，则LSL和USL保留为空，并在质量标准字段中保留原文本，迁移日志记录警告。 |
| **后置处理功能** | 1. 提供一键数据迁移工具，可后台执行，迁移完成后输出报告。 |
