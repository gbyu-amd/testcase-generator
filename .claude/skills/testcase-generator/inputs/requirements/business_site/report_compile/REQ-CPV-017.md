### 报告编制与审批相关（CPV-RP-01）

需求描述

| **需求描述** | **1. 页面布局** • 左侧为产品列表（取自产品管理模块），点击产品可筛选该产品下的报告。 • 右侧为报告列表，默认每页显示50条数据。 • 列表支持按报告名称、报告编号、所属产品进行精确或模糊搜索。 • 顶部提供状态快速切换页签：全部、草稿、审批中、升版中、已生效、已失效。  **2. 列表字段** 报告名称、报告编号、版本号、所属产品、报告状态、关联方案、关联CPV任务、报告文档、创建人、创建时间。 • **交互**：鼠标悬停在“报告文档”名称上时，点击超链接可快速跳转至报告的ONLYOFFICE预览页面，支持在线编辑。  **3. 新增报告编制（四个页签）** • **页签1-详情**： - 报告状态（回显）、版本号（回显）、所属产品、报告名称、报告模版。 - 点击“报告模版”后的选择按钮，弹出选择报告模版弹窗，自动过滤与所选产品相同的模版。弹窗列表：单选框、模版名称、模版编号、所属产品、模版状态。选择后回显模版名称、编号、所属产品、状态、版本号，并提供快速跳转至模版详情页的入口。 • **页签2-关联CPV任务**： - 点击“添加CPV任务”按钮，弹出选择CPV任务弹窗，自动筛选与所选产品相同且状态为 **“待执行”或“进行中”** 的任务（**不可关联已关闭、已完成或草稿的任务**）。弹窗列表：单选框、任务名称、编号、任务状态、产品、计划日期。选择后回显任务名称、编号、状态、产品、计划日期。 • **页签3-关联方案**：（只读状态，不可编辑） - 自动显示所选任务关联的方案，回显方案名称、编号、状态、版本号。 • **页签4-监控项目**： - 显示所选方案中关联的监控项目列表：序号、编号、分析状态、监控项目名称、监控类型、质量标准、标准类型、统计分析工具、备注。  **4. 预览报告编制页面** 展示以下信息：报告名称、报告编号、报告版本号、所属产品、报告模版（模版编号、模版名称、所属产品、模版状态、版本号）、创建成员、创建时间、生效人、生效时间、失效人、失效时间、失效原因、审批明细表。  **5. 报告详情页面（ONLYOFFICE集成）** • 报告打开时进入ONLYOFFICE文件预览页面，点击“编辑”按钮可进入编辑模式。 • **编辑功能**： - 编辑正文内容、格式设置。 - **插入附件和附件清单**：（附件清单列表做成可编辑区域，若用户发现附件页码不对，可手动调整） 同QRS系统原有功能。用户可上传附件（仅支持PDF和图片），在线创建附件，选择插入附件到正文。插入附件清单时，在光标位置自动生成表格，表头：类型（系统上传/在线创建）、附件名称、附件编号、附件页码。 |
| **页面交互** | • 列表页鼠标悬停报告文档名称可快速跳转至ONLYOFFICE预览页面。 • 新增报告时，各页签间数据联动：选择产品 → 选择任务 → 自动带出关联方案 → 自动带出监控项目。 • 弹窗选择器（任务、模版）支持搜索和单选。 |
| **异常逻辑** | • **任务关联限制**：选择CPV任务时，弹窗仅显示状态为 **“待执行”或“进行中”** 的任务。若任务状态变更导致不符合条件，已选任务在报告中会提示“任务状态已变更，请重新选择”。 • **任务被删除处理**：若已关联的CPV任务在报告编辑过程中被删除（如计划失效导致任务删除），系统在保存或提交报告时需校验任务是否存在。若任务不存在，拦截并提示“关联任务已不存在，请重新选择”。（此处删除的处理逻辑和 • **监控项目状态校验**：提交报告审批时，系统自动校验关联方案下的所有监控项目状态是否为 **“已确认”**。若存在任一监控项目状态不是“已确认”，则拦截提交并提示**“存在未确认的监控项目（共X个），请先完成监控项目确认”。** • **任务关联**：若所选任务已关联其他报告，此处选择任务将不可见。 • **必填项校验**：提交审批时，若未选择报告模版或未关联任务，系统拦截并提示。 |
| **后置流程处理** | • 报告生效后，推动关联任务状态更新（如从“进行中”变为“已完成”）。 • 报告失效时，若关联任务未完成，需给出提示并允许强制失效或阻止操作。 |

**补充说明：页签间数据联动关系**

**页签1选择产品** → 影响页签2任务弹窗的筛选范围（只显示该产品的任务）。

**页签2选择任务** → 自动触发页签3加载该任务关联的方案。

**页签3显示方案** → 自动触发页签4加载该方案下的所有监控项目列表。

若修改页签1的产品，已选的页签2任务应清空并提示“产品已变更，请重新选择任务”。

**异常逻辑补充（针对页签间联动）**

|  |  |
| --- | --- |
| **场景** | **处理方式** |
| 页签2选择任务后，该任务被删除（如计划失效导致） | **希望：**任务已被关联不允许删除，失效。 **备用：**页签2、3、4内容清空，并提示“关联任务已不存在，请重新选择”。 |
| 页签3显示的方案状态变更为“已失效” | 页签4监控项目列表置灰，并提示“关联方案已失效，请检查”。 |
| 提交报告时，页签4中存在分析状态不为“已确认”的监控项目 | 拦截提交，提示“存在未确认的监控项目（共X个），请先完成监控项目确认”。 |

**报告状态与可编辑性说明**

|  |  |  |
| --- | --- | --- |
| **报告状态** | **是否可编辑** | **说明** |
| **草稿** | ✅ 可编辑 | 初始状态，用户可编辑所有内容、删除、提交审批。 |
| **审批中** | ❌ 不可编辑 | 已提交审批，等待审批人处理。创建人可“撤回”回到草稿。 |
| **已退回** | ✅ 可编辑 | 审批未通过，退回后用户可修改后重新提交。 |
| **已撤回** | ✅ 可编辑 | 创建人在审批中主动撤回，回到草稿状态。 |
| **已生效** | ❌ 不可编辑 | 审批通过，报告正式生效。可进行“升版”操作。 |
| **升版中** | ✅ 可编辑 | 对已生效报告点击“升版”后进入，编辑新版本内容并重新提交审批。原版本仍为已生效。 |
| **已失效** | ❌ 不可编辑 | 升版后旧版本自动失效，或手动失效，只读状态。 |

**报告状态操作矩阵**

|  |  |  |
| --- | --- | --- |
| **当前状态** | **操作按钮** | **说明** |
| **草稿** | 编辑、删除、提交、复制 | 提交后进入“审批中”。 |
| **审批中** | 申请人：撤回 审批人：同意、退回 复制 | 撤回后回到“草稿”；审批通过后进入“已生效”；被退回后进入“已退回”。 |
| **已退回** | 编辑、提交、复制 | 编辑后可再次提交，重新进入“审批中”。 |
| **已撤回** | 编辑、提交、复制 | 同草稿状态，可编辑后重新提交。 |
| **已生效** | 升版、失效、复制 | 升版生成新版本，新版本状态为“升版中”；失效需填写原因，状态变为“已失效”。 |
| **升版中** | 编辑、提交、复制 | 编辑新版本草稿并提交审批，提交后进入“审批中”。 |
| **已失效** | 查看、复制 | 复制生成新的草稿记录，新记录状态为“草稿”。 |

**说明**：所有操作按钮均受权限控制，仅拥有相应权限的用户可见。“提交”操作可能触发审批流程，具体由工作流配置决定。
