## 原始需求内容

### 报告中插入分析结果相关优化

【业务背景】

当前报告编制模块中，插入分析结果时存在以下问题：分析图展示尺寸过小，无法满足A4纸打印及阅读需求；图表数据点模糊、压缩，影响数据可读性；批量插入分析结果数量无限制，可能导致报告体积过大或性能问题。需对插入分析结果的功能进行整体优化。

【需求描述】

|  |  |
| --- | --- |
| **项目** | **描述** |
| **需求描述** | **1. 分析图展示样式优化** • 分析图需增加**外围框线**（边框），与报告正文内容形成视觉区分。 |
| **页面交互** | • 分析图插入后，用户可在报告中**拖拽调整位置**，但尺寸应锁定为A4宽度（不可手动缩窄）。 |
| **异常逻辑** | • 批量插入过程中若因网络或系统原因中断，需提示“部分分析结果插入失败，请重试”。 |
| **后置流程处理** | • 插入完成后，报告或附件自动保存。 • 若因插入数量过多导致报告体积过大，系统应自动触发**异步处理**（如生成报告后压缩），避免前端卡顿。 |

**分析图样式规格表**

|  |  |  |
| --- | --- | --- |
| **参数** | **规格要求** | **说明** |
| **外围框线** | 1px实线边框，颜色#333或与报告主题一致 | 与正文内容形成视觉区分 |
| **宽度** | 100%父容器宽度（A4纸有效宽度） | 约占16cm-18cm，按页面边距自适应 |
| **高度** | 自适应（保持图表宽高比） | 通常为宽度的0.6-0.75倍 |
| **数据点** | 矢量渲染（SVG）或高清位图（≥150dpi） | 不可出现压缩、拉伸、模糊 |
| **字体** | ≥10px | 坐标轴标签、图例等可读性要求 |

**批量插入限制逻辑表**

|  |  |  |  |
| --- | --- | --- | --- |
| **场景** | **限制数量** | **提示文案** | **处理方式** |
| 报告中批量插入 | ≤100 | — | 正常插入 |
| 报告中批量插入 | >100 | “单次最多支持插入100个分析结果，当前已选择X个，请分批插入。” | 拦截操作，需用户调整 |
| 附件中批量插入 | ≤100 | — | 正常插入 |
| 附件中批量插入 | >100 | 同上 | 拦截操作，需用户调整 |

**说明**：以上优化需同时应用于**ONLYOFFICE编辑器中的报告正文**及**附件预览/编辑区域**。
