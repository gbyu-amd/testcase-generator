## 原始需求内容

### 能力分析等级配置

【需求背景】

在CPV系统的过程能力分析中，用户需要根据计算出的Cpk/Ppk值对工艺能力进行等级评定（如A++级、A+级、A级等），以直观判断过程能力水平。目前系统内置固定等级标准，无法灵活适应不同产品、不同客户的差异化要求。因此需在“配置管理”下新增“质量规则配置”二级菜单，其中“能力分析等级配置”页签用于集中管理Cpk和Ppk的等级划分规则，支持用户自定义等级区间、等级名称及说明，并在过程能力分析结果中自动应用对应的等级结论。

|  |  |
| --- | --- |
| **名称** | **需求描述** |
| **需求描述** | **一、菜单入口**：在系统左侧导航栏“配置管理”一级菜单下，新增“质量规则配置”二级菜单。点击后进入页面，包含两个页签：“参考限配置”（已另文档描述）和“能力分析等级配置”。默认选中“能力分析等级配置”。 |
| **页面布局** | 能力分析等级配置页签采用双卡片布局（左右或上下两栏）。每个卡片内显示以下信息： |
| **交互说明** | - 编辑弹窗中，等级列表支持新增行（点击“添加等级”按钮）、删除行（点击行内删除图标）。 - 最小值/最大值输入框支持负数、小数。 - “是否包含最小值/最大值”使用开关或复选框（“是/否”）。 - 表单实时校验，错误时提示具体位置。 - 保存前校验区间连续性、覆盖性，若校验失败，提示错误原因（如“区间有重叠”或“未全覆盖”）。 - 预览时不保存，仅展示当前配置的等级区间。 - 操作历史记录弹窗以时间倒序显示审计日志。 |
| **异常逻辑** | 1. 等级名称重复 → 提示“等级名称不能重复”。 2. 最小值与最大值均为空 → 提示“至少填写最小值或最大值”。 3. 区间重叠 → 提示“等级区间存在重叠，请调整”。 4. 区间未全覆盖（存在间隙） → 提示“等级区间未覆盖全部数值范围，请补充”。 5. 最小值 ≥ 最大值（当两者均存在时） → 提示“最小值必须小于最大值”。 6. 删除最后一个等级 → 提示“至少保留一个等级”。 7. 描述/说明为空 → 提示“描述不能为空”。 8. 默认等级规则不可删除至空，但允许修改。 |
| **后置处理功能** | 1. 保存后更新卡片的“更新人”、“更新时间”、“等级数量”。 2. 规则变更后，后续过程能力分析（控制图、监控项目能力图等）中的等级判定按照新规则执行，并记录审计日志（操作人、时间、变更前后规则摘要）。 3. 提供数据初始化脚本，用于新装或升级时创建默认6个等级（CPK和PPK共用同一套默认规则，但可独立编辑）。 4. 审计追踪模块增加“质量规则配置”的操作类型，记录编辑、预览（可选）等操作。 5、配置生效范围： 过程能力（正态）：CPK、PPK等级 过程能力（非正态）：非参数法（CPK），WEIBULL拟合（PPK） |

|  |  |  |  |

| **列名** | **输入类型** | **必填** | **说明** |

| 操作 | 按钮 | — | 删除该等级行 |

**默认等级数据表（CPK与PPK共用初始值）**

|  |  |  |  |  |  |
| --- | --- | --- | --- | --- | --- |
| **等级名称** | **最小值** | **是否包含最小值** | **最大值** | **是否包含最大值** | **描述/说明** |
| A++级 | 2.0 | 是 | 空（+∞） | 是 | 特优，可考虑成本的降低 |
| A+级 | 1.67 | 是 | 2.0 | 否 | 优，应当保持之 |
| A级 | 1.33 | 是 | 1.67 | 否 | 良，能力良好，状态稳定，但应尽力提升为A+级 |
| B级 | 1.0 | 是 | 1.33 | 否 | 一般，状态一般，制程因素有变异即有产生不良的危险，应利用各种资源及方法将其提升为A级 |
| C级 | 0.67 | 是 | 1.0 | 否 | 差，制程不良较多，必须提升其能力 |
| D级 | 空（-∞） | — | 0.67 | 否 | 不可接受，其能力太差，应考虑重新整改设计制程 |

注：默认区间覆盖 (-∞, +∞)。D级最小值负无穷，不含下界；A++级最大值正无穷，含上界。实际存储时用特殊值表示无穷（如null）。等级顺序按数值递增排列。
