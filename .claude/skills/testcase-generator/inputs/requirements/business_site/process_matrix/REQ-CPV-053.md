### 工艺矩阵

【需求背景】

工艺矩阵模块用于从宏观角度展示产品工艺的整体结构，包括工序、操作、设备、参数之间的关联关系，帮助质量/工艺人员快速理解工艺全貌，识别关键控制点。当前工艺数据分散在多个页签，缺乏统一的可视化视图，导致工艺关联性难以直观掌握。通过工艺矩阵，用户可一站式查看工序与各要素的关联，并支持点击下钻。

【需求描述】

|  |  |
| --- | --- |
| **名称** | **需求描述** |
| **需求背景** | 工艺矩阵模块用于从宏观角度展示产品工艺的整体结构，包括工序、操作、设备、参数之间的关联关系，帮助质量/工艺人员快速理解工艺全貌，识别关键控制点。当前工艺数据分散在多个页签，缺乏统一的可视化视图，导致工艺关联性难以直观掌握。通过工艺矩阵，用户可一站式查看工序与各要素的关联，并支持点击下钻。 |
| **需求说明** | 1. **统计卡片**：顶部显示当前产品下的工序总数、操作总数、关键设备总数、CMA总数、CPP总数、IPC总数、CQA总数（每个数字为可点击链接，点击后自动跳转至对应页签并筛选当前产品）。 |
| **页面布局** | 页面顶部为统计卡片（一行5-6个卡片，每个卡片显示指标名称和数值，数值带下划线或蓝色链接样式）。中间为关联表格（列：工序名称、操作、关键设备、CPP、IPC、CQA、操作）。表格支持分页（每页20条）。底部为工艺关联流程图，占据适当高度。如果无数据，显示提示信息。 |
| **异常情况处理** | 1. 若当前产品未选择或无任何工序，统计卡片数值全为0，表格显示“暂无数据”，流程图显示“请先在相应页签添加数据”。 |
| **后置处理功能** | 1. 监听其他页签的数据变更事件（通过全局状态或事件总线），自动刷新工艺矩阵内容。 |
