## 原始需求内容

### 报告模版相关功能（CPV-RT-01）

**1、报告模版模块详细功能说明**

|  |  |
| --- | --- |
| **项目** | **描述** |
| **模块功能概述** | 用于创建标准化的报告格式，作为报告编制的依据。支持模版文件上传、在线编辑、版本管理及全生命周期控制。 |
| **需求描述** | **1. 页面布局** • 左侧为产品列表（取自产品管理模块），点击产品可筛选该产品下的报告模版。 • 右侧为报告模版列表，默认每页显示50条数据。 • 列表支持按模版名称、模版编号、关联产品进行精确或模糊搜索。 • 顶部提供状态快速切换页签：全部、草稿、升版中、已生效、已失效。  **2. 列表字段** 模版名称、模版编号、版本号、所属产品、模版状态、模版文件（显示文件名或下载链接）、创建人、创建时间、更新人、更新时间、操作栏（根据状态显示不同操作按钮）。  **3. 新增/编辑报告模版** 填写以下属性： • 模版名称（必填） • 模版编号（必填，支持手动录入或系统生成，需校验唯一性） • 所属产品（必填，下拉选择） • 模版文件（必填，单文件上传，支持Word和PDF格式，上传后支持在线预览和编辑，编辑内容可实时保存） • 创建用户（系统自动记录）  **4. 报告模版详情页** 展示以下信息： • 模版名称、模版编号、版本号、所属产品、模版文件、状态 • 创建用户、创建时间 • 生效人、生效时间 • 升版人、升版时间 • 失效人、失效时间、失效原因 • 页面右侧可显示操作记录（审计追踪）  **5. 报告模版状态与生命周期** • **草稿**：新建后的初始状态，可编辑、删除、提交（生效）。 • **已生效**：草稿提交生效后的状态，可被报告编制引用。可进行“升版”操作。 • **升版中**：对已生效模版点击“升版”后产生的新版本状态，版本号自动+1。可编辑新版本内容并提交生效。原版本仍为“已生效”。 • **已失效**：当新版本生效时，旧版本自动变为“已失效”；也可手动失效已生效模版（需填写失效原因）。失效模版不可被引用。 |
| **页面交互** | • 新增/编辑页面，模版文件上传区域显示上传进度，上传成功后提供在线预览和编辑入口。 • 模版文件支持ONLYOFFICE在线编辑，编辑后自动保存或手动保存。 • 点击列表行可进入详情页，详情页以只读方式展示所有字段及历史记录。 |
| **异常逻辑** | • 上传非Word/PDF文件时，系统提示“仅支持上传Word或PDF格式文件”。或禁用掉非法格式 • 模版编号重复时，保存/提交时提示“编号已存在，请重新输入”。 • 升版时，若原模版已被报告引用，需提示“该模版已被报告引用，升版后旧版本将失效，是否继续？”。 |
| **后置流程处理** | • 模版生效后，可在报告编制模块中被选择引用。 • 模版失效后，报告编制中已引用的旧版本模版仍保留历史数据，但新建报告时不可再选择该失效模版。 |

**2、新增报告模版字段详情表**

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| **字段名称** | **输入类型** | **必填** | **默认值** | **说明** |
| 模版名称 | 文本框 | 是 | 无 | 最多200个字符。 |
| 模版编号 | 文本框 | 是 | 系统生成 | 支持手动录入，系统自动校验唯一性。建议格式：RT-YYYYMMDD-XXXX。 |
| 所属产品 | 下拉选择 | 是 | 无 | 从产品管理模块读取，单选。 |
| 模版文件 | 文件上传 | 是 | 无 | 单文件上传，支持格式：.doc, .docx, .pdf。上传后支持在线预览、编辑及实时保存。 |
| 版本号 | 只读 | 是 | V1.0 | 首次创建为V1.0，每次升版自动增加1（如V1.1、V2.0）。系统自动维护。 |
| 创建用户 | 只读 | 是 | 当前用户 | 系统自动记录。 |

**说明**：新增页面只显示需要用户填写的字段（模版名称、编号、所属产品、模版文件），其他字段由系统自动记录，在详情页展示。

**3、报告模版状态操作矩阵**

|  |  |  |
| --- | --- | --- |
| **当前状态** | **操作按钮** | **说明** |
| **草稿** | 编辑、删除、提交、复制 | 提交后状态变为“已生效”。 |
| **已生效** | 升版、失效、查看、复制 | 升版后生成新版本，新版本状态为“升版中”； 失效需填写失效原因，状态变为“已失效”。 |
| **升版中** | 编辑、提交、复制 | 编辑新版本草稿，提交后新版本生效，状态变为“已生效”，旧版本自动变为“已失效”。 |
| **已失效** | 查看、复制 | 复制生成新的草稿记录，新记录状态为“草稿”。 |

**说明**：所有操作按钮均受权限控制，仅拥有相应权限的用户可见。“提交”操作直接使模版生效，无需审批流程（如需审批可后续通过工作流配置扩展）。
