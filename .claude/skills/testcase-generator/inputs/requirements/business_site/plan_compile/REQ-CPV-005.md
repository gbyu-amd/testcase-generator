## 原始需求内容

### 方案编制相关功能（CPV-PP-01）

|  |  |
| --- | --- |
| **项目** | **描述** |
| **模块功能概述** | 用户基于方案模版，结合具体任务和监控项目，编制具体的执行方案，并管理其生命周期。 |
| **需求描述** | **1. 页面布局** • 左侧为产品列表（取自产品管理模块），点击产品可筛选该产品下的方案。 • 右侧为方案列表，默认每页显示50条数据。 • 列表支持按方案名称、方案编号、所属产品进行精确或模糊搜索。 • 顶部提供状态快速切换页签：全部、草稿、审批中、升版中、已生效、已失效。  **2. 列表字段** 方案名称、方案编号、所属产品、方案状态、版本号、创建人、创建时间、更新人、更新时间、操作栏（根据状态显示不同按钮）。  **3. 新增/编辑方案** 新增方案页面分为两个Tab页： • **Tab1-详情**：填写方案的基本属性。主要字段见下方《新增方案字段详情表》。 • **Tab2-监控项目**：管理方案中包含的监控项目。支持单个新增、批量导入、批量删除。字段包括：监控项目名称、监控类型、质量标准、标准类型、统计分析工具、备注。  **4. 方案详情页** 同样分为两个Tab页： • **详情**：展示方案属性（方案名称、编号、所属产品、所属CPV任务、方案模版、方案文档、版本号、状态、升版人/时间、生效人/时间、失效人/时间、失效原因），以及审批明细表。页面右侧显示该方案的审计追踪记录。 • **监控项目**：展示该方案下所有监控项目列表，支持查看各项目的分析状态。  **5. 方案状态与生命周期** • **草稿**：新建或复制后的初始状态，可编辑、删除、提交审批。关联的监控项目状态为“草稿”。 • **审批中**：提交审批后的状态，不可编辑。审批通过后进入“已生效”，被退回后进入“已退回”。 • **已撤回：**审批中时，撤回的方案编制 |
| **页面交互** | • 新增方案时，选择“所属CPV任务”和“方案模版”均通过弹窗实现，弹窗内支持搜索和快捷新增（可直接新增任务或模版）。 |
| **异常逻辑** | • 提交审批时，若监控项目列表为空，系统应拦截并提示“请至少添加一个监控项目”。 • 若所选CPV任务已被其他生效方案关联，需提示“该任务已关联生效方案，请确认”。 • 升版时，若原方案有正在进行的任务，需提示“存在进行中的任务，升版可能影响任务执行”。 |
| **后置流程处理** | • 方案生效后，自动生成监控项目实例，并推动关联CPV任务的状态更新（如从“待执行”变为“进行中”）。 • 方案失效时，需检查关联任务状态，若任务未完成，给出提示并允许强制失效或阻止操作。 |

**以下是导入文件相关的流程整理：**

|  |  |  |  |
| --- | --- | --- | --- |
| 1 | 上传文件 | 用户上传Excel文件到系统。 | 文件上传至服务器待处理。 |
| 2 | 文件格式校验 | 系统检查文件格式（如扩展名、模板结构等）是否符合要求。 | - 若格式错误，直接提示“文件内容不匹配”，流程终止。 - 若格式正确，继续下一步。 |
| 3 | 逐行数据校验 | 对每一行数据进行业务规则校验（如必填项、数据格式等），记录错误行号和错误原因。多个错误原因用分号隔开。 | 生成错误信息列表（包含行号及对应错误原因）。 |
| 4 | 判断是否有错误 | 检查校验过程中是否发现任何错误。 | -若无错误，转步骤5。 - 若有错误，转步骤6。 |
| 5 | 监控项目导入 | 将监控项目导入系统，导入完成后提示“XXXX个监控项目导入成功”。 | 显示成功提示，导入完成。 |
| 6 | 生成错误报告 | 整份文件存在一个以上错误时，不允许导入，并生成TXT格式错误报告。 报错内容格式： | 提供错误报告TXT文件下载，用户可获取详细失败记录。 |

**方案编制状态操作矩阵**

|  |  |  |
| --- | --- | --- |
| **当前状态** | **操作按钮** | **说明** |
| **草稿** | 编辑、删除、提交、复制 | 提交后进入“审批中”。 |
| **审批中** | 申请人：撤回 审批人：同意、退回 复制 | 无编辑入口。审批通过后变“已生效”；被退回后变“已退回”。 |
| **已撤回** | 编辑、提交、复制 | 审批中时，申请人撤回方案 |
| **已退回** | 编辑、提交、复制 | 编辑后可再次提交，重新进入“审批中”。 |
| **已生效** | 升版、失效、复制 | 升版生成新版本，状态为“升版中”；失效需填写原因，状态变为“已失效”。 |
| **升版中** | 编辑、提交、复制 | 编辑新版本草稿并提交审批，审批通过后新版本生效，状态变为“已生效”，旧版本变为“已失效”。 |
| **已失效** | 复制 | 复制生成新的草稿记录，新记录状态为“草稿”。 |
