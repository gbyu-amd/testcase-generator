## 原始需求内容

### 方案编制和报告编制关联模版相关优化

【需求背景】

当前方案编制和报告编制在选择关联模版时，未对模版的产品归属进行限制，可能导致选错模版（如选择了其他产品的模版），造成内容不匹配或数据混乱。需增加模版关联规则：仅允许选择与当前方案/报告**同一产品下的模版**或**未关联任何产品的模版**。

【需求描述】

|  |  |
| --- | --- |
| **项目** | **描述** |
| **需求描述** | **1. 适用范围** • 方案编制：选择关联的**方案模版**时 • 报告编制：选择关联的**报告模版**时  **2. 模版关联规则** • 模版选择弹窗中，仅显示以下两类模版： - **同产品模版**：模版所属产品与当前方案/报告所属产品相同 - **未关联产品模版**：模版未绑定任何产品（即产品字段为空）  **3. 规则示例** • 当前方案所属产品为“阿莫西林胶囊” - 可选择的方案模版包括： ✓ 产品为“阿莫西林胶囊”的模版 ✓ 产品为空的模版（通用模版） - 不可选择的方案模版： ✗ 产品为“头孢克肟胶囊”的模版 ✗ 产品为“阿奇霉素片”的模版  **4. 交互说明** • 模版选择弹窗中，列表仅展示符合规则的模版，不符合规则的模版不可见。 • 若当前产品下无任何可用的方案模版，显示为空； |
| **页面交互** | • 用户先选择/确认方案或报告的“所属产品”后，再点击“选择模版”按钮。 • 若未先选择所属产品，点击选择模版时提示“请先选择所属产品”。 • 模版选择弹窗中，列表显示模版名称、编号、所属产品、状态等信息，方便用户识别。 |
| **异常逻辑** | • 若用户先选择了模版，后修改了所属产品，需清空已选模版。 • 若已生效的方案/报告所引用的模版被修改了产品归属，不影响已生效数据，但编辑时需重新校验规则。 |
| **后置流程处理** | • 方案/报告保存时，校验所选模版是否符合关联规则，不符合则阻止保存并提示。 • 模版的产品归属变更时，不影响已引用该模版的已生效方案/报告，仅对新创建或编辑中的方案/报告生效。 |

**关联规则校验逻辑**

|  |  |  |
| --- | --- | --- |
| **校验时机** | **校验逻辑** | **不通过处理** |
| 选择模版时 | 模版弹窗仅显示符合条件的模版 | 无（根本看不到不符合的模版） |
| 保存方案/报告时 | 再次校验所选模版是否符合规则 | 提示“所选模版与当前产品不匹配，请重新选择” |
| 编辑方案/报告时（产品已变更） | 检测到产品与模版不匹配 | 清空模版选择，提示“产品已变更，请重新选择模版” |

**说明**：未关联产品的模版可在任何产品的方案/报告中引用，提高了模版的复用性。
