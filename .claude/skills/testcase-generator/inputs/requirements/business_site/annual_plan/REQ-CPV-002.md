## 原始需求内容

### 计划全生命周期管理（CPV-AP-01）

**1、整体需求描述：**

|  |  |
| --- | --- |
| **项目** | **描述** |
| **模块功能概述** | 负责定义CPV活动的年度或周期性计划，是周期性任务的源头。支持计划的创建、编辑、审批、生效、修订和失效，生效时自动生成周期性任务。 |
| **需求描述** | **1. 页面布局** • 左侧为产品列表（取自产品管理模块），点击产品可筛选该产品下的计划。 • 右侧为年度计划列表，默认每页显示50条数据。 • 列表支持按计划名称、计划编号、所属产品、周期类型进行精确或模糊搜索。 • 顶部提供状态快速切换页签：全部、草稿、审批中、已撤回、已退回、已生效、修订中、已失效。  **2. 列表字段** 产品名称、所属产品、版本号、状态、首次日期、周期类型、周期值、提醒天数、提醒对象、操作栏（根据状态显示不同操作按钮）。可设置列表字段信息的展示与否。  **3. 新增/编辑计划** 填写属性信息，详见下方《新增计划字段详情表》。  **4. 预览计划详情页** 显示以下信息：所属产品、计划名称、计划类型、首次日期、提醒天数、版本号、描述、周期类型和值、提醒对象、周期单位、状态、部门、生效人、生效时间、失效人、失效时间、失效原因。同时包含审批明细表（审批历史）。  **5. 计划生命周期（见下方计划状态操作矩阵）** • **草稿**：新建计划后的初始状态，可编辑、删除、提交（发起审批）。 • **审批中**：提交后进入审批流程，不可编辑。审批通过后变为“已生效”，被退回后变为“已退回”。撤回之后是“已撤回”。 •**已撤回：**计划审批进行中时，由申请人撤回。 |
| **页面交互** | • 新增/编辑计划使用抽屉式页面或独立页面。 |
| **异常逻辑** | 1）提交审批时，若必填项未填，鼠标失焦给出对应的提示。 |
| **后置流程处理** | 1）计划生效：自动触发任务生成引擎，根据周期规则（周期规则可配置）创建周期性任务，并发送通知给提醒对象。 |

**新增计划字段详情表**

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| **字段名称** | **输入类型** | **必填** | **默认值** | **说明** |
| 所属产品 | 下拉选择 | 是 | 无 | 从产品管理模块读取，单选。 |
| 计划名称 | 文本框 | 是 | 无 | 最多200个字符。 |
| 计划类型 | 单选按钮 | 是 | 时间 | 选项：时间、批数。（批数后续再做） |
| 首次日期 | 日期选择器 | 是 | 无 | 第一个任务的计划日期。修订状态下提交时，必须大于今日。 |
| 周期单位 | 下拉选择 | 是 | 月 | 选项：月、季度、年。 |
| 周期 | 数字输入框 | 是 | 1 | 与周期单位组合，如“2个月”。 |
| 提醒天数 | 数字输入框 | 是 | 7 | 任务开始前多少天发送提醒。 |
| 提醒对象 | 用户选择器 | 是 | 无 | 可选择多个用户、角色或部门。 |
| 描述 | 文本域 | 否 | 无 | 计划描述信息，最多300个字符。 |
| 部门 | 只读 | - | 无 | 回显创建人所在的部门，从组织架构读取。 |
| 版本号 | 只读 | - | V1.0 | 系统自动维护，每次修订自动增加0.1（如V1.1、V2.0）。 |
| 状态 | 只读 | - | 草稿 | 系统自动更新。 |

**计划状态操作矩阵（无相关权限的按钮不可见）**

|  |  |  |
| --- | --- | --- |
| **当前状态** | **操作按钮** | **说明** |
| 草稿 | 编辑、删除、提交、复制 | 提交后计划状态进入“待审批”。 |
| 审批中 | 创建人：撤回、复制 审批人：同意、退回、转审、复制 | 审批中状态下，用户可操作撤回。 不可编辑。审批通过后进入“已生效”；被退回后进入“已退回”。撤回变成“已撤回” |
| 已撤回 | 编辑、删除、提交、复制 | 已撤回状态下，用户可重新提交 |
| 已退回 | 编辑、提交、复制 | 编辑后可再次提交，重新进入“待审批”。 |
| 已生效 | **变更**、失效、复制 | 1、修订生成新版本，新版本状态为“**变更中**”版本号自动加1； 2、失效需填写原因，状态变为“已失效”，失效原因最大支持200个字符； 3、复制生成新的草稿。 |
| **变更中** | 编辑、提交、复制 | 编辑新版本内容并提交审批，提交后进入“待审批”。 |
| 已失效 | 复制 | 复制生成新的草稿记录，新记录状态为“草稿”。 |

说明：

**计划生效/失效逻辑详解**

|  |  |  |
| --- | --- | --- |
| **触发动作** | **执行逻辑** | **说明** |
| 计划生效（草稿→已生效 / 修订中→已生效） | 1. 删除现有任务：查找本计划下所有“计划日期 > 今天”的任务，进行物理或逻辑删除。 | 确保计划生效后任务列表准确无误，避免重复或遗漏。 |
| 计划失效（已生效→已失效） | 删除本计划下所有“计划日期 > 今天”的任务，保留历史任务数据。 | 计划作废后，未来任务不再执行，但历史数据保留作为审计证据。 |
