## 原始需求内容

### 相关性分析和监控项目支持在方案中直接分析

【需求背景】

当前方案编制中的相关性分析和监控项目，需要用户跳转到独立的相关性分析模块或监控项目模块才能执行数据分析、确认、创建监控项目等操作，流程割裂，工作效率低。用户期望在方案编制预览详情页面直接完成分析、确认、创建监控项目等操作，无需切换模块，实现更流畅的业务闭环。同时，用户需要支持批量操作（批量分析、批量确认、批量删除），以提升多记录处理效率。**原有“相关性分析模块”和“监控项目模块”的功能保持不变**，本次仅增强方案编制内部的操作能力。

【需求描述】

|  |  |
| --- | --- |
| **名称** | **需求描述** |
| **方案编制预览详情页改造** | 在方案编制预览详情页面的“相关性分析”页签和“监控项目”页签中，列表增加“复选框”列和“操作栏”列，支持用户单选或多选记录，并提供单个操作按钮和批量操作按钮。 |
| **单个操作栏按钮展示规则** | 根据当前数据分析记录状态显示不同按钮（与原有模块逻辑一致）： |
| **批量操作按钮及规则** | 列表顶部增加三个批量操作按钮：“批量分析”、“批量确认”、“批量删除”。按钮根据当前选中记录的状态动态启用/禁用，规则如下： • **批量分析**：仅对状态为“未分析”、“已分析”、“变更中”的记录生效。选中记录中包含其他状态时，按钮置灰并提示“仅支持对未分析、已分析、变更中状态进行批量分析”。 • **批量确认**：仅对状态为“已分析”的记录生效。选中记录中包含其他状态时，按钮置灰并提示“仅支持对已分析状态进行批量确认”。 • **批量删除**：仅对状态为“未分析”的记录生效。选中记录中包含其他状态时，按钮置灰并提示“仅支持对未分析状态进行批量删除”。 批量操作时需二次确认（如：“确认批量分析X条记录吗？”）。批量操作结果（成功/失败数量）通过弹窗反馈。 |
| **操作权限** | 方案编制中相关性分析和监控项目页签操作栏的按钮权限，**与相关性分析模块和监控项目模块中对应操作的权限保持一致**（复用已有权限点）。批量操作按钮的权限与单个操作按钮权限相同（例如，批量分析需要 correlation:analysis 权限）。 |
| **功能联动** | • 单个/批量“数据分析”：跳转到数据分析工作台（单个分析按原逻辑，批量分析建议转为异步任务，完成后结果回写）。 • 单个“创建监控项目”：弹出弹窗，生成监控项目并关联当前方案。 • 单个/批量“确认”：更新状态为“已确认”。 • 单个“关闭”：弹出关闭原因输入框。 • 单个“变更”：状态变为“修订中”。 • 批量删除：删除选中记录，需二次确认。 |
| **菜单名称与权限更新** | 1. 原“相关性分析”二级菜单更名为 **“相关性分析汇总”**。 2. 角色管理模块中对应的菜单名称同步更新为“相关性分析汇总”。 3. 审计追踪模块中涉及该菜单的操作记录，菜单名称也相应更新为“相关性分析汇总”（历史数据不做强制修改，新记录使用新名称）。 |
| **审计追踪** | 由于在方案编制中直接执行的分析、确认、关闭等操作，其产生的审计追踪**暂不考虑**（即不额外记录操作日志，仍沿用原有相关性分析模块和监控项目模块的审计逻辑，但本次不要求新增方案编制内的审计记录）。批量操作同理。 |
| **原有模块功能不变** | **本次所有修改仅影响方案编制内部的“相关性分析”和“监控项目”页签的交互体验。原有的“相关性分析汇总”菜单（原“相关性分析”模块）以及“监控项目”菜单的现有功能、操作逻辑、权限控制、审计追踪等均保持不变，用户仍可通过原模块进行所有操作。** |
| **页面布局** | 方案编制预览详情页面保持现有布局：左侧产品树（或方案概要），右侧为Tab页（详情、相关性分析、监控项目）。在“相关性分析”和“监控项目”页签的列表中，新增“复选框”列（第一列）和“操作”列（最后一列）。列表顶部增加“批量分析”、“批量确认”、“批量删除”按钮。列表其他字段不变。 |
| **交互说明** | • 鼠标悬停操作按钮时显示提示文字。 • 单个删除时弹出确认框；“批量删除”时弹出确认框，显示选中记录数量。 • 单个关闭时弹出输入框，要求填写关闭原因，必填。 • 批量分析时，若数据量大，显示进度提示，转为异步任务。 • 复选框支持全选/反选。 • 批量操作按钮根据选中记录状态动态变化（可用/置灰）。 |
| **异常逻辑** | 1. 若用户无对应操作权限，按钮置灰或隐藏。 2. 删除时若记录已被引用（如已生成监控项目），提示“无法删除，请先解除关联”。 3. 关闭时未填写原因，提示“请填写关闭原因”。 4. 数据分析时若数据源缺失或数据不足，提示相应错误信息。 5. 创建监控项目时若名称重复，提示“监控项目已存在”。 6. 批量操作时若选中记录为空，提示“请至少选择一条记录”。 7. 批量操作中部分失败时，提示“成功X条，失败Y条”，并提供失败详情。 8. 原模块的异常逻辑不受本次改动影响。 |
| **后置处理功能** | 1. 操作成功后，自动刷新当前页签列表（状态、按钮变化）。 2. 若相关性分析汇总菜单名称变更，需同步更新导航栏、角色权限树、审计追踪模块的显示名称。 3. 用于将已有相关性分析模块和监控项目模块的历史数据与方案编制的关联保持正常（不因菜单改名影响业务）。 4. 原模块的后置处理逻辑保持不变。 |
