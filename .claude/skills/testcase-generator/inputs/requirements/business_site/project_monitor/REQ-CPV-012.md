## 原始需求内容

#### **监控项目增加编辑功能**

【需求背景】

在CPV系统的监控项目模块中，用户有时需要在监控项目尚未分析时修正其配置信息（如质量标准、分析方法等），以避免因初始录入错误导致后续分析偏差。当前系统不支持编辑监控项目，且编辑操作缺乏审计记录和权限控制。因此需增强监控项目模块，支持编辑未分析状态的监控项目，同步更新关联方案中的对应信息，并增加编辑操作的审计追踪及权限管理。

【需求描述】

|  |  |
| --- | --- |
| **项目** | **内容** |
| **需求描述** | **一、监控项目编辑功能** • **适用范围**：仅当监控项目状态为“未分析”时，操作栏显示“编辑”按钮。若监控项目状态已变为“已分析”或“已确认”或“已关闭”或变更中，编辑按钮不显示。 • **编辑弹窗**：点击“编辑”后，弹出编辑弹窗，字段与新增监控项目时一致，包括：监控项目名称（下拉选择，从产品工艺参数库中选取）、监控项目类型（根据所选参数自动带出，只读）、分析方法（下拉，可选单值控制图、Xbar-R图等）、LSL、TARGET、USL（带出，不可编辑）、备注（文本域）。 • **保存逻辑**：用户修改后点击确定，更新监控项目记录。 • **同步方案编制**：自动更新方案编制中对应监控项目的配置信息（项目名称、类型、质量标准等），并记录同步日志。  **二、审计追踪** • **操作类型**：在审计追踪模块中新增操作类型“编辑”。 • **审计内容**：记录操作人、操作时间、操作模块（监控项目）、操作类型（编辑）。 • **详情对比**：： 1、基本信息：操作人、操作时间、操作模块、操作类型； 2、内容详情：监控项目名称、监控项目类型、分析方法、LSL、TARGET、USL、备注。内容详情中需要显示新旧值对比。内容如下： |
| **异常情况说明** | 1. 若编辑时监控项目名称与同产品下已有参数名称和分析方法重复，提示“名称已存在，请重新选择”。或报错文案同之前。 |
| **后置处理功能** | 1. 编辑成功后，刷新监控项目列表，显示最新信息。 2. 生成审计追踪记录，包含编辑前后的完整对比。 |
