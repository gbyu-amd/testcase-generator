### 监控项目（CPV-MI-01）

**监控项目模块详细功能说明**

|  |  |
| --- | --- |
| **项目** | **描述** |
| **模块功能概述** | 方案生效后自动生成监控项目列表，作为数据分析的核心对象。支持按状态筛选、批量数据分析、批量导出分析结果，管理监控项目的全生命周期。分析时跳转至插件化数据分析平台，但操作权限受限。 |
| **需求描述** | **1. 页面布局** • 左侧为产品列表（取自产品管理模块），点击产品可筛选该产品下的监控项目。 • 右侧为监控项目列表，默认每页显示50条数据。 • 顶部提供状态快速切换页签：全部、未分析、已分析、已确认、已关闭。 • 搜索框支持按监控项目名称、监控项目编号进行精确或模糊搜索。 • 产品列表可按方案的维度进行分类（如按方案分组展示）。 • 列表上方提供“批量分析”和“批量导出”操作按钮。  **2. 列表属性信息** 复选框、监控项目编号、监控项目名称、分析状态、监控类型、分析结果名称（可预览分析结果）、质量标准、标准类型、方案名称（可快速跳转至方案详情页面）、操作栏（根据状态显示不同操作按钮）。  **3. 相关按钮操作** • **数据分析（单个/批量）**： - 点击列表中某监控项目的“数据分析”按钮，或勾选多个监控项目后点击“批量分析”，跳转至**插件化数据分析平台**。 - **跳转规则**： \* 自动定位在“数据管理”页签，“数据处理”和“数据分析”页签置灰不可用。 \* 数据管理下的“本地数据”页签：隐藏“导入文件”按钮，列表无“删除”按钮。 \* 数据管理下的“系统集成”页签：隐藏“批量更改分类”按钮，列表无“编辑”按钮。 \* “数据处理”页签：列表无“删除”和“编辑”按钮。 \* 以上数据管理和数据处理模块的分类，无导入分类和维护分类的入口。 |
| **页面交互** | • 列表可按方案名称分组折叠，便于查看同一方案下的所有监控项目。 • 方案名称支持超链接，点击可快速跳转至对应方案的详情页面。 • 批量操作时，复选框支持全选/取消全选。 • 从监控项目跳转至插件化数据分析平台时，用户从我的数据和数据处理页面跳转至数据分析详情页面时，用户将看不到“数据分析”的列表页（直接进入分析详情）。 |
| **异常逻辑** | • 批量分析时，若选中的监控项目中包含“已确认”或“已关闭”等不可分析的状态，系统应提示“所选项目中包含不可分析的状态，请重新选择”。 • 关闭操作时，若未填写关闭原因，系统拦截并提示“关闭原因不能为空”。 • 批量导出时，若选中项目无分析结果，提示“所选项目中包含无分析结果的监控项目，是否继续导出？”（仅导出有结果的部分）。 • 从监控项目跳转至插件化数据分析平台时，若未获取到监控项目预设配置（如分析图类型），提示“监控项目配置不完整，请返回检查”。 |

#### 监控项目相关优化

【需求背景】

监控项目是CPV系统的核心执行单元，负责对工艺参数或质量属性进行持续监控、数据分析和结果确认。现有监控项目主要由方案直接生成，但部分场景下用户需要独立于方案手动创建监控项目（如补充监控或临时分析）。因此需增强监控项目模块的自维护能力，支持手动新增、批量操作以及独立关联方案，同时保持与原有分析状态的完整生命周期。

【需求描述】

|  |  |
| --- | --- |
| **项目** | **内容** |
| **需求描述** | **1. 页面布局与筛选** • 左侧产品列表：选择产品后，右侧联动显示该产品下的方案下拉框（可筛选全部或指定方案）。（顶部全部产品不可点击） • 顶部筛选：支持按分析状态（全部、未分析、已分析、已确认、已关闭、修订中）筛选；支持按监控项目名称、编号搜索。 • 列表单页默认50条数据，支持翻页。  **2. 列表字段** 全选、监控项目编号、监控项目名称、分析状态、监控类型、数据源属性信息、方案名称（超链接跳转方案详情）、统计分析工具、分析结果名称（点击可查看分析结果）、质量标准、标准类型、任务名称（超链接跳转任务详情）、操作栏（按钮随状态变化，见矩阵）。  **3. 页面操作按钮** • **批量删除**：勾选多条记录后删除（仅“未分析”状态可删）。 • **批量分析**：勾选多条记录后执行分析（仅“未分析/修订中/已分析”可分析）。 • **新增**：打开新增监控项目弹窗，手动创建监控项目。  **4. 新增监控项目** 点击“新增”打开弹窗，流程如下： - 选择产品 & 选择该产品下的CPV方案。若在方案内通过“添加监控项目”入口进入（如相关分析生成时），则方案名称自动回填。 - 支持动态添加多行监控项目：每行包含监控项目名称（下拉选择该产品关联的工艺参数库）、监控项目类型（根据所选参数自动带出，只读）、分析方法（下拉，可选控制图/能力分析等，不包含相关性分析/Pearson）、质量标准（自动带出，只读）、标准类型（自动带出，只读）、备注（可编辑）。 - 支持删除行、批量删除行、编辑。 - 提交后，系统为每行创建独立的监控项目实例，状态为“未分析”，自动关联所选方案。  **5. 其他功能** • **数据分析**：与现有监控项目分析逻辑一致，支持单个/批量分析。 • **确认/关闭/变更**：沿用现有监控项目状态流转规则。 其他功能同之前 |
| **异常情况说明** | 1. 新增监控项目时，若未选择方案 → 提示“请先选择方案”。 |
| **后置处理功能** | 1. 新增监控项目成功后，自动刷新列表，无需记录审计日志。 2. 监控项目状态变更（如分析、确认、关闭、变更）需同步更新方案编制中。 4. 批量操作（分析/删除）后，完成后发送站内通知。（参考之前） |

#### **监控项目增加编辑功能**

【需求背景】

在CPV系统的监控项目模块中，用户有时需要在监控项目尚未分析时修正其配置信息（如质量标准、分析方法等），以避免因初始录入错误导致后续分析偏差。当前系统不支持编辑监控项目，且编辑操作缺乏审计记录和权限控制。因此需增强监控项目模块，支持编辑未分析状态的监控项目，同步更新关联方案中的对应信息，并增加编辑操作的审计追踪及权限管理。

【需求描述】

|  |  |
| --- | --- |
| **项目** | **内容** |
| **需求描述** | **一、监控项目编辑功能** • **适用范围**：仅当监控项目状态为“未分析”时，操作栏显示“编辑”按钮。若监控项目状态已变为“已分析”或“已确认”或“已关闭”或变更中，编辑按钮不显示。 • **编辑弹窗**：点击“编辑”后，弹出编辑弹窗，字段与新增监控项目时一致，包括：监控项目名称（下拉选择，从产品工艺参数库中选取）、监控项目类型（根据所选参数自动带出，只读）、分析方法（下拉，可选单值控制图、Xbar-R图等）、LSL、TARGET、USL（带出，不可编辑）、备注（文本域）。 • **保存逻辑**：用户修改后点击确定，更新监控项目记录。 • **同步方案编制**：自动更新方案编制中对应监控项目的配置信息（项目名称、类型、质量标准等），并记录同步日志。  **二、审计追踪** • **操作类型**：在审计追踪模块中新增操作类型“编辑”。 • **审计内容**：记录操作人、操作时间、操作模块（监控项目）、操作类型（编辑）。 • **详情对比**：： 1、基本信息：操作人、操作时间、操作模块、操作类型； 2、内容详情：监控项目名称、监控项目类型、分析方法、LSL、TARGET、USL、备注。内容详情中需要显示新旧值对比。内容如下： |
| **异常情况说明** | 1. 若编辑时监控项目名称与同产品下已有参数名称和分析方法重复，提示“名称已存在，请重新选择”。或报错文案同之前。 |
| **后置处理功能** | 1. 编辑成功后，刷新监控项目列表，显示最新信息。 2. 生成审计追踪记录，包含编辑前后的完整对比。 |
