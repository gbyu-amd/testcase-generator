## 原始需求内容

#### 工艺能力汇总支持导出PDF

【需求背景】

在CPV系统的工艺能力汇总模块中，质量管理人员需要定期向管理层或跨部门团队汇报当前工艺的整体能力状况，包括Cpk/Ppk指标、超规格限比例、异常批次等关键信息。目前系统仅支持在线查看和交互式分析，无法直接生成可供会议演示、审计存档或书面决策的正式报告。因此，需增加导出功能，支持将当前方案下的监控项目能力数据一键导出为PDF文件，便于质量管理人员在汇报、审计或内部决策时使用。

【需求描述】

|  |  |
| --- | --- |
| **项目** | **内容** |
| **需求背景** | 在CPV系统的工艺能力汇总模块中，用户需要将当前方案下的监控项目能力数据（Cpk、Ppk、超规格限百分比、异常批次等）导出为PDF文件，便于管理者内部决策时使用。当前系统仅支持在线查看，缺少导出能力。同时，导出操作需受权限控制，并记录审计追踪，满足合规要求。 |
| **需求描述** | **1. 导出入口**：在工艺能力汇总页面的右上角增加“导出”按钮。按钮受权限控制，仅拥有权限的用户可见。点击后直接触发导出（同步实时导出，不转为异步任务）。 |
| **页面布局** | 工艺能力汇总页面顶部右侧（筛选区旁边）增加“导出”按钮。左侧产品树、右侧方案联动逻辑不变。列表支持监控项目名称搜索，分页每页50条。页面提供“连接视图”和“矩阵视图”Tab切换（不影响导出功能，导出时以当前视图的数据源为准，推荐导出列表视图的完整数据）。 |
| **交互说明** | - 点击导出按钮后，前端收集当前方案的所有监控项目数据（包括分页外的全部数据），发送请求至后端生成PDF。 - 后端生成PDF过程中，前端显示加载遮罩及“导出中，请稍候...”提示，不允许再次点击导出。 - 生成成功后，浏览器自动下载PDF文件，并清除加载状态。 - 若生成失败，弹出错误提示“导出失败，请重试”。 - 导出操作不依赖于当前视图（连接视图/矩阵视图），统一导出列表视图格式（因为矩阵视图可能不适合表格导出）。 |
| **异常逻辑** | 1. 导出时若数据量超过500条，提示“数据量超过500条，无法一次性导出，请缩小范围或联系管理员”。 2. 导出过程中网络中断或服务器异常，提示“导出失败，请检查网络后重试”。 3. 若用户无导出权限，导出按钮不显示。 4. 若当前方案下无监控项目，点击导出时提示“暂无数据可导出”。 |
| **后置处理功能** | 1. 导出成功后，在审计追踪模块记录一条操作日志（详见下方审计追踪要详情设计）。 2. 导出操作需记录到系统日志，便于排查问题。 3. 同步导出过程中，应避免长时间占用数据库连接，建议使用单独的PDF生成服务或优化查询性能。 |

**审计追踪详情设计**

|  |  |
| --- | --- |
| **项目** | **内容** |
| **审计操作类型** | 在审计追踪模块中，为“工艺能力汇总”模块增加操作类型“导出”。 |
| **审计列表字段** | 序号、操作人、操作时间、所属模块（工艺能力汇总）、操作类型（导出）、受影响对象（导出的文件名） |
| **审计详情页面** | 1、审计追踪模块，工艺能力汇总模块，操作类型的下拉选项中增加批量导出的操作类型； 2、审计追踪列表，受影响对象为：{方案名称}\_{导出监控项目数量}\_{时间戳}.pdf **基本信息**：操作人、操作时间、操作模块（工艺能力汇总）、操作类型（导出） |
