## 原始需求内容

### 组织机构管理（CPV-DM-01）

**1、组织机构管理模块详细功能说明**

|  |  |
| --- | --- |
| **项目** | **描述** |
| **模块功能概述** | 维护企业的组织架构（部门/机构），支持部门树形结构、状态管理、人员关联，便于权限分配、流程审批及数据隔离。 |
| **需求描述** | **1. 页面布局** • 顶部提供状态快速切换页签：全部、正常、已停用。 • 页面上方为搜索框，支持按**部门名称**和**部门状态**进行模糊或精确搜索。 • 列表上方提供“新增”按钮。 • 列表默认每页显示50条数据，采用树形表格展示部门层级，支持展开/折叠子部门。  **2. 列表属性信息** 部门名称（可点击展开子部门）、排序号、状态、负责人、联系电话、邮箱、创建人、创建时间、更新人、更新时间、操作栏（新增、编辑、删除）。  **3. 相关按钮操作** • **新增**：点击“新增”按钮（列表顶部或操作栏中的“新增子部门”），弹出新增部门表单，需填写以下字段： - 部门名称（必填） - 显示顺序（必填，数字类型，用于同级部门排序） - 负责人（可选） - 联系电话（可选） - 邮箱（可选） - 部门状态（默认“正常”，可选范围：正常、停用） - 上级部门（树形下拉选择，若从某部门操作栏点击“新增子部门”进入，则上级部门自动选中当前部门） • **编辑**：点击“编辑”按钮，可修改部门信息（名称、排序、负责人、联系电话、邮箱、状态、上级部门等），部门编码（若有）不可修改。 • **删除**：点击“删除”按钮，弹出确认提示框：“确定删除该部门吗？”确认后删除。若部门下存在子部门或已关联用户，需提示“该部门包含子部门或关联用户，无法删除”。 |
| **页面交互** | • 列表默认展示一级部门，点击部门名称前的箭头可展开查看子部门。 • 搜索时对部门名称进行模糊匹配，并保持树形结构的展开状态。 • 新增/编辑表单使用弹窗或抽屉式页面，上级部门选择器以树形下拉展示所有正常状态的部门。 • 部门删除需二次确认，并给出明确提示。 |
| **异常逻辑** | • 新增/编辑时，部门名称在同一父级下不可重复，若有重复提示“同级部门名称不能重复”。 • 删除部门时，若部门存在子部门或关联用户，阻止删除并提示。 • 停用部门后，该部门在上级部门选择器中不可选（避免关联到停用部门）。 |

**2、新增/编辑部门字段详情表**

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| **字段名称** | **输入类型** | **必填** | **默认值** | **说明** |
| 部门名称 | 文本框 | 是 | 无 | 最多50个字符。 |
| 上级部门 | 树形下拉 | 否 | 无 | 选择上级部门，若为顶级部门则选择“根部门”。 |
| 显示顺序 | 数字输入框 | 是 | 0 | 同级部门排序，数字越小越靠前。 |
| 负责人 | 用户选择器 | 否 | 无 | 可从系统用户中选择。 |
| 联系电话 | 文本框 | 否 | 无 | 格式校验（手机或固话）。 |
| 邮箱 | 文本框 | 否 | 无 | 格式校验。 |
| 部门状态 | 下拉选择 | 是 | 正常 | 选项：正常、停用。停用部门在流程选人、数据权限中不可见。 |

**3、状态操作矩阵**

|  |  |  |
| --- | --- | --- |
| **当前状态** | **操作按钮** | **说明** |
| **正常** | 新增子部门、编辑、删除 | 可进行所有操作。 |
| **停用** | 编辑、删除 | 停用后不可新增子部门，且部门在上级部门选择器中不可选。 |

**说明**：所有操作按钮受权限控制，仅拥有相应权限的用户可见。删除操作需二次确认，且受数据完整性校验约束。
