## 原始需求内容

### 用户登录日志管理

**1、用户登录日志模块详细功能说明**

|  |  |
| --- | --- |
| **项目** | **描述** |
| **模块功能概述** | 记录系统用户的登录行为，包括登录时间、登出时间、登录状态、客户端信息等，用于安全审计、异常行为追溯及用户活跃度分析。 |
| **需求描述** | **1. 页面布局** • 顶部提供状态快速切换页签：全部、正常、失败。 |
| **页面交互** | • 状态页签与搜索框联动，筛选结果实时刷新。 • 登录时间、登出时间支持日期范围选择，如“2025-01-01 至 2025-01-31”。 • 列表支持按时间排序（默认按登录时间倒序）。 |
| **异常逻辑** | • 搜索条件无结果时，显示“暂无数据”提示。 • 数据库日志量过大时，可配置自动归档或清理策略（如保留最近6个月数据）。 |

**用户登录日志列表字段说明**

|  |  |
| --- | --- |
| **字段名称** | **说明** |
| 用户名 | 登录账号。 |
| 姓名 | 用户真实姓名。 |
| 部门 | 所属部门。 |
| 岗位 | 所属岗位。 |
| 响应信息 | 登录验证结果：成功 / 失败（可细化失败原因，如密码错误、账号锁定、验证码错误等）。 |
| 客户端IP | 用户登录时的IP地址，支持IPv4和IPv6。 |
| 登录时间 | 用户成功登录或尝试登录的时间。 |
| 登出时间 | 用户登出时间（主动登出、会话超时等），若未登出可为空。 |

**说明**：日志数据仅供安全审计使用，普通用户仅可查看自己的登录日志，管理员可查看所有用户日志，具体权限通过角色管理配置。
