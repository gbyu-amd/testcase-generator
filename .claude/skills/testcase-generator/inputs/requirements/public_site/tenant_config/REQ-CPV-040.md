## 原始需求内容

### 公共管理系统与各业务站点的关系

|  |  |
| --- | --- |
| **项目** | **描述** |
| **系统定位** | • **公共管理系统**：负责全局基础数据管理（站点、用户、权限），不处理具体业务（如CPV计划、任务、方案、报告等）。 |
| **数据同步** | • **公共管理 → 业务站点**：用户信息、站点信息、角色权限分配等基础数据实时或定时同步至各业务站点。 |
| **页面跳转与站点切换** | • 公共管理系统和各业务站点的页面顶部导航栏均显示当前用户可访问的站点列表（包含“公共管理”及所有有权限的业务站点）。 |
| **统一认证** | • 所有站点共用同一套用户认证体系（基于JWT或Session），Token在站点间通用。 |
| **业务功能概述** | • 业务站点的功能与之前需求描述一致：年度计划设置、任务管理、方案模版与编制、监控项目、报告模版与编制、数据分析、工作流配置、审计追踪等。 |
| **站点管理员权限** | • 站点管理员在业务站点内拥有最高权限，可管理本站点的业务配置、本地用户（如有）、部门、角色等。 |
| **数据分析插件化平台** | • 各个站点共用一套数据分析平台，站点与站点之间的数据进行隔离。包括数据管理，数据处理、数据分析； |

**补充说明：公共管理站点本身的特殊性**

|  |  |
| --- | --- |
| **项目** | **描述** |
| **公共管理站点** | • 系统初始化时自动创建，站点编号：PUBLIC\_ADMIN，站点名称：公共管理 |
| **公共管理站点用户** | • 公共管理站点的用户即全局用户，通过用户管理模块统一维护 |
| **公共管理站点菜单** | • 仅包含：站点管理、用户管理、权限管理全局功能 |
