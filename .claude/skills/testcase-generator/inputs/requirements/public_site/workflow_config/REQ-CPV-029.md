## 原始需求内容

### 工作流配置

**1、工作流配置模块详细功能说明**

|  |  |
| --- | --- |
| **项目** | **描述** |
| **需求描述** | **1. 列表页面** • 展示系统内置的4大工作流：年度计划设置审批工作流、任务管理工作流、方案编制工作流、报告编制工作流。 |
| **页面交互** | • 工作流配置页面以流程图形式展示节点和连线，支持拖拽节点调整位置，点击节点弹出配置侧边栏。 • 增加/删减审批层级时，流程图动态刷新，自动维护节点连线。 • 审批人配置弹窗支持多选组织机构成员，支持按部门、岗位筛选。 • 配置完成后支持“确定”功能，验证流程正确性。 |
| **异常逻辑** | • 若审批层级中某节点未配置审批人，提交时提示“第X级审批人未配置，请补充”。 • 流程配置变更时，若已有进行中的流程实例，需提示“已有流程实例正在运行，新配置将应用于新提交的流程，现有流程不受影响”。 • 若提交时，若发现没有给对应的成员配置模块的工作流，提交时，出提示：**您当前暂无审批流程，请联系管理员配置。(暂时不做）** • **若提交时，**配置所属产品时，其他同模块的工作流已经配置过产品了，保存时需要提示：xx产品已经配置过工作流，请修改后重试。 • 提交时，如果选择了指定审批人，但未进行审批人的设置，保存时需要提示：第xx级审批人可选范围为指定审批人，请先选择审批人。 |
| **后置流程处理** | • 配置确定保存后，立即生效于对应模块的新提交流程，年度计划设置、任务管理、方案编制、报告编制这四大模块提交时，需要选择对应审批层级的审批人，每一个层级选择审批成员时，调用组织机构弹窗，可选择对应的审批成员，每一个审批层级至少要选择一个审批人，否则无法提交。审批层级根据各模块工作流中配置的审批层级自动获取。 |

**2、工作流配置列表字段说明**

|  |  |
| --- | --- |
| **字段名称** | **说明** |
| 模块名称 | 工作流所属业务模块名称及备注，如“年度计划设置审批工作流（计划生效审批）”。 |
| 创建人 | 工作流配置的创建者。 |
| 创建时间 | 工作流配置的创建时间。 |
| 更新人 | 最后修改工作流配置的用户。 |
| 更新时间 | 最后修改时间。 |
| 操作栏 | 编辑（修改基本信息）、配置（配置审批流程）。 |

**3、新增工作流（基本信息）字段详情表**

|  |  |  |  |
| --- | --- | --- | --- |
| **字段名称** | **输入类型** | **必填** | **说明** |
| 工作流名称 | 文本框 | 是 | 最多50字符，可编辑。 |
| 所在模块 | 下拉框 | 是 | 下拉框内容：年度计划设置、任务管理、方案编制、报告编制 |
| 所属产品 | 下拉框 | 是 | 下拉框内容为产品管理所有已配置的产品，显示为：产品名称（产品编号） |
| 工作流描述 | 文本域 | 否 | 最多500字符，用于说明流程用途。 |

**4、编辑工作流（基本信息）字段详情表**

|  |  |  |  |
| --- | --- | --- | --- |
| **字段名称** | **输入类型** | **必填** | **说明** |
| 工作流名称 | 文本框 | 是 | 最多50字符，可编辑。 |
| 所在模块 | 只读 | 是 | 默认带出，不可修改。 |
| 所属产品 | 下拉框 | 是 | 系统初始工作流，如果点编辑，页面上无所属产品字段。同时保存时不应对所属产品进行必填校验 |
| 工作流描述 | 文本域 | 否 | 最多500字符，用于说明流程用途。 |

**5、审批节点配置字段详情表**

|  |  |  |  |
| --- | --- | --- | --- |
| **配置项** | **配置选项** | **默认值** | **说明** |
| 审批发起方式 | 人工审批 | 人工审批 | 后续可扩展“自动通过”、“自动拒绝”等。 |
| 审批对象 | 发起人自选审批人 / 固定审批人 | 发起人自选审批人 | 发起人自选审批人：触发审批时，需要由发起人自己选择审批人。 固定审批人：固定审批人需指定具体人员。触发审批时，无需再选择审批人。 |
| 审批人可选范围 | 不限范围/同部门审批人/严格指定审批人 | 不限范围 | 审批对象，选择为：发起人自选审批人。审批人可选范围，选项：不限范围；同部门审批人；指定审批人 |
| 多人审批方式 | 会签 / 或签 | 会签 | 会签需所有审批人同意；或签一人同意或拒绝即可。 |
| 电子签名 | 开启 / 关闭 | 开启 | 开启后审批人必须手写电子签名方可提交审批意见。 |
| 允许加签 | 开启 / 关闭 | 关闭 | 开启后当前审批人可新增临时审批人。 |
| 允许转交 | 开启 / 关闭 | 关闭 | 开启后当前审批人可将审批事项转交给临时审批人。 |

**说明**：所有配置项支持后续扩展，如增加条件分支、超时处理、自动提醒等高级功能。配置完成后，需在对应业务模块中生效，确保提交审批时按配置流程执行。
