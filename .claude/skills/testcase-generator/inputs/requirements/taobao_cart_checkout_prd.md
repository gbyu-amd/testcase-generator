# 淘宝购物车结算功能 PRD

## 1. 文档信息

| 项目 | 内容 |
|---|---|
| 文档名称 | 淘宝购物车结算功能 PRD |
| 适用端 | Web 端、移动 H5 |
| 关联模块 | 购物车、结算下单、登录、商品详情、优惠券、收货地址 |
| 目标用户 | 普通登录用户、未登录访客 |
| 文档状态 | 草稿 |

## 2. 需求背景

购物车结算是淘宝交易链路中承接“加购”和“下单”的核心环节。用户在购物车中管理商品、勾选商品、确认数量和金额后进入结算页，并在结算页确认地址、优惠、商品状态和应付金额。

为降低错误下单、金额不一致、库存变化、优惠失效、地址不可配送等交易风险，需要明确购物车到结算页的交互规则、数据校验规则、异常处理和验收标准。

## 3. 需求目标

- 用户可以在购物车中查看、勾选、取消勾选、修改数量和删除商品。
- 用户可以从购物车中选择有效商品进入结算页。
- 结算页应展示准确的商品、地址、优惠和金额信息。
- 商品状态、库存、价格、优惠、地址发生变化时，系统应重新校验并给出明确提示。
- 未登录或登录态过期时，系统应阻断结算和提交订单。
- 重复点击结算或提交订单时，不应产生重复订单或错误扣库存。

## 4. 功能范围

### 4.1 本期范围

- 购物车商品列表展示。
- 商品勾选、取消勾选、全选和取消全选。
- 修改商品数量。
- 删除单个商品和批量删除商品。
- 购物车金额、优惠金额和合计金额计算。
- 从购物车进入结算页。
- 结算页地址选择、优惠选择、订单金额确认。
- 提交订单前的商品、库存、价格、优惠、地址和登录态校验。
- 异常状态下的拦截和提示。

### 4.2 非本期范围

- 支付流程。
- 售后退款流程。
- 商家后台库存管理。
- 优惠券领取流程。
- 跨境商品清关信息填写。

## 5. 用户角色

| 用户角色 | 说明 |
|---|---|
| 未登录访客 | 不能查看个人购物车和进入结算页，需要先登录 |
| 普通登录用户 | 可以管理购物车、选择商品并提交订单 |
| 异常账号用户 | 账号被冻结、限制交易或风控未通过时，不能提交订单 |

## 6. 功能入口

- 首页购物车入口。
- 商品详情页“加入购物车”后进入购物车。
- 顶部导航购物车角标入口。
- 购物车页“结算”按钮。
- 登录成功后回跳购物车或结算页。

## 7. 购物车页面规则

### 7.1 商品列表展示

| 字段 | 展示规则 |
|---|---|
| 商品名称 | 展示当前商品标题，点击可进入商品详情页 |
| 商品图片 | 展示商品主图 |
| 商品规格 | 展示用户选择的 SKU 信息 |
| 商品单价 | 展示当前有效售价 |
| 商品数量 | 默认展示已加入购物车数量 |
| 商品状态 | 展示在售、失效、下架、库存不足等状态 |
| 小计金额 | 单价乘以数量，并根据优惠规则参与合计 |

### 7.2 勾选规则

- 在售且库存充足的商品可以勾选。
- 下架、删除、失效商品不可勾选结算。
- 库存不足商品可展示在购物车中，但应提示库存不足并禁止进入结算或要求用户修改数量。
- 全选只应选中当前可结算商品。
- 取消勾选后，该商品不参与合计金额和结算。

### 7.3 数量规则

| 场景 | 处理规则 |
|---|---|
| 数量小于 1 | 系统阻止修改或恢复为 1 |
| 数量等于库存 | 允许保存，金额同步刷新 |
| 数量超过库存 | 系统阻止修改或恢复为最大可购数量，并提示库存不足 |
| 数量修改成功 | 小计金额、合计金额和优惠金额同步刷新 |
| 数量修改接口失败 | 保留原数量，提示修改失败 |

## 8. 结算页规则

### 8.1 结算信息展示

| 信息 | 展示规则 |
|---|---|
| 商品信息 | 展示已勾选商品的名称、规格、单价、数量和小计 |
| 收货地址 | 默认展示用户默认地址；无默认地址时提示选择或新增 |
| 优惠信息 | 展示可用优惠券、不可用优惠券和优惠原因 |
| 金额明细 | 展示商品总价、优惠金额、运费和应付金额 |
| 提交订单按钮 | 校验通过时可点击；关键数据异常时置灰或拦截 |

### 8.2 金额计算规则

- 商品总价等于所有有效结算商品的小计金额之和。
- 优惠金额只能来自当前订单可用优惠。
- 应付金额等于商品总价加运费减优惠金额。
- 修改商品数量、切换地址或选择优惠券后，应重新计算应付金额。
- 商品价格变化时，结算页应提示用户价格已变化，并刷新金额。

### 8.3 地址规则

- 存在默认地址时，结算页默认选中该地址。
- 用户可以切换收货地址。
- 地址不可配送时，提交订单按钮不可用或提交时被拦截。
- 无收货地址时，系统应提示用户新增地址。

### 8.4 优惠规则

- 可用优惠券允许选择，并同步刷新优惠金额和应付金额。
- 不可用优惠券展示不可用原因。
- 优惠券过期、门槛不满足或已被使用时，禁止参与结算。
- 提交订单前优惠券状态发生变化时，系统应重新校验并提示用户确认最新金额。

## 9. 主流程

### 9.1 用户从购物车提交订单成功

1. 用户登录淘宝账号。
2. 用户打开购物车页面。
3. 用户勾选一个或多个在售且库存充足的商品。
4. 系统实时计算商品总价、优惠金额和合计金额。
5. 用户点击“结算”按钮。
6. 系统校验登录态、商品状态、库存和价格。
7. 用户进入结算页。
8. 用户确认默认地址、商品信息、优惠和应付金额。
9. 用户点击“提交订单”。
10. 系统再次校验登录态、库存、价格、优惠和地址。
11. 校验通过后生成订单，并进入支付前页面。

### 9.2 用户修改购物车数量后结算

1. 用户打开购物车页面。
2. 用户修改已勾选商品数量。
3. 系统保存数量并刷新小计金额和合计金额。
4. 用户点击“结算”。
5. 系统使用最新数量进入结算页。
6. 结算页金额与购物车最新金额保持一致。

## 10. 分支流程与异常处理

| 场景 | 处理规则 |
|---|---|
| 未登录访问购物车 | 跳转登录页或弹出登录引导，登录成功后回到购物车 |
| 未登录点击结算 | 跳转登录页，登录成功后继续回到购物车或结算页 |
| 登录态过期 | 拦截结算或提交订单，重新登录后重新校验数据 |
| 未勾选商品点击结算 | 提示“请先选择商品”，不进入结算页 |
| 勾选失效商品结算 | 阻止结算，提示商品已失效 |
| 商品库存不足 | 阻止结算或提交订单，提示库存不足并引导修改数量 |
| 商品价格变化 | 刷新价格和金额，提示用户确认最新价格 |
| 优惠券失效 | 移除优惠或置为不可用，刷新应付金额 |
| 地址不可配送 | 阻止提交订单，提示该地址不可配送 |
| 提交订单接口失败 | 页面提示提交失败，不生成订单或保持订单失败状态 |
| 重复点击提交订单 | 提交期间按钮置灰或加锁，只生成一笔订单 |
| 购物车数量修改失败 | 保留原数量，提示稍后重试 |

## 11. 页面跳转规则

- 从购物车点击“结算”后进入结算页。
- 未登录用户触发结算时先进入登录页，登录成功后回到原结算入口。
- 结算页点击商品名称可回到商品详情页。
- 提交订单成功后进入支付前页面。
- 提交订单失败时停留在结算页，并展示失败原因。
- 浏览器返回购物车时，购物车应重新拉取最新商品状态和金额。

## 12. 与其他模块联动

| 关联模块 | 联动规则 |
|---|---|
| 登录 | 购物车和结算页必须校验登录态；登录态过期不能继续结算 |
| 商品详情 | 商品状态、SKU、价格和库存变化应影响购物车和结算页 |
| 优惠券 | 优惠券可用性和优惠金额影响订单应付金额 |
| 收货地址 | 地址可配送性影响提交订单按钮状态 |
| 订单 | 提交订单成功后生成唯一订单号，失败时不应生成重复订单 |
| 多 Tab | 多 Tab 修改购物车或提交订单时，应以服务端最新状态为准 |

## 13. 安全与风控要求

- 提交订单必须校验用户登录态和账号状态。
- 订单金额必须以后端最终计算结果为准，前端展示金额不能直接作为下单依据。
- 重复提交必须有幂等控制，不能生成重复订单。
- 商品库存扣减必须与订单生成保持一致。
- 异常接口响应不能暴露后端堆栈或内部错误信息。
- 风控异常账号不能绕过结算页提交订单。

## 14. 验收标准

| 编号 | 验收标准 |
|---|---|
| AC_CART_CHECKOUT_001 | 用户勾选在售且库存充足商品后，购物车合计金额应正确刷新 |
| AC_CART_CHECKOUT_002 | 用户修改商品数量后，小计金额、合计金额和结算页金额应保持一致 |
| AC_CART_CHECKOUT_003 | 用户未勾选商品点击结算时，系统应提示先选择商品，不进入结算页 |
| AC_CART_CHECKOUT_004 | 失效、下架或删除商品不能参与结算 |
| AC_CART_CHECKOUT_005 | 商品数量超过库存时，系统应阻止结算或提交订单，并提示库存不足 |
| AC_CART_CHECKOUT_006 | 未登录用户访问购物车或点击结算时，应先引导登录 |
| AC_CART_CHECKOUT_007 | 登录态过期后点击提交订单，系统应拦截操作，不生成订单 |
| AC_CART_CHECKOUT_008 | 结算页默认展示用户默认收货地址；无地址时提示新增地址 |
| AC_CART_CHECKOUT_009 | 地址不可配送时，系统应阻止提交订单并展示明确提示 |
| AC_CART_CHECKOUT_010 | 用户选择可用优惠券后，应付金额应实时减少 |
| AC_CART_CHECKOUT_011 | 优惠券失效或不可用时，系统应展示不可用原因并刷新应付金额 |
| AC_CART_CHECKOUT_012 | 商品价格在结算期间变化时，系统应刷新金额并提示用户确认 |
| AC_CART_CHECKOUT_013 | 提交订单接口失败时，页面应提示提交失败，不生成重复订单 |
| AC_CART_CHECKOUT_014 | 用户连续快速点击提交订单时，系统只允许生成一笔订单 |
| AC_CART_CHECKOUT_015 | 多 Tab 同时修改购物车或提交订单时，页面应以服务端最新状态刷新 |

## 15. 数据埋点建议

| 事件 | 触发时机 |
|---|---|
| cart_page_view | 用户进入购物车页 |
| cart_item_select | 用户勾选或取消勾选商品 |
| cart_quantity_change | 用户修改商品数量 |
| cart_delete_item | 用户删除购物车商品 |
| cart_checkout_click | 用户点击结算按钮 |
| checkout_page_view | 用户进入结算页 |
| checkout_coupon_select | 用户选择优惠券 |
| checkout_address_change | 用户切换收货地址 |
| order_submit_click | 用户点击提交订单 |
| order_submit_success | 订单提交成功 |
| order_submit_fail | 订单提交失败 |

## 16. 待确认问题

- 购物车商品数量上限是否按商品、店铺或用户维度限制。
- 商品价格变化是否需要用户显式确认后才能继续提交订单。
- 运费计算是否纳入本期结算页金额规则。
- 优惠券与店铺活动、满减活动是否存在叠加规则。
- 多 Tab 购物车同步是实时推送，还是用户刷新/操作时重新拉取。
- 提交订单失败后是否需要保留用户已选择的地址和优惠券。
