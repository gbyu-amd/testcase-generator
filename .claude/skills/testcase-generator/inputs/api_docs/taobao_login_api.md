# 淘宝登录接口文档（样例）

> 本文件为接口文档样例，供 Agent 生成"接口驱动"测试用例时参考格式与提取重点。
> 实际项目应替换为真实接口定义。

## 1. 发送短信验证码

### 基本信息

| 项目 | 内容 |
|---|---|
| 接口名称 | 发送登录验证码 |
| 请求路径 | `/api/login/sms/send` |
| 请求方式 | POST |
| 是否需要登录 | 否 |
| 幂等规则 | 同一手机号 60 秒内重复请求返回 `429`，不重复下发 |

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| mobile | string | 是 | 11 位手机号 |
| scene | string | 是 | 固定值 `login` |

### 返回字段

| 字段 | 类型 | 说明 |
|---|---|---|
| code | int | 业务状态码，0 表示成功 |
| message | string | 提示文案 |
| data.expireSeconds | int | 验证码有效期秒数，默认 300 |

### 状态码与错误码

| HTTP 状态码 | 业务 code | 错误码 | 业务错误提示 | 前端处理 |
|---|---|---|---|---|
| 200 | 0 | - | 发送成功 | 启动 60 秒倒计时 |
| 200 | 1001 | MOBILE_INVALID | 请输入正确的手机号 | 不启动倒计时，输入框报错 |
| 429 | 1002 | SEND_TOO_FREQUENT | 操作过于频繁，请稍后再试 | 保持倒计时，禁用按钮 |
| 500 | 9000 | SERVER_ERROR | 发送失败，请稍后重试 | 允许用户重新获取，不启动倒计时 |

## 2. 登录（账号密码 / 验证码）

### 基本信息

| 项目 | 内容 |
|---|---|
| 接口名称 | 用户登录 |
| 请求路径 | `/api/login/submit` |
| 请求方式 | POST |
| 是否需要登录 | 否 |
| 幂等规则 | 同一请求重复提交返回上一次结果，不重复建立多个会话 |

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| loginType | string | 是 | `password` 或 `sms` |
| account | string | 条件必填 | 手机号 / 邮箱 / 会员名，`password` 时必填 |
| password | string | 条件必填 | 密码，6-32 位，`password` 时必填 |
| mobile | string | 条件必填 | 手机号，`sms` 时必填 |
| smsCode | string | 条件必填 | 短信验证码，`sms` 时必填 |
| redirect | string | 否 | 登录成功后回跳地址 |

### 返回字段

| 字段 | 类型 | 说明 |
|---|---|---|
| code | int | 业务状态码，0 表示成功 |
| message | string | 提示文案 |
| data.token | string | 登录态令牌 |
| data.nickname | string | 用户昵称 |
| data.redirect | string | 实际回跳地址 |

### 状态码与错误码

| HTTP 状态码 | 业务 code | 错误码 | 业务错误提示 | 前端处理 |
|---|---|---|---|---|
| 200 | 0 | - | 登录成功 | 建立登录态，按 redirect 回跳 |
| 200 | 2001 | CREDENTIAL_INVALID | 账号或密码错误 | 停留登录页，保留账号输入 |
| 200 | 2002 | SMS_CODE_INVALID | 验证码错误，请重新输入 | 停留登录页，清空验证码框 |
| 200 | 2003 | SMS_CODE_EXPIRED | 验证码已过期，请重新获取 | 提示重新获取验证码 |
| 200 | 2004 | ACCOUNT_FROZEN | 账号状态异常，请联系客服 | 不建立登录态 |
| 401 | 2005 | NEED_SECOND_VERIFY | 需要安全验证 | 跳转二次验证流程 |
| 500 | 9000 | SERVER_ERROR | 登录失败，请稍后重试 | 保留输入，允许重试 |

## 3. 接口测试用例提取重点

读取本类接口文档时，应转化为以下前端可执行场景：

- **成功路径**：每个 `code=0` 分支对应一条正向用例（建立登录态、正确回跳）。
- **业务错误码**：每个错误码对应一条异常用例，验证页面提示文案与处理动作。
- **HTTP 异常**：500 / 超时对应接口失败用例，验证"稍后重试"且不清空输入。
- **幂等规则**：重复提交对应一条回归/异常用例，验证不重复下发验证码、不重复建会话。
- **必填字段**：条件必填参数缺失对应字段校验用例。
- **权限规则**：`NEED_SECOND_VERIFY`、`ACCOUNT_FROZEN` 对应权限/异常账号用例。
