# 界面设计图说明

## 存放内容

本目录用于存放 CPV 系统的界面设计图、截图说明、交互说明、流程图、弹窗说明和视觉标注文档。设计资料可覆盖公共管理站点、自定义业务站点、年度计划、任务、方案、监控项目、数据分析、一键分析、报告编制、审计追踪、电子签名和异步任务等页面。

## 推荐目录结构

UI 目录应按拆分后的需求编号组织，便于和 `inputs/requirements/**/REQ-*.md` 一一对应：

```text
ui_design/
  _incoming/
    README.md
    待归类图片.png
  REQ-CPV-001/
    README.md
    01_list.png
    02_detail.png
  REQ-CPV-002/
    README.md
  _unassigned/
    README.md
```

- UI 原图必须放在 `inputs/ui_design/` 下，`docx/` 仅作为临时杂文件目录，不作为 UI 来源。
- 新收到但未确认归属的图片可先放入 `_incoming/`，确认后人工移动到对应 `REQ-*` 目录。
- `REQ-*` 目录由 `scripts/split_requirements.py` 自动创建。
- 每个 `README.md` 记录关联需求、图片清单、页面说明和交互说明。
- `_unassigned/README.md` 仅作为人工整理清单，不由拆分脚本自动维护。

生成方式：

```bash
python scripts/split_requirements.py --prd "inputs/requirements/current_prd.md" --overwrite
```

## 阅读重点

读取界面设计图时，应重点提取：

- 页面入口
- 页面标题
- 输入框、按钮、弹窗、提示条等控件
- 默认状态、选中状态、禁用状态和错误状态
- 列表、卡片、详情页、审批记录、状态标签、进度条、图表和报告预览等展示字段
- 菜单权限、按钮权限和无权限状态
- 站点切换、当前站点高亮和公共管理/自定义站点差异
- 表单字段、必填标识、字段类型、枚举项、长度限制和校验提示
- 审批、批准、生效、退回、修订、失效等状态和操作按钮
- 电子签名弹窗、签名失败提示和签名后记录展示
- 审计追踪列表、筛选条件、操作前后值和导出入口
- 数据分析图表、字段配置、动态表单、未分析标签和悬停原因
- 异步任务状态、失败原因、重试入口和结果下载入口
- 页面跳转关系
- 空状态、加载状态和失败状态

## 用例生成要求

界面设计图不仅用于验证页面展示，还应转化为交互测试用例。例如按钮是否可点击、无权限或状态不允许时是否禁用、错误提示是否展示、审批弹窗和电子签名是否触发、页面刷新或返回后状态是否保留、跨模块跳转后数据是否一致。

## CPV 典型关注点

- 登录页站点下拉是否只展示当前用户有权限访问的公共管理站点和自定义业务站点。
- 顶部站点列表是否高亮当前站点，跨站点切换后菜单和业务数据是否刷新。
- 公共管理站点是否只展示全局管理菜单，不展示计划、任务、方案、报告等 CPV 业务菜单。
- 年度计划、任务、方案、监控项目和报告列表中的状态标签、筛选条件、操作按钮是否与业务状态一致。
- 草稿、审批中、已生效、退回、修订、失效、已分析、已确认、关闭等状态下按钮是否正确显示、置灰或隐藏。
- 审批、批准、生效等关键动作是否弹出电子签名确认，并展示明确的失败提示。
- 方案生效后生成的监控项目是否能在页面中追溯到对应方案和任务。
- 数据分析页面的算法选择、字段配置、图表配置和动态表单是否联动刷新。
- 替换数据源弹窗的分类树、数据管理/数据处理页签、系统集成/本地数据页签、检索和单选交互是否符合设计。
- 一键分析全部成功、部分失败、全部失败时的弹窗文案、数量展示、未分析标签和悬停原因是否准确。
- 字段删减后配置区域是否清空失效字段，且不显示 `NULL` 或无效引用。
- 报告编制页面是否正确展示任务、方案、监控项目、分析结果和图表引用关系。
- 异步导入、导出、报告生成、一键分析等长耗时操作是否展示进度、失败原因、重试入口和结果下载入口。
- 审计追踪页面是否展示操作人、操作时间、模块、操作类型、受影响对象、操作前后值和客户端信息。
