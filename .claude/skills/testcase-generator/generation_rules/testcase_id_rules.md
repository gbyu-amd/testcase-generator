# 用例编号规则

## 编号格式

用例编号采用以下格式：

```text
模块名-类型-三位序号
```

示例：

- `登录-功能-001`
- `搜索-边界-001`
- `商品详情-异常-001`
- `购物车-回归-001`
- `结算下单-功能-001`

## 类型取值

| 类型 | 含义 |
|---|---|
| 功能 | 正常业务流程 |
| 异常 | 错误、失败、状态异常 |
| 边界 | 数量、长度、金额等边界 |
| 权限 | 登录态、账号状态、角色限制 |
| 回归 | 历史缺陷修复验证 |
| 兼容 | 浏览器、设备、网络兼容 |
| 联动 | 跨页面或跨模块状态同步 |

## 递增规则

- 同一输出文件内，同一模块、同一类型下按三位序号递增。
- 生成新输出文件时，从 `001` 开始编号。
- 追加到 `outputs/origin_exports/<module_name>_testcases.md` 时，必须读取该输出文件内已有编号并继续递增。
- `testcase_templates/` 中的参考用例编号只用于理解场景，不参与本次输出编号。

## 新模块规则

新增模块时，应先确定中文模块名和输出文件名，再从 `001` 开始编号并保存到 `outputs/origin_exports/`。
