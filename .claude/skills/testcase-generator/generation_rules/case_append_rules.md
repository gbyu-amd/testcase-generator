# 用例补充生成规则

## 补充生成流程

1. 判断新需求所属功能模块。
2. 读取 `testcase_templates/modules/` 下对应模块的 `module_overview.md`（只读参考）。
3. 读取对应模块的 `testcases.md`（只读参考）。
4. 检查已有用例是否覆盖相同场景。
5. 如果已有用例可以复用，应在本次输出中说明复用或补充点，不重复生成相同场景。
6. 如果确认为新场景，在本次输出文件中从 `001` 开始生成编号。
7. 将本次生成结果保存到 `outputs/origin_exports/<module_name>_testcases.md`。

## 模块不存在时

如果新需求不属于现有模块：

1. 不修改 `testcase_templates/modules/`。
2. 基于用户资料和通用模板生成扩展模块用例。
3. 保存到 `outputs/origin_exports/<module_name>_testcases.md`。
4. 在输出中说明该模块是扩展模块，以及缺失的参考资料和生成假设。

## 去重规则

以下情况视为可能重复：

- 测试目标相同。
- 前置条件相同。
- 操作步骤高度相似。
- 预期结果相同。

重复场景应合并，不应简单新增。

## 来源记录

每条新增用例必须记录来源，来源可以是：

- 需求文档
- 界面设计图
- 接口文档
- 业务补充资料
- 历史缺陷
- 项目知识库
