# 覆盖维度规则

## 必须覆盖维度

每个 CPV 功能模块生成用例时，至少评估以下 10 个维度。若某维度与当前需求无关，必须在生成假设或覆盖率对照中说明原因；涉及核心业务链路时，每个维度至少生成 1 条用例。

| 维度 | 说明 | 对应用例类型 |
|---|---|---|
| 正向主流程 | 用户按角色权限完成当前模块的标准业务操作，如创建、提交、审批、生效、分析、报告输出 | 正向 |
| 异常流程 | 接口失败、网络中断、数据源失效、插件不可用、审批失败、异步任务失败等异常处理 | 异常 |
| 边界值 | 字段长度、文件大小、批次数量、周期范围、阶段数、控制限、规格限、日期区间等临界值 | 边界 |
| 字段与数据校验 | 必填项、格式、唯一性、数据类型、单位、范围、完整性、字段匹配和无效引用校验 | 异常 / 边界 |
| 权限与站点隔离 | 未登录、登录态过期、无菜单权限、无按钮权限、无产品数据权限、无站点权限、跨站点数据访问拦截 | 权限 |
| 状态流转 | 草稿、审批中、已生效、退回、修订、失效、已分析、已确认、关闭等状态变化及可操作范围 | 正向 / 异常 |
| 数据一致性 | 计划、任务、方案、监控项目、相关性分析、分析结果、报告之间的引用、状态和版本一致 | 联动 / 回归 |
| 合规追溯 | 审计追踪、电子签名、审批意见、版本历史、操作前后值、失败原因和重试记录 | 回归 / 权限 |
| 跨模块联动 | 当前模块操作对下游任务、方案、监控项目、相关性分析、分析结果、报告、待办和通知的影响 | 联动 |
| 兼容性与稳定性 | 不同浏览器、分辨率、弱网、重复点击、页面刷新、返回、多页签、长耗时任务恢复 | 兼容 / 回归 |

## 各模块特有维度

### 登录与站点选择（核心链路）

除必须覆盖维度外，还需覆盖：

| 额外维度 | 示例场景 |
|---|---|
| 站点列表权限 | 登录页只展示当前账号可访问的公共管理站点和自定义业务站点 |
| 站点类型跳转 | 选择公共管理后进入统一门户，选择自定义站点后进入对应业务站点首页或门户 |
| 禁用站点拦截 | 站点被禁用后不可登录访问，关联业务数据不可见或不可操作 |
| 统一认证 | 公共管理与自定义站点之间切换时不要求重复登录 |
| 会话失效 | Token/Session 过期后继续保存、提交、审批、导出时被拦截并提示 |

### 公共管理站点

| 额外维度 | 示例场景 |
|---|---|
| 业务边界 | 公共管理不展示计划、任务、方案、监控项目、相关性分析、数据分析、报告等 CPV 业务菜单 |
| 用户账号唯一性 | 新增或导入用户时校验账号唯一，并同步到授权业务站点 |
| 站点状态控制 | 禁用站点后，站点入口、用户授权和业务数据访问同步受限 |
| 权限配置 | 公共管理站点通过权限管理配置菜单和按钮权限；启用、停用和多权限并集后展示正确 |
| 业务站点角色同步 | 业务站点角色管理中的菜单、按钮和产品数据权限变更后，在业务站点及时生效 |
| 系统管理员权限 | 系统管理员拥有顶级权限，可见所有站点下的菜单、按钮和产品数据 |
| 公共配置范围 | 登录配置、密码策略、电子签名策略、水印配置仅公共管理可维护，并对所有站点生效 |

### 产品与基础数据

| 额外维度 | 示例场景 |
|---|---|
| 产品唯一性 | 产品编码、剂型、工艺路线等基础信息按规则校验唯一或必填 |
| 关键对象关联 | CPP、CQA、CMA、IPC 与产品、工艺路线、阶段、批次关系正确 |
| 引用保护 | 已被计划、方案、监控项目或报告引用的数据不可随意删除或失效 |
| 导入导出 | 批量导入校验字段、重复数据、失败明细和部分成功结果；导出结果与筛选条件一致 |
| 历史版本 | 关键基础数据变更保留版本、原因、操作人和操作时间 |

### 年度计划与任务管理（核心链路）

除必须覆盖维度外，还需覆盖：

| 额外维度 | 示例场景 |
|---|---|
| 生效生成任务 | 年度计划审批生效后按产品、周期、频率自动生成周期性任务 |
| 未生效拦截 | 草稿、审批中、退回或失效计划不生成正式任务 |
| 修订与失效 | 计划修订或失效后，不继续生成不符合当前规则的新任务 |
| 任务进度 | 任务关联方案和报告后，进度、状态、日历视图和待办同步更新 |
| 任务关闭 | 已完成或关闭任务不允许继续执行不符合状态的编辑、提交或关联操作 |

### 方案模板与方案编制（核心链路）

除必须覆盖维度外，还需覆盖：

| 额外维度 | 示例场景 |
|---|---|
| 模板引用 | 方案只能引用已生效模板；失效模板不可被新方案引用 |
| 方案审批 | 提交、审核、退回、批准、生效按工作流流转，并保留意见和电子签名 |
| 监控项目生成 | 方案生效后按方案内容生成或定义监控项目 |
| 升版影响 | 模板或方案升版后，历史已生效版本保持不变，新方案引用最新有效版本 |
| 内容锁定 | 已批准、已生效或已归档方案的关键内容不可被普通编辑直接覆盖 |

### 监控项目

| 额外维度 | 示例场景 |
|---|---|
| 状态流转 | 监控项目按草稿、未分析、已分析、已确认、关闭等状态流转 |
| 分析前置 | 方案未生效或监控项目未进入可分析状态时，不允许执行数据分析 |
| 结果确认 | 未分析或未确认结果不应作为最终报告结论依据 |
| 重新分析 | 数据源、字段或配置变化后，重新分析结果正确更新并保留历史记录 |
| 锁定保护 | 已确认结果不允许普通编辑覆盖，需走退回、修订或重新分析流程 |

### 相关性分析

| 额外维度 | 示例场景 |
|---|---|
| 状态流转 | 相关性分析按草稿、未分析、已分析、已确认、关闭等状态流转 |
| 分析图范围 | 相关性分析模块仅允许相关性分析和 Pearson 相关性分析，其他监控类分析图置灰或不可操作 |
| 结果确认 | 未分析或未确认的相关性分析结果不应作为最终报告结论依据 |
| 结果回传 | 分析结果推送至 CPV 系统后，回传数据、分析图、结论和失败原因记录正确 |
| 报告引用 | 报告管理引用相关性分析结果时，来源站点、任务链路和确认状态应一致 |

### 数据分析（核心链路）

除必须覆盖维度外，还需覆盖：

| 额外维度 | 示例场景 |
|---|---|
| 文件校验 | Excel 格式、大小、空文件、损坏文件、字段缺失、非法字段、数据类型和范围校验 |
| 插件状态 | 选择算法后检查插件已安装、未安装、安装失败、不可用等状态 |
| 配置联动 | 分析算法、分析项目、分析图配置、维度字段和动态表单实时联动 |
| 开始分析校验 | 算法、分析项目、字段配置、动态表单缺失时禁止开始分析 |
| 结果处理 | 分析结果可保存到当前系统或推送到其他业务系统，并记录处理状态 |
| 回传失败 | 推送失败时记录接口状态、返回信息、失败原因，并支持重试或人工处理 |
| 入口差异 | 数据分析工作台、监控项目和相关性分析三类入口的数据来源、分析图范围和结果处理规则正确 |

### 替换数据源与一键分析（核心链路）

除必须覆盖维度外，还需覆盖：

| 额外维度 | 示例场景 |
|---|---|
| 入口范围 | 数据分析工作台、监控项目和相关性分析的数据分析详情页均可通过数据替换或数据处理触发一键分析 |
| 数据源选择 | 替换数据源弹窗分类树、数据管理/数据处理页签、系统集成/本地数据页签和单选规则正确 |
| 处理规则同步 | 选择数据管理表时清空排序、筛选、转换、异常数据剔除；选择数据处理表时保留排序、筛选、转换并清空异常数据剔除 |
| 自动一键分析 | 替换数据源成功后，系统自动按字段名称、变量类型和数据分布匹配已有分析项 |
| 结果提示 | 全部成功、部分成功部分失败、全部失败三类提示文案、成功数和失败数正确 |
| 未分析原因 | 定类变量不支持、字段删减、正态性变化、阶段内容不一致等原因按优先级展示 |
| 配置清理 | 字段删减后清空失效字段，不显示 `NULL` 或无效引用 |
| 异常恢复 | 数据源失效、无公共字段、网络中断或服务器异常时保留可恢复页面状态 |

### 报告模板与报告编制（核心链路）

除必须覆盖维度外，还需覆盖：

| 额外维度 | 示例场景 |
|---|---|
| 模板引用 | 报告只能引用已生效报告模板，失效模板不可被新报告引用 |
| 数据引用 | 报告引用的任务、方案、监控项目、相关性分析、分析结果和图表均来自当前站点和当前业务链路 |
| 结论前置 | 未确认监控项目、未确认相关性分析或未完成分析结果不应进入最终报告结论 |
| 审批生效 | 报告审核、批准、生效按工作流流转，关键动作需电子签名 |
| 任务回推 | 报告生效后推进关联任务状态，如从进行中变为已完成 |
| 输出与异步 | 报告导出、报告生成、批量处理进入异步任务，并支持状态查询、下载和失败重试 |

### 审计追踪与电子签名

| 额外维度 | 示例场景 |
|---|---|
| 操作留痕 | 新增、编辑、删除、启用/禁用、提交、审核、批准、退回、导入导出等关键动作均记录审计 |
| 前后值记录 | 关键字段变更记录操作前值、操作后值、变更原因、操作人和时间 |
| 签名校验 | 审批、批准、生效等关键动作校验电子签名密码和权限 |
| 失败处理 | 电子签名失败、密码错误、权限不足、签名超时应阻断动作并记录失败原因 |
| 审计检索 | 审计追踪支持按模块、操作类型、受影响对象、操作人和时间范围检索与导出 |

### 系统配置与异步任务

| 额外维度 | 示例场景 |
|---|---|
| 配置权限 | 非管理员不能访问用户、角色、权限、工作流、系统服务等配置页面 |
| 工作流影响 | 工作流配置变更影响后续新流程，不破坏已进行中的审批流程 |
| 外部接口 | 外部数据源接口配置错误时提示失败原因，并记录日志 |
| 异步执行 | 导入、导出、报告生成、一键分析等耗时任务支持提交、查询、完成、失败、重试 |
| 重复提交防护 | 弱网或重复点击时，提交、审批、导入导出、报告生成、一键分析不重复执行 |

## CPV 高风险维度（任何模块均应评估）

以下场景一旦涉及，必须生成对应用例：

| 高风险场景 | 涉及模块 | 最低优先级 |
|---|---|---|
| 未生效年度计划生成正式周期性任务 | 年度计划、任务管理 | P0 |
| 失效模板被新方案或新报告引用 | 方案模板、方案编制、报告模板、报告编制 | P0 |
| 方案未生效即生成正式监控项目或进入分析 | 方案编制、监控项目、数据分析 | P0 |
| 未分析或未确认监控项目、相关性分析进入最终报告结论 | 监控项目、相关性分析、报告编制 | P0 |
| 报告生效后未推进关联任务状态 | 报告编制、任务管理 | P0 |
| 跨站点读取或写入计划、任务、方案、监控项目、相关性分析、报告数据 | 登录、站点管理、所有业务模块 | P0 |
| 无按钮权限或接口越权仍可提交、审批、批准、导出 | 权限管理、工作流、所有业务模块 | P0 |
| 无产品数据权限仍可查询、分析、导出或引用业务数据 | 业务站点角色管理、所有业务模块 | P0 |
| 审批、批准、生效未进行电子签名或未记录审计追踪 | 工作流、电子签名、审计追踪 | P0 |
| 已批准、已归档、已锁定数据被普通编辑覆盖 | 方案编制、监控项目、报告编制、基础数据 | P0 |
| 数据源失效、字段删减或 `NULL` 残留仍继续生成有效分析结果 | 数据分析、一键分析、监控项目、相关性分析 | P0 |
| 替换数据源后数据处理规则保留/清空逻辑错误 | 数据分析、一键分析 | P0 |
| 一键分析全部失败或部分失败时未提示未分析原因 | 数据分析、一键分析 | P0 |
| 分析结果回传失败但监控项目或相关性分析状态仍更新为已分析 | 数据分析、监控项目、相关性分析 | P0 |
| 报告引用跨任务、跨方案、跨站点或未确认的相关性分析结果 | 报告编制、监控项目、相关性分析、数据分析 | P0 |
| 批量导入、审批、导出、报告生成重复提交导致重复数据或重复任务 | 异步任务、年度计划、报告编制、数据分析 | P0 |
| 关键业务变更缺失操作人、时间、原因、前后值或版本历史 | 审计追踪、所有核心模块 | P0 |

## 覆盖检查表

生成用例后，逐项确认以下问题：

**主流程覆盖**
- [ ] 是否覆盖用户按角色权限完成当前模块核心操作的完整正向路径？
- [ ] 正向路径是否验证页面跳转、状态变化、业务数据落库和下游模块影响？

**拦截覆盖**
- [ ] 是否覆盖用户不能完成操作时的拦截提示，如无权限、状态不允许、数据无效、插件不可用、数据源失效？
- [ ] 拦截后是否验证系统没有执行被拦截的业务动作，如不生成任务、不生效方案、不更新监控项目或相关性分析状态、不推进报告状态？

**状态与审批覆盖**
- [ ] 是否覆盖草稿、审批中、退回、已生效、修订、失效、关闭等状态下的可操作范围？
- [ ] 是否覆盖提交、审核、批准、退回、生效等工作流动作及电子签名要求？

**数据一致性**
- [ ] 是否覆盖计划、任务、方案、监控项目、相关性分析、分析结果、报告之间的引用一致性？
- [ ] 是否验证跨页面、跨模块、刷新、返回、重新进入详情页后的数据与服务端状态一致？

**站点与权限**
- [ ] 是否覆盖公共管理站点和自定义业务站点的菜单边界、数据隔离和站点切换？
- [ ] 是否覆盖业务站点角色的菜单权限、按钮权限、产品数据权限、启停和多角色权限并集？
- [ ] 是否覆盖公共管理站点权限管理中的菜单权限、按钮权限、启停和多权限并集？
- [ ] 是否覆盖接口越权、登录态失效和用户禁用后的访问控制？

**数据分析**
- [ ] 是否覆盖文件校验、插件状态、算法配置、字段配置、动态表单和开始分析前校验？
- [ ] 是否覆盖数据分析工作台、监控项目和相关性分析三类入口的分析图范围差异？
- [ ] 是否覆盖替换数据源、一键分析、未分析原因优先级、字段删减清理和数据源失效？

**合规追溯**
- [ ] 是否覆盖审计追踪、电子签名、审批意见、操作前后值、版本历史和失败原因记录？
- [ ] 是否确认已批准、已归档、已锁定数据不会被普通编辑动作覆盖？

**跨模块联动**
- [ ] 是否覆盖本模块操作对待办、通知、任务进度、监控项目状态、相关性分析状态、报告状态和异步任务的影响？
- [ ] 是否覆盖多页签、浏览器返回、页面刷新、弱网和重复点击场景？

**优先级分布**
- [ ] P0 用例是否聚焦在 CPV 核心链路、合规风险、数据完整性、权限隔离和报告结论准确性？
- [ ] P0 占比是否在 20%–35% 合理区间？
