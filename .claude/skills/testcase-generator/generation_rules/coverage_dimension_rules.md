# 覆盖维度规则

## 使用原则

生成或补充 CPV 测试用例时，先按本文件评估覆盖范围，再结合 PRD、UI 设计图、知识库和参考用例生成具体场景。

- 每个功能模块必须评估“必须覆盖维度”，只保留与当前需求有直接关系的场景。
- CPV 核心业务链路重点覆盖正向主流程、异常拦截、状态流转、数据一致性、合规追溯和跨模块联动。
- 涉及专项能力时，按“专项规则路由”读取对应规则文件，不在本文件展开业务细节。
- 本文件只回答“要从哪些维度评估覆盖”，不承担优先级、难度或具体模块用例模板职责。

## 必须覆盖维度

| 维度 | 覆盖重点 | 对应用例类型 |
|---|---|---|
| 正向主流程 | 用户按角色权限完成标准业务操作，如创建、提交、审批、生效、分析、报告输出 | 正例 |
| 异常流程 | 接口失败、网络中断、数据源失效、插件不可用、审批失败、异步任务失败等异常处理 | 反例 |
| 边界值 | 字段长度、文件大小、批次数量、周期范围、阶段数、控制限、规格限、日期区间等临界值 | 边界 |
| 字段与数据校验 | 必填项、格式、唯一性、数据类型、单位、范围、完整性、字段匹配和无效引用校验 | 反例 / 边界 |
| UI 设计图覆盖 | 页面区域、标题说明、字段、默认值、禁用态、按钮、表格列、结果区、提示文案和布局与 UI 设计图一致 | UI |
| 权限与站点隔离 | 未登录、登录态过期、无菜单权限、无按钮权限、无产品数据权限、无站点权限、跨站点访问拦截 | 权限 |
| 状态流转 | 草稿、审批中、已生效、退回、修订、失效、已分析、已确认、关闭等状态变化和可操作范围 | 正例 / 反例 |
| 数据一致性 | 计划、任务、方案、监控项目、相关性分析、分析结果、报告之间的引用、状态和版本一致 | 联动 / 回归 |
| 合规追溯 | 审计追踪、电子签名、审批意见、版本历史、操作前后值、失败原因和重试记录 | 回归 / 权限 |
| 跨模块联动 | 当前模块操作对下游任务、方案、监控项目、相关性分析、分析结果、报告、待办和通知的影响 | 联动 |
| 兼容性与稳定性 | 不同浏览器、分辨率、弱网、重复点击、页面刷新、返回、多页签、长耗时任务恢复 | 兼容 / 回归 |

## CPV 高风险场景

以下场景一旦涉及，必须评估并生成对应用例；具体优先级以 `priority_rules.md` 为唯一判定来源。

| 高风险场景 | 覆盖重点 |
|---|---|
| 未生效、失效、未确认或已锁定数据被继续引用或编辑 | 状态拦截、数据不被写入、提示可验证 |
| 跨站点、无菜单权限、无按钮权限、无产品数据权限或接口越权 | 入口隐藏 / 禁用、接口拦截、业务数据不泄露 |
| 审批、批准、生效、报告输出等关键动作缺少电子签名或审计追踪 | 操作人、时间、原因、前后值、版本历史 |
| 数据源失效、字段删减、空值残留仍生成有效分析结果 | 分析阻断、未分析原因、状态不误更新 |
| 替换数据源或一键分析后数据处理规则、分析状态、结果回传错误 | 配置保留 / 清空、失败原因、重新分析 |
| 报告引用跨任务、跨方案、跨站点或未确认的分析结果 | 引用拦截、结论准确性、数据一致性 |
| 批量导入、审批、导出、报告生成重复提交 | 不产生重复数据、重复任务或重复审计 |

## 专项规则路由

命中以下场景时，必须读取对应专项规则，再生成或补充用例。未命中专项规则路由时，不得读取对应专项规则文件；不得为了泛化覆盖、参考风格或扩大覆盖面主动读取无关专项规则。

| 场景 | 必读规则 |
|---|---|
| 涉及 Excel / CSV / 模板文件导入、批量导入、导入模板、错误报告 | `generation_rules/import_coverage_rules.md` |
| 新增或补充分析方法、数据分析详情页、分析结果、数据源预览、替换数据源、一键分析 | `generation_rules/data_analysis_coverage_rules.md` |
| 涉及审计追踪、电子签名记录、操作日志、操作前后值、详情导出、合规追溯 | `generation_rules/audit_trail_coverage_rules.md` |

专项规则只补充覆盖场景，不替代：

- 字段写法遵守 `testcase_writing_guidelines.md`。
- 优先级判定遵守 `priority_rules.md`。
- 难度判定遵守 `difficulty_level_rules.md`。
