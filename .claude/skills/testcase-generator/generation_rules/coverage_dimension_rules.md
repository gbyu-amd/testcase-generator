# 覆盖维度规则

## 使用原则

生成或补充 CPV 测试用例时，先按本文件评估覆盖范围，再结合 PRD、UI 设计图、知识库和参考用例生成具体场景。

- 每个功能模块必须评估“必须覆盖维度”，只保留与当前需求有直接关系的场景。
- CPV 核心业务链路重点覆盖正向主流程、异常拦截、状态流转、数据一致性、合规追溯和跨模块联动。
- 涉及文件导入、导入模板、批量导入或错误报告时，必须检查“导入功能专项覆盖规则”。
- 新增或补充分析图 / 分析方法时，必须检查“数据分析专项覆盖规则”。

## 必须覆盖维度

| 维度 | 覆盖重点 | 对应用例类型 |
|---|---|---|
| 正向主流程 | 用户按角色权限完成标准业务操作，如创建、提交、审批、生效、分析、报告输出 | 正例 |
| 异常流程 | 接口失败、网络中断、数据源失效、插件不可用、审批失败、异步任务失败等异常处理 | 反例 |
| 边界值 | 字段长度、文件大小、批次数量、周期范围、阶段数、控制限、规格限、日期区间等临界值 | 边界 |
| 字段与数据校验 | 必填项、格式、唯一性、数据类型、单位、范围、完整性、字段匹配和无效引用校验 | 反例 / 边界 |
| UI 设计图覆盖 | 页面区域、标题说明、字段、默认值、禁用态、按钮、表格列、结果区、提示文案和布局与 UI 设计图一致 | UI |
| 权限与站点隔离 | 未登录、登录态过期、无菜单权限、无按钮权限、无产品数据权限、无站点权限、跨站点访问拦截 | 权限 |
| 状态流转 | 草稿、审批中、已生效、退回、修订、失效、已分析、已确认、关闭等状态变化和可操作范围 | 正例 / 反例 |
| 数据一致性 | 计划、任务、方案、监控项目、相关性分析、分析结果、报告之间的引用、状态和版本一致 | 联动 / 回归 |
| 合规追溯 | 审计追踪、电子签名、审批意见、版本历史、操作前后值、失败原因和重试记录 | 回归 / 权限 |
| 跨模块联动 | 当前模块操作对下游任务、方案、监控项目、相关性分析、分析结果、报告、待办和通知的影响 | 联动 |
| 兼容性与稳定性 | 不同浏览器、分辨率、弱网、重复点击、页面刷新、返回、多页签、长耗时任务恢复 | 兼容 / 回归 |

## CPV 高风险场景

以下场景一旦涉及，必须评估并生成对应用例；具体优先级以 `priority_rules.md` 为唯一判定来源。

| 高风险场景 | 覆盖重点 |
|---|---|
| 未生效、失效、未确认或已锁定数据被继续引用或编辑 | 状态拦截、数据不被写入、提示可验证 |
| 跨站点、无菜单权限、无按钮权限、无产品数据权限或接口越权 | 入口隐藏 / 禁用、接口拦截、业务数据不泄露 |
| 审批、批准、生效、报告输出等关键动作缺少电子签名或审计追踪 | 操作人、时间、原因、前后值、版本历史 |
| 数据源失效、字段删减、空值残留仍生成有效分析结果 | 分析阻断、未分析原因、状态不误更新 |
| 替换数据源或一键分析后数据处理规则、分析状态、结果回传错误 | 配置保留 / 清空、失败原因、重新分析 |
| 报告引用跨任务、跨方案、跨站点或未确认的分析结果 | 引用拦截、结论准确性、数据一致性 |
| 批量导入、审批、导出、报告生成重复提交 | 不产生重复数据、重复任务或重复审计 |

## 导入功能专项覆盖规则

涉及 Excel / CSV / 模板文件导入时，先按 PRD 或模板字段表确认字段名、必填性、条件必填、格式、长度和引用数据规则。

### 基础规则

- 前置条件、执行步骤、预期结果中涉及模板字段时，必须使用 PRD / 模板原文，不得自行改名或概括。
- 模板字段说明类用例需逐字段悬停表头校验批注；步骤不得写 `对照 PRD 检查`，预期需写清字段批注内容，字段较多时按字段类别拆分。
- 导入成功至少拆分 `只填必填项导入成功`、`填写所有字段导入成功`；多行成功需验证成功条数、列表刷新、字段落库和引用关系一致。
- 前置条件不得写 `文件合格`、`字段完整`、`数据正常` 等模糊描述，也不得写步骤中才执行的动作。

### 异常与边界

- 文件级异常覆盖文件类型、文件大小、空文件、无数据、无表头、表头不匹配、合并单元格、加密、损坏。
- 字段级异常按模板字段逐项评估，覆盖必填为空、唯一性冲突、格式非法、长度边界、引用数据不存在、条件必填。
- 明确最大长度的字段，至少覆盖等于最大长度成功、超过最大长度失败。
- 同一字段同时包含格式规则和引用存在性规则时，格式错误和引用对象不存在必须拆成独立用例。
- 枚举 / 下拉字段若非法值可录入并保存，覆盖非法值导入失败；若 Excel 模板通过下拉或数据有效性拦截非法值，改为覆盖模板下拉选项是否完整正确，以及必填为空等实际可发生的导入校验。
- 条件字段需分别覆盖条件满足时的必填、格式、引用存在性，以及条件不满足时允许或不允许填写的取值规则。
- 同一行多个字段错误仅用于验证错误报告展示，不替代单字段独立校验。
- 若 PRD 要求整体成功或整体失败，必须验证任一行错误时整份文件拒绝导入，不产生部分导入脏数据。

### 命名与分组

- 导入失败类用例名称必须写清具体失败原因，优先使用 `具体原因，导入失败` 或 `具体原因，上传失败`。
- 导入失败类用例的 `二级分组` 统一填写 `导入失败校验`，`三级分组` 按被校验字段或对象填写。
- 相同优先级内，校验同一字段或同一类导入对象的用例必须相邻放置。

## 数据分析专项覆盖规则

新增或补充分析图 / 分析方法用例时，除 PRD 显式需求外，还需评估以下质量风险。

### 通用能力

- 读取 `knowledge_base/core_flows/data_analysis_flow.md`，确认三类入口、分析图范围和结果回传差异。
- 覆盖文件校验、插件状态、算法配置、字段配置、动态表单和开始分析前校验。
- 覆盖数据分析工作台、监控项目、相关性分析三类入口的分析图范围差异。
- 生成分析图所属分类用例时，分类名必须取自 PRD 中当前分析图所在的 `选择算法-<分类名>` 上级标题里的 `<分类名>` 原文，不得自行概括或改写。
- 变量名称、变量数量、必填性、结果区域字段必须以当前 PRD 或 UI 设计图为准，不得沿用其他分析图或参考模板中的字段名。
- 生成结果页用例前，按页面区域区分 `分析流程`、`算法配置`、`分析图表`、`过程数据`、`判异检查`、`参考线检查`、`输入结论`，不得跨区域复用字段。
- 覆盖数据源预览、替换数据源、数据处理、一键分析、未分析原因、字段删减清理和数据源失效。

### 默认覆盖项

新增或补充任一分析图 / 分析方法时，默认检查以下项；若当前 PRD / UI 明确不适用，可以不生成。

| 默认覆盖项 | 检查要求 |
|---|---|
| 变量区域基础交互 | 删除变量后回到选择变量列表；定量变量不允许添加定类变量；变量数量限制 |
| 未分析首次进入字段匹配 | 数据源字段匹配不上则不带出，匹配上则带出 |
| 数据源预览 | 从监控项目进入数据分析详情页，数据源处预览数据，显示正确 |
| PNG/JPG 导出 | 导出 PNG/JPG 图片后，图片内容仅包含分析图表，不覆盖分析流程、详细结论、过程数据等结果页其他区域 |
| 分析结果名称 | 重命名分析结果名称后监控项目列表同步更新；已分析结果在分析流程顶部显示当前分析结果名称 |
| 分析图标题 | 分析图标题不以 UI 图中的示例数据为准，默认按 `<纵轴变量>_<分析图名称>` 生成；当前分析图没有纵轴变量时，按当前分析方法已确认的变量命名规则生成 |
| 小数位数默认选项 | 小数位数不存在为空或未选择状态；生成用例时需校验默认值和下拉选项，但具体默认值、选项范围必须以当前分析方法的 PRD 为准 |
| 小数位数影响范围 | 小数位数会影响哪些结果区域或字段，必须以当前分析方法的 PRD 为准；资料未明确时，可参考模板保留该覆盖维度，但不得从参考模板或其他分析方法推断为影响过程数据等其他区域；对应用例备注必须注明：PRD 未明确，需要确认 |
| 不同数据类型 | 不同数据类型覆盖必须优先以当前分析方法的 PRD 为准；当前分析方法明确不支持的数据类型，应归入对应异常 / 边界流程处理；若资料未明确限制，则默认需要支持小数列、负数列、百分位数列、断层混合型数据列均能按当前分析方法规则处理 |

### 一键分析

新增或补充任一分析图 / 分析方法用例时，一键分析为默认强制覆盖项，不以 PRD 是否显式描述为前提。

#### 通用能力

- 读取 `knowledge_base/core_flows/one_click_analysis_flow.md`。
- 参考 `testcase_templates/modules/business_site/scheme_execution/monitoring_items_box_plot_template.md`、`testcase_templates/modules/business_site/scheme_execution/monitoring_items_paired_t_template.md` 中一键分析相关用例风格。
- 一键分析默认覆盖五类分组：`数据替换`、`数据处理`、`未分析的图参与一键分析`、`定类变量暂不支持分析`、`其他原因`。

#### 默认覆盖项

| 默认覆盖项 | 检查要求 |
|---|---|
| 未分析的图参与一键分析 | 结合当前分析图 PRD 的异常 / 边界流程处理生成，覆盖字段匹配、数据修复、样本量 / 数据量 / 变量值等条件恢复后重新参与一键分析，以及仍不满足条件时保持未分析并提示原因 |
| 多变量配置项 | 当前分析图 / 分析方法存在多个变量配置项时，按 PRD 或 UI 设计图列出每个变量的缺失、字段不匹配和修复后参与一键分析场景；变量缺失类用例必须按变量拆分，不得只生成一条泛化用例 |
| 其他原因 | 结合当前分析图 PRD 的异常 / 边界流程处理生成，例如样本量不足、数据量不一致、变量值全相同、阶段数超限等当前分析图适用的异常 |
| 手动分析与一键分析共用异常 | 同一异常若既适用于手动开始分析又适用于一键分析，需补充一键分析下的未分析状态、悬浮原因、配置保留 / 清空和修复后重新分析结果 |
