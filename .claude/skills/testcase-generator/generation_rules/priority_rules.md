# 优先级规则

## 优先级取值

测试用例优先级只能使用：

- `P0`：阻塞性，必须在发布前通过。一旦失败，会阻断 CPV 核心业务链路，影响合规、数据完整性、权限隔离或报告结论准确性。
- `P1`：重要，应在版本内完成。失败会影响关键功能完整性、常用分支流程、重要异常处理或用户完成业务工作的效率。
- `P2`：一般，有时间时覆盖。失败通常不影响核心业务闭环，但存在低频边界、兼容性、展示体验或辅助功能问题。

---

## P0 规则

满足以下**任一**条件，标记为 `P0`：

- 年度计划、任务、方案、监控项目、相关性分析、数据分析、报告编制等 **CPV 核心业务链路**的正向主流程。
- 计划生效生成任务、方案生效生成监控项目、监控项目分析确认、报告生效推进任务状态等**跨模块闭环**验证。
- 可能导致**错误报告结论、错误任务状态、错误监控结果、错误数据引用**的关键数据一致性场景。
- 影响 **GMP、数据完整性、审计追踪、电子签名、版本可追溯**的合规场景。
- 未登录、登录态过期、无站点权限、跨站点访问、无菜单/按钮/产品数据权限、接口越权后仍可执行敏感操作。
- 已批准、已生效、已归档、已锁定的数据被普通编辑、删除、覆盖或无审计记录地修改。
- 数据源失效、字段删减、变量类型不匹配、`NULL` 残留、正态性变化等情况下仍生成有效分析结果。
- 分析结果保存或回传失败，但监控项目或相关性分析状态仍被更新为已分析，或报告仍引用该结果。
- 审批、批准、生效、报告发布等关键动作未执行电子签名，或未记录审计追踪。
- 弱网或重复点击导致重复提交、重复审批、重复生成任务、重复导入、重复报告生成或重复一键分析。
- 接口失败、异步任务失败、插件不可用或服务器异常导致页面无提示、长时间卡死，且影响核心业务继续处理。

**P0 典型示例：**

| 模块 | 典型 P0 场景 |
|---|---|
| 登录与站点 | 正确账号登录授权站点成功；登录态过期后提交、审批、导出被拦截；跨站点业务数据不可访问 |
| 年度计划与任务 | 年度计划审批生效后自动生成周期性任务；未生效或已失效计划不生成正式任务 |
| 方案编制 | 方案审批生效后生成监控项目；已失效模板不可被新方案引用；已生效方案关键内容不可普通编辑覆盖 |
| 监控项目/相关性分析 | 完成分析并确认后才可进入报告；已确认结果修改需走退回、修订或重新分析流程 |
| 数据分析 | 文件校验通过后才能分析；插件不可用时阻断分析；字段删减或数据源失效时不生成有效结果 |
| 一键分析 | 替换数据源后规则清空/保留逻辑正确；全部失败或部分失败时展示未分析原因；无效字段不显示 `NULL` |
| 报告编制 | 报告引用当前任务、方案和已确认监控结果；报告生效后推进任务状态并锁定报告内容 |
| 审计与电子签名 | 审批、批准、生效动作需电子签名；关键变更记录操作人、时间、原因和前后值 |

## P1 规则

满足以下**任一**条件，标记为 `P1`：

- 重要**分支流程**，如审批退回、重新提交、修订、失效、升版、任务关闭、报告重新生成。
- 常见**异常提示**场景，如账号密码错误、无按钮权限、字段必填缺失、格式错误、数据范围越界、接口超时。
- 高频查询、筛选、排序、分页、批量选择、导入导出、状态筛选、待办入口等日常操作。
- 重要状态变化的页面展示和操作范围，如草稿、审批中、已生效、退回、已分析、已确认、关闭。
- 数据分析中的常见配置异常，如算法未选、分析项目缺失、字段配置缺失、动态表单必填项未填。
- 一键分析的部分失败、未分析标签、悬停原因、手动修复后重新分析成功等重要分支。
- 异步任务提交、查询、失败提示、重试入口、结果下载等非阻塞但影响业务效率的场景。
- 已知的**中等风险操作路径**或常见历史问题的回归验证。
- 弱网、接口超时、页面刷新、浏览器返回、多页签状态同步等影响常用流程恢复的场景。

**P1 典型示例：**

| 模块 | 典型 P1 场景 |
|---|---|
| 登录与站点 | 账号密码错误提示；用户无任何可访问站点时无法登录；当前站点在顶部列表高亮 |
| 公共管理 | 权限启停、多权限并集和按钮权限变更后刷新生效；禁用站点后入口和授权同步受限 |
| 业务站点权限 | 角色启停、多角色并集、菜单/按钮/产品数据权限变更后刷新生效 |
| 产品与基础数据 | 产品编码重复时提示；导入部分失败展示失败明细；已引用数据删除时被拦截 |
| 年度计划与任务 | 计划退回后回到可修订状态；任务日历、进度和待办同步展示 |
| 方案编制 | 审批退回保留原因；模板升版后历史方案仍引用原版本 |
| 数据分析 | 上传空文件、损坏文件、字段缺失时提示；插件未安装时提示安装并阻断后续配置 |
| 一键分析 | 部分成功部分失败时展示总数、成功数、失败数和未分析原因入口 |
| 报告编制 | 报告导出失败后展示失败原因并允许重试；报告退回后保留审批意见 |

## P2 规则

满足以下**任一**条件，标记为 `P2`：

- **低频边界场景**，如超长名称截断、极大列表展示、备注超长、特殊字符展示、日期控件边界显示。
- **文案、样式、布局类**问题，如字号、颜色、间距、对齐、空状态插图、提示文案轻微不一致。
- 不影响核心业务判断的**辅助功能**，如最近访问记录、默认展开状态、列宽调整、表格列固定、列表排序记忆。
- 纯兼容或展示适配场景，如不同浏览器、不同分辨率、缩放比例、横屏、打印预览样式。
- 不影响任务、方案、监控项目、分析结果和报告状态的体验优化，如加载动画、骨架屏、按钮 hover 状态。
- 低风险操作路径、低频配置项、说明性内容或帮助入口验证。

**P2 典型示例：**

| 模块 | 典型 P2 场景 |
|---|---|
| 首页/工作台 | 待办为空时展示空状态；看板卡片在小分辨率下自适应 |
| 公共管理 | 站点 Logo 为空时展示默认图；用户列表列宽拖拽后展示正常 |
| 产品与基础数据 | 产品名称超长时省略展示；备注包含特殊字符时页面显示不乱 |
| 任务管理 | 日历视图跨月展示样式；筛选条件清空按钮显示正常 |
| 数据分析 | 分析结果图表标题超长时换行或省略；加载中展示骨架屏 |
| 报告编制 | 报告预览分页样式；导出按钮 hover、禁用和 loading 状态展示 |

## 优先级判断原则

无法直接判断时，按以下顺序评估：

1. **业务与合规影响**：是否影响 CPV 闭环、报告结论、数据完整性、审计追踪或电子签名？是 → P0。
2. **数据影响范围**：是否会产生错误数据、错误状态、跨站点数据泄露、错误引用或不可追溯修改？是 → P0。
3. **流程阻塞程度**：是否阻断计划、任务、方案、监控项目、分析或报告继续流转？核心阻断 → P0，分支阻断 → P1。
4. **用户触达频率**：是否为高频操作路径？高频主流程偏 P0，高频分支偏 P1，低频边界偏 P2。
5. **是否有补救机制**：失败后是否可自助恢复或重试？无补救或恢复成本高 → 提高一级。
6. **风险程度**：高风险路径 → P0，中等风险 → P1，低风险体验类 → P2。

## 优先级分布建议

一个模块的完整用例集，优先级分布参考如下：

| 优先级 | 建议占比 | 说明 |
|---|---|---|
| P0 | 20%–35% | 聚焦 CPV 核心链路、合规风险、数据完整性、权限隔离和报告结论准确性，不应过多 |
| P1 | 40%–55% | 覆盖重要分支、常见异常、高频操作和可恢复问题 |
| P2 | 15%–30% | 覆盖低频边界、兼容、样式、展示和辅助体验 |

若 P0 用例超过 50%，说明优先级标注过于保守，应重新评估；若核心链路、合规追溯或跨站点权限场景未标为 P0，应优先修正。
