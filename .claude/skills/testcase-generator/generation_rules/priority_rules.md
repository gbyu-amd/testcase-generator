# 优先级规则

## 优先级取值

测试用例优先级只能使用：

- `P0`
- `P1`
- `P2`

## P0 规则

满足以下任一条件，应标记为 `P0`：

- 登录、购物车、结算、提交订单等核心链路。
- 订单金额、库存、优惠、地址等关键数据一致性。
- 可能导致资损、重复订单、错误扣库存的问题。
- 登录态过期后仍可执行敏感操作。
- 历史高严重缺陷回归。

## P1 规则

满足以下任一条件，可标记为 `P1`：

- 重要分支流程。
- 常见异常提示。
- 筛选、排序、规格选择等高频功能。
- 商品失效、库存不足、价格变化等重要状态。
- 非核心但影响用户购买决策的功能。

## P2 规则

满足以下任一条件，可标记为 `P2`：

- 低频边界场景。
- 文案、样式、布局类问题。
- 搜索历史、辅助提示、兼容性补充。
- 不影响核心交易链路的体验优化。

## 判断原则

无法判断时，优先考虑业务影响范围、用户频率和是否可能导致错误交易数据。
