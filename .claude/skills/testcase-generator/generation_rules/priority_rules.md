# 优先级规则

## 优先级取值

测试用例优先级只能使用：

- `P0`：阻塞性，必须在发布前通过。只用于 CPV 核心业务正向主链路、跨模块闭环成功和核心结论准确性。
- `P1`：重要，应在版本内完成。用于重要分支、权限访问控制、常用操作、关键配置正常维护和中等风险回归。
- `P2`：一般，有时间时覆盖。用于字段校验、异常提示、边界值、展示体验、兼容适配、弱网和防重复类稳定性。

---

## P0 规则

满足以下**任一**条件，标记为 `P0`：

- 年度计划、任务、方案、监控项目、相关性分析、数据分析、报告编制等 CPV 核心业务链路的正向主流程。
- 计划生效生成任务、方案生效生成监控项目、监控项目分析确认、报告生效推进任务状态等跨模块闭环的成功验证。
- 数据分析、监控项目、报告编制等主链路成功后，核心计算结果、分析结论、报告引用结果必须正确的场景。
- 一键分析、报告插入分析结果等高频核心入口的正常成功链路。
- 明确属于合规发布阻断项的审计追踪核心验证：PRD 指定必须审计的关键操作成功后，审计记录新增、详情基本信息和内容详情字段完整且准确。

**P0 典型示例：**

| 模块 | 典型 P0 场景 |
|---|---|
| 年度计划与任务 | 年度计划审批生效后自动生成周期性任务；未生效或已失效计划不生成正式任务 |
| 方案编制 | 方案审批生效后生成监控项目；已失效模板不可被新方案引用；已生效方案关键内容不可普通编辑覆盖 |
| 监控项目/相关性分析 | 完成分析并确认后才可进入报告；已确认结果修改需走退回、修订或重新分析流程 |
| 数据分析 | 分析成功主链路，分析状态、分析结果、核心计算结果正确 |
| 一键分析 | 正常一键分析成功；结果数据和分析状态更新正确 |
| 报告编制 | 报告引用当前任务、方案和已确认监控结果；报告生效后推进任务状态并锁定报告内容 |
| 审计追踪 / 合规追溯 | PRD 明确要求审计的关键操作产生审计记录，且详情基本信息和内容详情字段准确 |

## P1 规则

满足以下**任一**条件，标记为 `P1`：

- 重要分支流程，如审批退回、重新提交、修订、失效、升版、任务关闭、报告重新生成。
- 影响用户继续完成常用业务操作的重要异常处理，如账号密码错误、无按钮权限或关键数据不可用。
- 未登录、登录态过期、无站点权限、跨站点访问、无菜单/按钮/产品数据权限、接口越权后仍可执行敏感操作。
- 高频查询、筛选、排序、分页、批量选择、导入导出、状态筛选、待办入口等日常操作。
- 重要状态变化的页面展示和操作范围，如草稿、审批中、已生效、退回、已分析、已确认、关闭。
- 数据分析中阻断常用分析流程的系统级异常或关键数据不可用；单一分析方法的数据异常拦截、边界处理、提示展示按 `P2`。
- 分析图变量配置的正常添加、删除、修改、替换、带出或回显。变量通常是分析图必选配置，影响后续是否能生成正确分析结果；变量配置过程中的异常提示或限制校验按 `P2`。
- 一键分析的异常和重要分支，如字段删减、样本不足、数据量不一致、变量类型不支持、部分失败、未分析标签、悬停原因、替换数据源 / 数据处理后的计算分支、手动修复后重新分析成功。
- 已知的中等风险操作路径或常见历史问题的回归验证。

**P1 典型示例：**

| 模块 | 典型 P1 场景 |
|---|---|
| 登录与站点 | 账号密码错误提示；未登录、登录态过期、无站点权限、跨站点访问被拦截；用户无任何可访问站点时无法登录 |
| 公共管理 | 权限启停、多权限并集和按钮权限变更后刷新生效；禁用站点后入口和授权同步受限 |
| 业务站点权限 | 角色启停、多角色并集、菜单/按钮/产品数据权限变更后刷新生效 |
| 产品与基础数据 | 已引用数据删除时被拦截；批量操作失败后可恢复或重试 |
| 年度计划与任务 | 计划退回后回到可修订状态；任务日历、进度和待办同步展示 |
| 方案编制 | 审批退回保留原因；模板升版后历史方案仍引用原版本 |
| 数据分析 | 关键数据不可用时阻断常用流程；分析图变量正常添加、删除、修改或回显正确 |
| 一键分析 | 字段删减、样本不足、数据量不一致、定类变量等异常场景正确保持未分析；未分析原因入口可查看 |
| 报告编制 | 报告导出失败后展示失败原因并允许重试；报告退回后保留审批意见 |

## P2 规则

满足以下**任一**条件，标记为 `P2`：

- 字段、文件和配置项校验场景，如必填校验、字符长度校验、字段格式校验、变量类型限制、默认值、禁用态、范围提示、非法文件格式或内容、表头不匹配、文件大小限制等；分析图变量配置过程中的类型不支持、重复添加、超过数量限制、必填提示等异常校验也按 `P2`。
- 数据分析异常拦截和边界处理，如样本量不足、样本数据量不一致、变量值全相同、同一数据列导致结果显示特殊值、缺失值静默排除后继续计算等单一分析方法异常场景。
- 低频边界场景，如超长名称截断、极大列表展示、备注超长、特殊字符展示、日期控件边界显示。
- 文案、样式、布局类问题，如字号、颜色、间距、对齐、空状态插图、提示文案轻微不一致。
- 模板说明和提示文案场景，如模板字段批注、下拉选项说明、错误提示文案、帮助说明等。
- 结果提示语校验场景，如一键分析全部成功、部分失败、全部失败后的提示文案、总数、成功数、失败数展示。
- 弱网、重复点击和防重复提交类稳定性场景，如重复点击导致重复提交、重复审批、重复生成任务、重复导入、重复报告生成或重复一键分析。
- 不影响核心业务判断的**辅助功能**，如最近访问记录、默认展开状态、列宽调整、表格列固定、列表排序记忆。
- 纯兼容或展示适配场景，如不同浏览器、不同分辨率、缩放比例、横屏、打印预览样式。
- 不影响任务、方案、监控项目、分析结果和报告状态的体验优化，如加载动画、骨架屏、按钮 hover 状态。
- 低风险操作路径、低频配置项、说明性内容或帮助入口验证。

**P2 典型示例：**

| 模块 | 典型 P2 场景 |
|---|---|
| 首页/工作台 | 待办为空时展示空状态；看板卡片在小分辨率下自适应 |
| 公共管理 | 站点 Logo 为空时展示默认图；用户列表列宽拖拽后展示正常 |
| 产品与基础数据 | 必填、长度、格式、唯一性冲突和文件大小校验提示正确；产品名称超长时省略展示 |
| 导入导出 | 非法文件格式或内容、表头不匹配、模板字段说明、下拉选项和提示文案展示正确 |
| 任务管理 | 日历视图跨月展示样式；筛选条件清空按钮显示正常 |
| 数据分析 | 配置项必填、范围和默认值校验；变量配置异常提示；单一分析方法异常拦截；分析结果图表标题超长时换行或省略 |
| 一键分析 | 全部成功、部分失败、全部失败后的提示文案和数量展示 |
| 报告编制 | 报告预览分页样式；导出按钮 hover、禁用和 loading 状态展示 |

## 优先级判断原则

无法直接判断时，按以下顺序评估：

1. 是否为 CPV 核心正向主链路或跨模块闭环成功验证？是 → `P0`。
2. 是否直接验证分析结果、监控结果或报告引用结果的核心正确性？是 → `P0`。
3. 是否为 PRD 明确要求的合规审计核心记录或详情字段准确性验证，且会影响发布合规性？是 → `P0`。
4. 是否为权限访问控制、重要分支流程、常用操作或正常关键配置维护？是 → `P1`。
5. 是否为分析图变量的正常添加、删除、修改、替换、带出或回显？是 → `P1`。
6. 是否为字段校验、配置异常、提示文案、边界值、兼容展示、弱网或重复点击稳定性？是 → `P2`。
7. 同一场景同时满足多个等级时，按最高业务影响取值；但本文件已明确归入 `P2` 的异常提示、边界和展示类场景不再上提。

## 优先级分布建议

一个模块的完整用例集，优先级分布参考如下：

| 优先级 | 建议占比 | 说明 |
|---|---|---|
| P0 | 15%–20% | 聚焦核心主链路、跨模块闭环、结论准确性和合规发布阻断项，不应过多 |
| P1 | 35%–50% | 覆盖重要分支、权限访问控制、常用操作、正常关键配置和中等风险回归 |
| P2 | 30%–45% | 覆盖字段校验、异常提示、边界、兼容、样式、展示和稳定性 |

若 P0 用例明显超过 20%，说明优先级标注可能过于保守，应重新评估；若核心正向主链路、跨模块闭环或核心结果正确性场景未标为 P0，应优先修正。
