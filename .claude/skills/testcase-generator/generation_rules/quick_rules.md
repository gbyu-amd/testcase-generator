# 测试用例生成快速规则

本文件用于单个 `REQ-CPV-*` 需求的快速生成场景。优先读取本文件；只有需求涉及复杂覆盖、追加合并、难度争议或校验失败时，再读取对应详细规则文件。

## 标准输出字段

输出 Markdown 表格必须使用以下 16 列，顺序不可变：

| 一级分组 | 二级分组 | 三级分组 | 用例名称 | 优先级 | 创建人 | 用例描述 | 前置条件 | 用例步骤 | 预期结果 | 备注 | 用例标签 | 是否自动化 | 关联接口 | 用例测试类 | 关联项目 |
|---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|---|

## 字段填写

- `一级分组`：填写 `公共管理站点` 或 `业务站点`。
- `二级分组`：填写一级菜单或业务域，例如 `年度计划与任务`、`方案编制`、`报告编制`、`权限管理`。
- `三级分组`：填写二级菜单、页面、操作类型或子流程。
- `用例名称`：描述具体验证目标，避免重复和空泛。
- `优先级`：只能填写 `P0`、`P1`、`P2`。
- `创建人`：默认填写 `AI`。
- `用例描述`：建议使用 `正例`、`反例`、`边界`、`权限`、`兼容`、`回归`、`联动`。
- `前置条件`：必须明确角色、站点、权限、测试数据或业务状态。
- `用例步骤`：必须可执行，使用编号步骤。
- `预期结果`：必须可验证，包含页面反馈、数据状态或业务状态。
- `备注`：必须写明来源，例如 `来源：需求文档`、`来源：UI设计图`、`来源：priority_rules.md-P0`。
- `用例标签`：只保留难度等级，取值为 `简单`、`一般` 或 `困难`。
- `是否自动化`、`关联接口`、`用例测试类`、`关联项目`：新生成用例必须留空。

## 覆盖要求

- 每条验收标准至少对应一条测试用例。
- 核心链路必须覆盖正向、异常、权限、状态流转、数据一致性和审计追踪要求。
- 涉及 UI 设计图时，必须覆盖控件展示、按钮状态、弹窗、提示、空状态、禁用状态和页面跳转。
- 涉及导入、导出、异步任务、审批、生效、电子签名、一键分析和报告时，必须补充失败、重试、权限和状态变化场景。

## 优先级速判

- `P0`：主流程、审批/生效、数据完整性、权限隔离、审计追踪、报告结论、核心阻断类异常。
- `P1`：常用分支、字段校验、状态变化、查询筛选、导入导出、配置生效。
- `P2`：兼容性、展示细节、低频操作、提示文案、辅助交互。

## 难度速判

按 `generation_rules/difficulty_level_rules.md` 的脚本规则生成难度标签。简化理解：

- `简单`：单页面、单动作、少量步骤，例如新增、编辑、删除、查询、导出、下拉选项。
- `一般`：组合查询、批量删除、批量导出、跨页面或详情校验。
- `困难`：多角色、审批/工作流、电子签名、跨环境、第三方工具、数据完整性、一键分析、报告审批/生效。

## 输出与校验

生成完成后只校验当前输出文件：

```bash
python scripts/validate_cases.py --source outputs/origin_exports/<site_type>/<module_name>_testcases.md
```

校验通过后只导出当前输出文件：

```bash
python scripts/export_testcases.py --source outputs/origin_exports/<site_type>/<module_name>_testcases.md
```

不带 `--source` 会扫描全量输出目录，单需求生成时不要使用。
