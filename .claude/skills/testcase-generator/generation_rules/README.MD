# generation_rules 读取说明

`generation_rules` 用于维护测试用例生成规范，负责回答“测试用例应该怎么写”。生成测试用例时，本目录规则按需读取，不会默认全量读取。

## 读取原则

- `quick_rules.md` 是快速生成规则，通常优先读取，用于确定标准表头、字段填写、优先级、难度和基础覆盖要求。
- 其他详细规则文件按需读取。复杂场景、覆盖维度不足、优先级或难度存在争议、追加合并、校验失败时，再读取对应详细规则。
- 除非信息不足、校验失败或用户明确要求，不主动读取全部 `generation_rules` 文件，避免无关规则增加上下文噪音。

## 使用方式

- 先根据当前需求判断需要哪类生成规则，例如字段填写、覆盖维度、优先级、难度或追加合并。
- 优先使用 `quick_rules.md` 完成常规生成；只有常规规则不足时，再读取对应详细规则文件。
- 规则文件只约束测试用例写法，不承载具体业务流程事实。

## 当前规则文件

- `quick_rules.md`：快速生成规则，包含标准表头、字段填写、优先级和难度速判。
- `testcase_writing_guidelines.md`：用例编写规范，包含字段写法、步骤和预期结果质量要求。
- `coverage_dimension_rules.md`：覆盖维度规则，包含正向、异常、边界、权限、状态、数据一致性、合规追溯等维度。
- `priority_rules.md`：优先级判定规则。
- `difficulty_level_rules.md`：用例难度等级判定规则。
- `case_append_rules.md`：已有用例追加、覆盖、另存和去重规则。
- `testcase_id_rules.md`：旧版编号规则说明；当前标准输出表头不再使用独立用例编号字段。
