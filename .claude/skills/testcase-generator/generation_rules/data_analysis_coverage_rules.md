# 数据分析专项覆盖规则

## 使用原则

新增或补充分析方法、数据分析详情页、分析结果、替换数据源或一键分析用例时，必须先读取本文件。

- 除 PRD 显式需求外，还需评估本文件中的默认质量风险。
- 当前 PRD / UI 明确不适用的默认覆盖项可以不生成，但需在覆盖率说明中记录原因；适用的核心链路、异常原因和结果正确性场景不得遗漏。
- 本文件只定义数据分析专项覆盖场景；优先级仍按 `priority_rules.md` 判定，难度仍按 `difficulty_level_rules.md` 判定。

---

## 通用能力

- 读取 `knowledge_base/core_flows/data_analysis_flow.md`，确认三类入口、分析方法范围和结果处理差异。
- 按当前入口和分析方法覆盖文件校验、插件状态、算法配置、字段配置、动态表单和开始分析前校验。
- 覆盖数据分析工作台、监控项目、相关性分析汇总三类入口的分析方法范围差异。
- 生成分析方法所属分类用例时，分类名必须取自 PRD 中当前分析方法所在的 `选择算法-<分类名>` 上级标题里的 `<分类名>` 原文，不得自行概括或改写。
- 变量名称、变量数量、必填性、结果区域字段必须以当前 PRD 或 UI 设计图为准，不得沿用其他分析方法或参考模板中的字段名。
- 生成结果页用例前，按当前 PRD / UI 实际存在的页面区域区分 `分析流程`、`算法配置`、`分析图表`、`过程数据`、`判异检查`、`参考线检查`、`输入结论`，不得跨区域复用字段或强行补不存在的区域。
- 覆盖数据源预览、替换数据源、数据处理、一键分析、未分析原因、失效字段清理和数据源失效。

## 分析方法默认覆盖项

新增或补充任一分析方法时，默认检查以下项；若当前 PRD / UI 明确不适用，可以不生成。

| 默认覆盖项 | 检查要求 |
|---|---|
| 变量区域基础交互 | 按当前分析方法支持的变量角色、变量类型和变量数量限制生成；不支持定类变量时需校验拦截，支持定类变量时不得生成“定类变量不支持”类用例 |
| 未分析项字段匹配 | 未分析项重新进入配置时，数据源字段匹配不上则不自动带出原字段，匹配上则自动带出原字段 |
| 数据源预览 | 按当前入口进入数据分析详情页后，数据源预览入口、预览内容、字段和数据行显示正确，不固定为监控项目入口 |
| PNG/JPG 导出 | 当前分析方法支持导出图片时，导出内容应仅包含分析图表，不覆盖分析流程、详细结论、过程数据等结果页其他区域 |
| 分析结果名称 | 重命名分析结果名称后，应同步到当前入口对应列表；数据分析工作台入口应同步到已保存分析结果记录，监控项目和相关性分析汇总入口推送至 CPV 后应同步到对应业务列表的分析结果名称字段 |
| 分析结果标题 | 分析结果标题不以 UI 图中的示例数据为准；PRD / UI 未明确命名规则时，默认按 `<当前分析方法的主分析变量>_<分析方法名称>` 生成。主分析变量按当前方法实际变量角色判断，可能是纵轴变量、图形变量、样本1、样本2、分析列等；横轴变量、频率变量、分组变量、类别变量等辅助维度不作为主分析变量；存在多个主分析变量时，变量名之间用顿号连接后再拼接分析方法名称 |
| 小数位数默认选项 | 当前分析方法存在小数位数配置时，需校验默认值和下拉选项；具体默认值和选项范围必须以当前分析方法的 PRD / UI 为准 |
| 小数位数影响范围 | 小数位数会影响哪些结果区域或字段，必须以当前分析方法的 PRD / UI 为准；资料未明确时，可参考模板保留该覆盖维度，但不得从参考模板或其他分析方法推断为影响过程数据等其他区域；对应用例备注必须注明：PRD 未明确，需要确认 |
| 不同数据类型 | 不同数据类型覆盖必须优先以当前分析方法的 PRD / UI、算法限制和字段类型为准；资料未明确限制时，将小数列、负数列、百分位数列、断层或混合型数据列作为风险覆盖维度评估，不得直接断言当前方法全部支持 |

## 一键分析

新增或补充支持数据替换或数据处理，并会对当前已生成分析结果重新分析的分析方法用例时，一键分析为默认覆盖项，不以 PRD 是否显式描述为前提。若当前 PRD / UI 明确没有一键分析入口或不支持重新分析，应在覆盖率说明中记录不适用原因，不强行生成一键分析用例。

### 通用能力

- 读取 `knowledge_base/core_flows/one_click_analysis_flow.md`。
- 参考 `testcase_templates/modules/business_site/scheme_execution/monitoring_items_box_plot_template.md`、`testcase_templates/modules/business_site/scheme_execution/monitoring_items_paired_t_template.md` 中一键分析相关用例风格。
- 一键分析默认先覆盖触发方式：`数据替换`、`数据处理`。
- 已存在未分析项时，还需覆盖未分析项在数据替换或数据处理后重新参与一键分析的场景。
- 一键分析失败原因按当前分析方法的 PRD / UI、变量类型和算法限制筛选，不固定套用所有分组。

| 优先级 | 候选失败原因 | 适用条件 |
|---|---|---|
| 1 | 定类变量暂不支持分析 | 仅适用于当前分析方法不支持定类变量的场景；若当前分析方法支持定类变量，不得生成该类用例。 |
| 2 | 字段删减 | 当前分析方法依赖字段在替换或处理后的数据中不存在。 |
| 3 | 正态性变化 | 仅适用于过程能力分析（正态）和过程能力分析（非正态）。 |
| 4 | 阶段内容不一致 | 仅适用于存在阶段配置或参考线依赖阶段的分析方法。 |
| 5 | 其他原因 | 当前分析方法 PRD / UI 明确的样本量、数据量、变量值、阶段数等异常。 |

### 默认覆盖项

| 默认覆盖项 | 检查要求 |
|---|---|
| 未分析项参与一键分析 | 结合当前分析方法 PRD / UI 的异常 / 边界流程处理生成，覆盖字段匹配、数据修复、样本量 / 数据量 / 变量值等条件恢复后重新参与一键分析，以及仍不满足条件时保持未分析并提示原因 |
| 多变量配置项 | 当前分析方法存在多个变量配置项时，按 PRD 或 UI 设计图列出每个变量的缺失、字段不匹配和修复后参与一键分析场景；变量缺失类用例必须按变量拆分，不得只生成一条泛化用例 |
| 适用失败原因 | 从上方失败原因优先级表中筛选当前分析方法适用的原因，覆盖未分析状态、悬停原因、配置保留 / 清空、修复后重新分析和仍失败时的处理 |
| 手动分析与一键分析共用异常 | 同一异常若既适用于手动开始分析又适用于一键分析，需分别覆盖手动分析拦截和一键分析下的未分析状态，不得只生成其中一种入口 |
