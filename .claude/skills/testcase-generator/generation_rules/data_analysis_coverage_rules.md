# 数据分析专项覆盖规则

## 使用原则

新增或补充分析方法、数据分析详情页、分析结果、替换数据源或一键分析用例时，必须先读取本文件。

- 除 PRD 显式需求外，还需评估本文件中的默认质量风险。
- 当前 PRD / UI 明确不适用的默认覆盖项可以不生成，但不得删除核心链路、异常原因或结果正确性场景。
- 本文件只定义数据分析专项覆盖场景；优先级仍按 `priority_rules.md` 判定，难度仍按 `difficulty_level_rules.md` 判定。

---

## 通用能力

- 读取 `knowledge_base/core_flows/data_analysis_flow.md`，确认三类入口、分析方法范围和结果处理差异。
- 覆盖文件校验、插件状态、算法配置、字段配置、动态表单和开始分析前校验。
- 覆盖数据分析工作台、监控项目、相关性分析三类入口的分析方法范围差异。
- 生成分析方法所属分类用例时，分类名必须取自 PRD 中当前分析方法所在的 `选择算法-<分类名>` 上级标题里的 `<分类名>` 原文，不得自行概括或改写。
- 变量名称、变量数量、必填性、结果区域字段必须以当前 PRD 或 UI 设计图为准，不得沿用其他分析方法或参考模板中的字段名。
- 生成结果页用例前，按页面区域区分 `分析流程`、`算法配置`、`分析图表`、`过程数据`、`判异检查`、`参考线检查`、`输入结论`，不得跨区域复用字段。
- 覆盖数据源预览、替换数据源、数据处理、一键分析、未分析原因、字段删减清理和数据源失效。

## 分析方法默认覆盖项

新增或补充任一分析方法时，默认检查以下项；若当前 PRD / UI 明确不适用，可以不生成。

| 默认覆盖项 | 检查要求 |
|---|---|
| 变量区域基础交互 | 删除变量后回到选择变量列表；定量变量不允许添加定类变量；变量数量限制 |
| 未分析首次进入字段匹配 | 数据源字段匹配不上则不带出，匹配上则带出 |
| 数据源预览 | 从监控项目进入数据分析详情页，数据源处预览数据，显示正确 |
| PNG/JPG 导出 | 导出 PNG/JPG 图片后，图片内容仅包含分析图表，不覆盖分析流程、详细结论、过程数据等结果页其他区域 |
| 分析结果名称 | 重命名分析结果名称后监控项目列表同步更新；已分析结果在分析流程顶部显示当前分析结果名称 |
| 分析结果标题 | 分析结果标题不以 UI 图中的示例数据为准，默认按 `<纵轴变量>_<分析方法名称>` 生成；当前分析方法没有纵轴变量时，按当前分析方法已确认的变量命名规则生成 |
| 小数位数默认选项 | 小数位数不存在为空或未选择状态；生成用例时需校验默认值和下拉选项，但具体默认值、选项范围必须以当前分析方法的 PRD 为准 |
| 小数位数影响范围 | 小数位数会影响哪些结果区域或字段，必须以当前分析方法的 PRD 为准；资料未明确时，可参考模板保留该覆盖维度，但不得从参考模板或其他分析方法推断为影响过程数据等其他区域；对应用例备注必须注明：PRD 未明确，需要确认 |
| 不同数据类型 | 不同数据类型覆盖必须优先以当前分析方法的 PRD 为准；当前分析方法明确不支持的数据类型，应归入对应异常 / 边界流程处理；若资料未明确限制，则默认需要支持小数列、负数列、百分位数列、断层混合型数据列均能按当前分析方法规则处理 |

## 一键分析

新增或补充任一分析方法用例时，一键分析为默认强制覆盖项，不以 PRD 是否显式描述为前提。

### 通用能力

- 读取 `knowledge_base/core_flows/one_click_analysis_flow.md`。
- 参考 `testcase_templates/modules/business_site/scheme_execution/monitoring_items_box_plot_template.md`、`testcase_templates/modules/business_site/scheme_execution/monitoring_items_paired_t_template.md` 中一键分析相关用例风格。
- 一键分析默认覆盖五类分组：`数据替换`、`数据处理`、`未分析的方法参与一键分析`、`定类变量暂不支持分析`、`其他原因`。

### 默认覆盖项

| 默认覆盖项 | 检查要求 |
|---|---|
| 未分析的方法参与一键分析 | 结合当前分析方法 PRD 的异常 / 边界流程处理生成，覆盖字段匹配、数据修复、样本量 / 数据量 / 变量值等条件恢复后重新参与一键分析，以及仍不满足条件时保持未分析并提示原因 |
| 多变量配置项 | 当前分析方法存在多个变量配置项时，按 PRD 或 UI 设计图列出每个变量的缺失、字段不匹配和修复后参与一键分析场景；变量缺失类用例必须按变量拆分，不得只生成一条泛化用例 |
| 其他原因 | 结合当前分析方法 PRD 的异常 / 边界流程处理生成，例如样本量不足、数据量不一致、变量值全相同、阶段数超限等当前分析方法适用的异常 |
| 手动分析与一键分析共用异常 | 同一异常若既适用于手动开始分析又适用于一键分析，需补充一键分析下的未分析状态、悬浮原因、配置保留 / 清空和修复后重新分析结果 |
