# 用例难度等级定义规则

## 适用范围

本规则用于生成、补充、评审测试用例时判断用例难度。当前标准输出模板没有独立难度字段，因此必须将难度等级写入 `用例标签`，且该字段只保留难度等级，例如 `简单`、`一般` 或 `困难`。

难度等级只能使用：

- `简单`
- `一般`
- `困难`

写入 `用例标签` 时，只写入最新判定出的难度等级；若已有旧难度标签或业务标签，必须先移除后再写入最新难度值。

---

## 核心判定维度

判定用例难度时，必须同时评估以下 3 个维度：

| 维度 | 判断重点 |
|---|---|
| 前置准备 | 是否需要构造测试数据、数据关系、审批流程、分析配置、跨环境或第三方依赖 |
| 执行步骤 | 从进入目标功能页面后开始计算的有效操作步骤数量，以及交互路径是否复杂 |
| 验证结果 | 需要验证的数据量、校验点数量、跨模块联动、多人协作或多环境一致性 |

---

## 关键字判定流程

当前脚本按字段可信度和关键字类型分层判定难度：

1. 用例名称命中高可信关键字，直接按关键字等级判定。
2. 用例步骤或预期结果命中高可信关键字，直接按关键字等级判定。
3. 用例名称或用例步骤命中组合关键字，按组合规则判定。
4. 用例名称、用例步骤或预期结果命中简单关键字，判定为 `简单`。
5. 以上都未命中时，再按前置条件、执行步骤和验证结果做兜底判断。

关键字匹配不使用一级分组、二级分组、三级分组、前置条件、用例描述和用例标签，避免分组、背景描述或已写入标签导致误判。

---

## 简单

### 判定标准

满足以下特征时，标记为 `简单`：

- 前置准备：无需构造特殊数据，可直接复用现有数据。
- 执行步骤：通常为 1-2 步，操作路径短，交互简单。
- 验证结果：验证点简单，通常为单一操作结果，如 1 条数据、1 个提示、1 个页面状态。

### 典型场景

- 列表页常规操作：翻页、排序、筛选重置、刷新、启用/停用、重置。
- 单项操作：单页面提交、新增、增加、添加、编辑、删除、搜索、查询。
- 单字段校验：必填、长度、格式提示、下拉多选、提示语校验。
- 单文件导出：导出单个审计追踪详情后查看导出结果。

### 简单关键字

用例名称、用例步骤或预期结果中出现以下任一关键字，且未命中高可信关键字和组合关键字时，判定为 `简单`。

`列表页`、`翻页`、`排序`、`筛选重置`、`刷新`、`启用/停用`、`启用`、`停用`、`重置`、`单项操作`、`单页面提交`、`新增`、`增加`、`添加`、`编辑`、`删除`、`搜索`、`查询`、`单字段校验`、`必填`、`长度`、`格式提示`、`下拉多选`、`下拉选项`、`提示语校验`、`导出`、`单文件导出`

### 示例

| 场景 | 难度 |
|---|---|
| 列表翻页 | 简单 |
| 输入权限名称查询审计追踪记录 | 简单 |
| 点击导出单条审计追踪详情并确认文件生成 | 简单 |

---

## 一般

### 判定标准

命中一般高可信关键字、一般组合关键字，或关键字都未命中但兜底判断为中等复杂度时，标记为 `一般`。一般用例通常具备以下特征：

- 前置准备：需要构造 1-2 类特定测试数据，或环境准备有少量特殊要求。
- 执行步骤：通常为 3-5 步，存在一定页面交互，但整体路径不复杂。
- 验证结果：校验逻辑不复杂，通常包含 1-2 个关键校验点，重点验证数据正确性或配置生效。

### 典型场景

- 配置应用正确性：完成配置后经过流程流转，验证分析结果或展示结果正确。
- 数据批量导出：原始数据存在，选择多个分类数据表后批量导出，并查看结果。
- 构造特定测试数据：创建单选类数据表、配置字典映射、执行数据分析并保存分析结果。
- 报告生成：选择模板后生成报告，并对比报表中插入的分析结果。
- 权限或审计中的组合查询：按模块、操作类型、受影响对象组合查询并验证结果范围。

### 一般高可信关键字

用例名称命中以下任一关键字时，判定为 `一般`：

`导入`、`批量导出`、`跨页面`、`跨模块`、`审计追踪详情`、`批量删除`、`组合搜索`、`组合查询`

用例步骤或预期结果命中以下任一关键字时，判定为 `一般`：

`导入`、`跨页面`、`跨模块`

### 一般组合关键字

用例名称或用例步骤同时命中以下组合时，判定为 `一般`：

| 组合 | 难度 |
|---|---|
| `批量` + `删除` | 一般 |
| `批量` + `导出` | 一般 |
| `组合` + `搜索` | 一般 |
| `组合` + `查询` | 一般 |
| `审计追踪` + `详情` | 一般 |
| `查看` + `详情` | 一般 |
| `报告` + `生成` | 一般 |
| `报告` + `导出` | 一般 |

### 示例

| 场景 | 难度 |
|---|---|
| 配置权限后验证审计追踪内容详情 | 一般 |
| 按模块和操作类型组合搜索审计追踪记录 | 一般 |
| 选择模板生成报告并核对插入的分析结果 | 一般 |

## 困难

### 判定标准

满足以下特征时，标记为 `困难`：

- 前置准备：需要构造多类关联数据，或前置流程超过 3 个独立准备操作。
- 执行步骤：通常大于 5 步，操作路径长，操作复杂，可能涉及回顾对象、任务流转、导入、审批、转审、加签、分析、报告、检查等连续操作。
- 验证结果：验证逻辑复杂，需要跨 3 个及以上模块联动校验，或涉及多人角色校验、多环境协同、第三方工具对比、数据完整性和一致性验证。

### 困难高可信关键字

用例名称、用例步骤或预期结果命中以下任一关键字时，应判定为 `困难`：

`一键分析`、`一键更新`、`数据迁移`、`跨环境`、`第三方`、`minitab`、`多角色`、`多人`、`会签`、`电子签名`、`审批`、`审核`、`转审`、`加签`、`批准`、`工作流`、`数据完整性`、`一致性验证`、`分析结果数据正确性校验`、`报告审批`、`报告批准`、`报告生效`、`报告回推`、`报告链路`

### 困难组合关键字

用例名称或用例步骤同时命中以下组合时，判定为 `困难`：

| 组合 | 难度 |
|---|---|
| `报告` + `审批` | 困难 |
| `报告` + `审核` | 困难 |
| `报告` + `批准` | 困难 |
| `报告` + `生效` | 困难 |
| `报告` + `回推` | 困难 |
| `跨模块` + `校验` | 困难 |

### 典型场景

- 一键分析：准备数据表数据、补充检验标准、导入数据并提交、审核通过、配置并执行分析、检查分析数据、保存分析模板，再检查分析模板提交结果。
- 一键更新：回顾对象配置、数据表导入、分析执行、生成结果、生效分析模板、报告生成，并检查任务列表。
- 依赖第三方工具：使用外部工具或算法结果与系统分析结果进行对比校验。
- 数据迁移：A 环境导出数据，验证完整性后导入 B 环境，并校验数量和一致性。
- 多角色协同：数据创建提交、电子签名配置、工作流会签、转审、加签、多人会签审核、审计追踪记录验证。
- 报告链路：报告起草、审核、批准、生效、导出、任务回推和审计追踪记录连续验证。

### 示例

| 场景 | 难度 |
|---|---|
| 一键分析全流程并保存分析模板 | 困难 |
| A 环境导出数据后导入 B 环境并校验一致性 | 困难 |
| 多人会签审批后验证电子签名和审计追踪记录 | 困难 |

## 取值规则

### 取高原则

当同一条用例同时命中多个规则时，必须按以下顺序取值：用例名称高可信关键字 > 用例步骤/预期结果高可信关键字 > 组合关键字 > 简单关键字 > 兜底判断。

示例：

- 用例名称包含 `批量删除`，即使步骤中包含 `导出`，也先按标题高可信关键字判定为 `一般`。
- 用例步骤包含 `报告审批`，即使标题中没有复杂词，也按步骤高可信关键字判定为 `困难`。
- 用例名称包含 `组合搜索`，同时命中一般组合和简单关键字 `搜索`，按组合规则判定为 `一般`。
- 用例仅包含 `翻页`、`查询`、`下拉选项` 等简单关键字，且未命中高可信和组合规则，则难度为 `简单`。

### 步骤数统计规则

执行步骤数从进入目标功能页面后开始计算，登录操作不计入步骤数。

三类关键字都未命中时，脚本再按以下规则兜底判断：

- 前置条件超过 3 条、有效执行步骤超过 5 步，或预期结果校验点达到 3 条及以上，判定为 `困难`。
- 前置条件不超过 1 条、有效执行步骤不超过 2 步，且预期结果不超过 1 条，判定为 `简单`。
- 其他情况判定为 `一般`。

有效执行步骤从进入目标功能页面后开始计算，登录操作不计入步骤数。

### 关键字维护规则

满足以下任一情况，但当前关键字未覆盖时，应优先补充复杂、简单或一般关键字，不建议临时绕过脚本判定：

- 需要构造特定业务数据或状态数据。
- 需要跨页面或跨模块核对结果。
- 需要验证 2 个及以上关键结果。
- 涉及导入、导出、批量操作、组合查询或配置生效。
- 涉及 3 个及以上模块联动。
- 涉及多角色、多审批节点、转审、加签、电子签名或多人协同。
- 涉及第三方工具对比、跨环境迁移或数据完整性验证。
- 涉及报告生成、报告审批、报告生效、报告导出或报告回推等报告链路。
- 涉及长流程连续操作，且前置准备、执行步骤和验证结果均较复杂。

---

## 使用检查表

生成或评审用例难度时，逐项确认：

- [ ] 是否明确了前置准备复杂度？
- [ ] 是否从进入目标功能页面后统计有效执行步骤？
- [ ] 是否识别了跨模块、跨角色、跨环境或第三方依赖？
- [ ] 是否按前置准备、执行步骤、验证结果三个维度综合判定？
- [ ] 若多个维度等级不同，是否按最高等级取值？
