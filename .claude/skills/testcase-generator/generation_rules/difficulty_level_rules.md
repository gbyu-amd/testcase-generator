# 用例难度等级定义规则

## 目标

`用例标签` 只填写一个难度值：`简单`、`一般` 或 `困难`。

判断难度时不读取已有 `用例标签`。实际执行逻辑以 `scripts/case_utils.py` 中的 `infer_case_difficulty_with_reason()` 为准，本文件负责维护可配置的关键字和组合关键字。

难度判断会使用 `用例名称`、`用例描述`、`前置条件`、`用例步骤`、`预期结果`，但不同规则的匹配字段不同，详见“字段匹配口径”。

---

## 判定顺序

1. 先做报告误命中排除，避免把导入失败产物里的“错误报告”当成报告编制链路。
2. 命中困难强规则或困难组合关键字时，直接判为 `困难`。
3. 未命中困难规则时，先判断是否符合简单优先规则；导入文件级基础校验、导入模板下载即使仅命中 `导入` 复杂度信号，也可判为 `简单`。
4. 其他场景进入综合评分，按前置准备、执行步骤、验证结果和复杂度信号累计分数。
5. `UI` 用例和导入场景未命中困难规则时，即使综合分达到 `困难`，最高仍判为 `一般`。

综合分定级：

| 综合分 | 难度 |
|---|---|
| 0-2 | `简单` |
| 3-4 | `一般` |
| 5+ | `困难` |

---

## 通用预处理

### 字段匹配口径

字段范围按规则类型确定，避免因背景描述、前置准备或结果枚举中的词语误抬高难度。

| 规则类型 | 匹配字段 | 说明 |
|---|---|---|
| 高置信困难强规则关键字 | `用例名称`、`用例步骤`、`预期结果` | 命中直接判为 `困难`，不读取 `用例描述` 和 `前置条件` |
| 仅用例名称困难强规则关键字 | `用例名称` | 只在标题表达核心验证对象时判为 `困难` |
| 困难组合关键字 | `用例名称` + `用例步骤` | 合并后匹配组合，不读取 `用例描述`、`前置条件`、`预期结果` |
| 复杂度单关键字（非 `UI` 用例） | `用例名称`、`用例描述`、`前置条件`、`用例步骤`、`预期结果` | 只作为综合评分中的复杂度信号，最多 `+1` |
| 复杂度单关键字（`UI` 用例） | `用例名称`、`用例步骤`、`预期结果` | 不读取 `用例描述` 和 `前置条件`，避免页面入口和 `UI` 标记误加分 |
| 复杂度组合关键字 | `用例名称` + `用例步骤` | 合并后匹配组合，不读取 `预期结果`，最多 `+1` |
| 简单优先关键字 | `用例名称` | 只作为低复杂度信号，仍需满足前置、步骤和校验点数量限制 |

`用例描述` 为 `UI` 的用例按 `UI` 展示类处理：未命中困难规则时最高判为 `一般`，且展示校验点最多加 1 分。

### 报告误命中排除词

以下词属于导入失败产物，不属于报告编制或报告插入链路；匹配 `报告` 相关规则前先排除，避免误判：

<!-- difficulty-rule:report_lifecycle_exclusions -->
`txt错误报告`、`错误报告`、`失败报告`
<!-- /difficulty-rule -->

## 困难规则

命中本节任一规则时，直接判定为 `困难`，不再进入综合评分。

### 高置信困难强规则关键字

`用例名称`、`用例步骤` 或 `预期结果` 命中任一关键字：

<!-- difficulty-rule:difficult_high_confidence_keywords -->
`数据迁移`、`跨环境`、`第三方`、`minitab`
<!-- /difficulty-rule -->

### 仅用例名称困难强规则关键字

仅 `用例名称` 命中时判为 `困难`。这些词如果只出现在 `预期结果` 中，通常只是结果区域覆盖项，不直接抬高难度：

<!-- difficulty-rule:difficult_title_only_keywords -->
`判异检查结果`、`超出参考线检查`
<!-- /difficulty-rule -->

### 困难组合关键字

`用例名称` 和 `用例步骤` 合并后同时命中任一组合。该规则不读取 `用例描述`、`前置条件`、`预期结果`：

<!-- difficulty-rule:difficult_keyword_combinations -->
- `导入` + `跨环境`：导入涉及跨环境迁移或环境间一致性验证
- `导入` + `接口越权`：导入接口越权访问或权限绕过校验
- `导入` + `权限生效`：导入后权限配置需要生效验证
- `报告` + `分析结果`：报告插入分析结果正确性验证
- `分析成功` + `结果`：分析成功后的结果正确性验证
- `未分析` + `一键分析`：未分析的方法参与一键分析
- `一键分析` + `异常`：一键分析失败时异常提示校验
<!-- /difficulty-rule -->

---

## 简单优先规则

未命中困难规则和复杂度信号时，以下低复杂度场景可优先判为 `简单`。

代码按下表**自上而下的导入类优先**短路判定：导入文件级 → 导入模板下载 → 单字段提示类 → 基础配置项 / 字段级操作。任一分支命中即返回 `简单`，未命中则进入综合评分。

| 场景 | 适用条件 |
|---|---|
| 导入文件级基础校验 | 只验证文件类型、文件大小、空文件、无数据、无表头、表头不匹配、合并单元格、加密、损坏、文件不存在等上传或解析前置校验；前置条件不超过 3 条、有效步骤不超过 3 步、验证点不超过 3 个；不得涉及错误报告明细、字段逐项校验、落库、回滚、关联关系或权限生效 |
| 导入模板下载 | 只验证模板文件下载成功、文件名称、弹窗状态或模板基础结构；前置条件不超过 3 条、有效步骤不超过 3 步、验证点不超过 4 个；不得涉及字段逐项说明、下拉选项、表头批注或数据有效性细项校验 |
| 单字段提示类校验 | 同一字段的必填、格式或长度校验；前置条件不超过 3 条、有效步骤不超过 3 步、验证点不超过 3 个；不得命中任何复杂度信号 |
| 基础配置项、字段级操作或单项边界提示 | 前置条件不超过 4 条、有效步骤不超过 4 步、验证点不超过 3 个；不得命中任何复杂度信号 |

单字段提示类关键字：

<!-- difficulty-rule:simple_field_validation_keywords -->
`单字段校验`、`必填`、`格式提示`、`长度`
<!-- /difficulty-rule -->

基础配置项和字段级操作关键字：

<!-- difficulty-rule:simple_operation_keywords -->
`变量区域`、`放入横轴变量`、`子组大小`、`横轴旋转角度`、`新增参考线`、`下拉框`、`输入框`、`默认值`、`拖拽失败`、`添加失败`、`删除变量`、`删除纵轴变量`、`分析方法列`、`第二个定量变量`、`纵轴范围`、`参考线数据范围`、`阻止添加`、`只读`
<!-- /difficulty-rule -->

导入文件级基础校验关键字：

<!-- difficulty-rule:simple_import_file_validation_keywords -->
`非excel文件`、`文件超过`、`空文件`、`有表头无数据`、`有数据无表头`、`表头不匹配`、`合并单元格`、`加密文件`、`文件损坏`、`已删除`
<!-- /difficulty-rule -->

导入模板下载关键字：

<!-- difficulty-rule:simple_import_template_download_keywords -->
`导入模板`、`模板下载`、`下载模板`、`下载模版`
<!-- /difficulty-rule -->

## 综合评分

未命中困难规则且不符合简单优先规则时，按下表累计分数。

| 维度 | 条件 | 分值 |
|---|---|---|
| 前置准备 | 前置条件 0-1 条 | 0 |
| 前置准备 | 前置条件 2-3 条 | +1 |
| 前置准备 | 前置条件超过 3 条 | +2 |
| 执行步骤 | 有效步骤 0-2 步 | 0 |
| 执行步骤 | 有效步骤 3-5 步 | +1 |
| 执行步骤 | 有效步骤超过 5 步 | +2 |
| 验证结果 | 0-1 个校验点 | 0 |
| 验证结果 | 2-3 个校验点 | +1 |
| 验证结果 | 4 个及以上校验点 | +2 |
| 复杂度信号 | 命中任一复杂度信号 | +1 |

统计口径：

- 有效步骤从进入目标功能页面后开始统计，登录动作不计入。
- 预期结果按换行、编号点和分号拆分为独立校验点。
- 复杂度信号最多加 1 分，不按命中数量叠加。
- `UI` 用例的展示校验点最多加 1 分；复杂度单关键字不读取 `用例描述` 和 `前置条件`。
- `UI` 用例和导入场景未命中困难规则时，即使综合分达到 `5+`，最高仍判为 `一般`。

---

## 复杂度信号

命中任一复杂度关键字或复杂度组合关键字时，统一只加 1 分，不按命中数量叠加。

复杂度单关键字：

- 非 `UI` 用例匹配 `用例名称`、`用例描述`、`前置条件`、`用例步骤`、`预期结果`。
- `UI` 用例只匹配 `用例名称`、`用例步骤`、`预期结果`。

<!-- difficulty-rule:complexity_keywords -->
`导入`、`批量导出`、`跨页面`、`跨模块`、`审计追踪详情`、`批量删除`、`组合搜索`、`组合查询`、`联动`、`数据一致性`、`报告`、`工作流`、`会签`、`多角色`、`一键分析`、`越权`
<!-- /difficulty-rule -->

复杂度组合关键字：

仅匹配 `用例名称` + `用例步骤`，不读取 `预期结果`。

<!-- difficulty-rule:complexity_keyword_combinations -->
- `报告` + `生成`
- `报告` + `导出`
- `操作历史` + `详情`
- `操作历史` + `记录`
<!-- /difficulty-rule -->

## 等级含义

| 难度 | 含义 | 典型场景 |
|---|---|---|
| `简单` | 路径短、验证点单一、无需构造复杂数据 | 单字段必填、格式或长度提示；基础配置项默认值、下拉框、输入框、变量区域操作 |
| `一般` | 需要少量准备、存在一定交互路径或验证配置生效 | 导入、批量导出、组合查询、UI 多区域展示、权限或审计的少量校验 |
| `困难` | 准备复杂、路径长、涉及跨环境、第三方工具或高风险链路 | minitab 对比、数据迁移、跨环境导入、导入后权限生效、报告插入分析结果 |

---

## 维护要求

- 脚本只读取带 `difficulty-rule` 标记的配置块；调整关键字时优先修改这些清单。
- 标题、说明和示例可以优化，但不要删除或改名 `difficulty-rule` 标记。
- 需要校验规则效果时，按 `workflow_rules.md` 的“校验和导出闭环”执行。
- 若出现难度 WARN，按新规则同步 Markdown 中的 `用例标签`。
