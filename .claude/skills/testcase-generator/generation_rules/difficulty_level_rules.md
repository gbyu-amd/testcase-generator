# 用例难度等级定义规则

## 目标

`用例标签` 只填写一个难度值：`简单`、`一般` 或 `困难`。

判断难度时，不读取已有 `用例标签`。脚本实现以 `scripts/case_utils.py` 中的 `infer_case_difficulty_with_reason()` 为准。

---

## 判定流程

难度采用“强规则 + 单字段校验优先 + 综合评分 + UI 展示限制 + 简单信号限制”：

1. 命中强规则，直接判为 `困难`。
2. 未命中强规则，且属于单字段必填、为空、格式或长度提示校验时，优先判为 `简单`。
3. 其他场景按前置准备、执行步骤、验证结果、一般复杂度信号、业务风险信号累计评分。
4. `用例描述` 为 `UI` 且未命中强规则时，最高判为 `一般`。
5. 综合分 `0-1` 为 `简单`，`2-4` 为 `一般`，`5+` 为 `困难`。
6. `新增`、`编辑`、`删除`、`导出`、`查询` 等简单词只作为低复杂度参考，不能单独决定难度。

参与判断字段：`用例名称`、`用例描述`、`前置条件`、`用例步骤`、`预期结果`。

---

## 强规则

命中以下规则时直接判定为 `困难`。

### 困难强规则关键字

`用例名称`、`用例步骤` 或 `预期结果` 命中任一关键字：

`一键分析`、`数据迁移`、`跨环境`、`第三方`、`minitab`、`多角色`、`多人`、`会签`、`工作流`、`数据完整性`、`一致性验证`、`分析结果数据正确性校验`

### 困难组合关键字

`用例名称` 或 `用例步骤` 同时命中任一组合：

| 组合 | 场景 |
|---|---|
| `跨模块` + `校验` | 跨模块数据一致性校验 |

说明：`错误报告`、`失败报告`、`TXT错误报告` 属于导入失败产物，不按报告编制链路处理，不触发 `报告` 组合强规则。

## 综合评分

未命中强规则时，按下表累计分数。

| 维度 | 条件 | 分值 |
|---|---|---|
| 前置准备 | 前置条件 0-1 条 | 0 |
| 前置准备 | 前置条件 2-3 条 | +1 |
| 前置准备 | 前置条件超过 3 条 | +2 |
| 执行步骤 | 有效步骤 0-2 步 | 0 |
| 执行步骤 | 有效步骤 3-5 步 | +1 |
| 执行步骤 | 有效步骤超过 5 步 | +2 |
| 验证结果 | 0-1 个校验点 | 0 |
| 验证结果 | 2-3 个校验点 | +1 |
| 验证结果 | 4 个及以上校验点 | +2 |
| 一般复杂度信号 | 命中任一一般复杂度信号 | +1 |
| 业务风险信号 | 命中任一业务风险信号 | +1 |

统计规则：

- 有效步骤从进入目标功能页面后开始统计，登录动作不计入。
- 预期结果按换行、编号点和分号拆分为独立校验点。
- `UI` 用例的展示校验点最多加 1 分，不按普通业务校验点加到 2 分。
- `UI` 用例的业务风险信号不读取前置条件，避免因权限、模块入口等准备描述误判为高风险。
- `UI` 用例未命中强规则时，即使综合分达到 5+，最高仍判为 `一般`。
- 一般复杂度信号最多加 1 分。
- 业务风险信号最多加 1 分。

定级规则：

| 综合分 | 难度 |
|---|---|
| 0-1 | `简单` |
| 2-4 | `一般` |
| 5+ | `困难` |

## 评分信号

### 一般复杂度信号

关键字：`导入`、`批量导出`、`跨页面`、`跨模块`、`审计追踪详情`、`批量删除`、`组合搜索`、`组合查询`

组合：`批量 + 删除`、`批量 + 导出`、`组合 + 搜索`、`组合 + 查询`、`审计追踪 + 详情`、`查看 + 详情`、`报告 + 生成`、`报告 + 导出`

### 业务风险信号

`流转`、`联动`、`跨模块`、`数据一致性`、`报告`、`分析结果`、`回传`、`推送`、`数据分析`、`监控项目`

### 简单信号

简单信号只在低复杂度场景下辅助判为 `简单`，不能覆盖强规则和综合评分。

关键字：`列表页`、`翻页`、`排序`、`筛选重置`、`刷新`、`启用/停用`、`启用`、`停用`、`重置`、`单项操作`、`单页面提交`、`新增`、`增加`、`添加`、`编辑`、`删除`、`搜索`、`查询`、`单字段校验`、`必填`、`长度`、`格式提示`、`下拉多选`、`下拉选项`、`提示语校验`、`导出`、`单文件导出`

单字段提示类校验优先判为 `简单` 必须同时满足：

- `用例名称` 命中 `单字段校验`、`必填`、`格式提示`、`长度`。
- 预期结果不超过 1 个校验点。
- 前置条件不超过 3 条，有效步骤不超过 3 步。
- 未命中一般复杂度信号。
- 未出现导入、批量、跨模块、多字段、组合、数据一致性、数据完整性、计算、状态更新或成功提交类复杂场景。

简单信号辅助判为 `简单` 必须同时满足：

- 未命中强规则。
- 综合分不超过 1 分。
- 前置条件不超过 1 条。
- 有效步骤不超过 2 步。
- 预期结果不超过 1 个校验点。
- 未命中业务风险信号。

---

## 等级含义

| 难度 | 含义 | 典型场景 |
|---|---|---|
| `简单` | 路径短、验证点单一、无需构造复杂数据 | 列表翻页、单字段校验、单项查询、无业务联动的单文件导出 |
| `一般` | 需要少量准备、存在一定交互路径或验证配置生效 | 导入、批量导出、组合查询、UI 多区域展示、权限/审计/状态的少量校验 |
| `困难` | 准备复杂、路径长、涉及跨模块/多角色/第三方校验或数据正确性验证 | 一键分析、minitab 对比、数据完整性或跨模块一致性验证 |

---

## 维护要求

调整难度规则时必须同步修改：

- `generation_rules/difficulty_level_rules.md`
- `scripts/case_utils.py` 中的 `infer_case_difficulty_with_reason()`

修改后必须执行：

```bash
python scripts/validate_cases.py --source outputs/origin_exports
```

若出现难度 WARN，应按新规则更新 Markdown 中的 `用例标签`，再重新导出对应 Excel。
