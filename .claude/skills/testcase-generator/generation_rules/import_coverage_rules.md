# 导入功能专项覆盖规则

## 使用原则

涉及 Excel / CSV / 模板文件导入、批量导入、导入模板或错误报告时，必须先读取本文件，再生成或补充用例。

- 先按 PRD、模板字段表或用户补充说明确认字段名、必填性、条件必填、格式、长度和引用数据规则。
- 本文件只定义导入专项覆盖场景；优先级仍按 `priority_rules.md` 判定，难度仍按 `difficulty_level_rules.md` 判定。
- 只生成与当前导入需求直接相关的场景，不因为本文件存在而扩展无关导入对象。

---

## 基础规则

- 前置条件、执行步骤、预期结果中涉及模板字段时，必须使用 PRD / 模板原文，不得自行改名或概括。
- 模板字段说明类用例需逐字段悬停表头校验批注；步骤不得写 `对照 PRD 检查`，预期需写清字段批注内容，字段较多时按字段类别拆分。
- 导入成功至少拆分 `只填必填项导入成功`、`填写所有字段导入成功`；多行成功需验证成功条数、列表刷新、字段落库和引用关系一致。
- 前置条件不得写 `文件合格`、`字段完整`、`数据正常` 等模糊描述，也不得写步骤中才执行的动作。

---

## 异常与边界

- 文件级异常覆盖文件类型、文件大小、空文件、无数据、无表头、表头不匹配、合并单元格、加密、损坏。
- 字段级异常按模板字段逐项评估，覆盖必填为空、唯一性冲突、格式非法、长度边界、引用数据不存在、条件必填。
- 明确最大长度的字段，至少覆盖等于最大长度成功、超过最大长度失败。
- 同一字段同时包含格式规则和引用存在性规则时，格式错误和引用对象不存在必须拆成独立用例。
- 枚举 / 下拉字段若非法值可录入并保存，覆盖非法值导入失败；若 Excel 模板通过下拉或数据有效性拦截非法值，改为覆盖模板下拉选项是否完整正确，以及必填为空等实际可发生的导入校验。
- 条件字段需分别覆盖条件满足时的必填、格式、引用存在性，以及条件不满足时允许或不允许填写的取值规则。
- 同一行多个字段错误仅用于验证错误报告展示，不替代单字段独立校验。
- 若 PRD 要求整体成功或整体失败，必须验证任一行错误时整份文件拒绝导入，不产生部分导入脏数据。

---

## 错误报告

涉及导入失败报告、TXT 错误报告或错误明细下载时，至少评估：

- 是否展示错误行号、字段名和失败原因。
- 同一行多个错误是否按约定格式展示。
- 错误报告下载后内容与页面失败原因一致。
- 导入失败后列表、权限、引用关系和审计记录不产生脏数据。

---

## 命名与分组

- 导入失败类用例名称必须写清具体失败原因，优先使用 `具体原因，导入失败` 或 `具体原因，上传失败`。
- 导入失败类用例的 `二级分组` 统一填写 `导入失败校验`，`三级分组` 按被校验字段或对象填写。
- 模板说明类用例的 `二级分组` 可填写 `模板下载`，`三级分组` 按字段类别填写。
- 相同优先级内，校验同一字段或同一类导入对象的用例必须相邻放置。
