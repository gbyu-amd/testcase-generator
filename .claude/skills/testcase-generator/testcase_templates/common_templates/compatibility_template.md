# 兼容性用例模板

## 适用范围

适用于验证 CPV 系统页面在 Chrome 浏览器下，不同操作系统、屏幕分辨率、窗口宽度、浏览器缩放比例等环境中的展示和基础交互表现。

## 编写要求

- 明确 Chrome 浏览器版本、操作系统、屏幕分辨率、浏览器缩放比例、窗口宽度和网络环境。
- 明确页面或功能入口。
- 关注布局、交互、数据状态和错误提示，尤其是表格、筛选区、弹窗、图表、报告预览和按钮区是否被遮挡或错位。
- 兼容性用例只要求覆盖 Chrome 浏览器，不需要为其他浏览器生成用例。
- 操作系统需覆盖 Windows 和 macOS；Windows 侧使用 Windows 10 或 Windows 11 任一版本即可。
- 兼容性用例不能只写“显示正常”，必须说明关键元素和操作结果。
- 兼容性用例重点验证展示适配和基础交互，不覆盖状态同步、页面恢复、任务恢复等稳定性或回归场景。

## 分辨率覆盖建议

生成兼容性用例时，优先覆盖以下 Chrome 分辨率组合。若需求或 UI 设计指定了其他分辨率，以需求或设计为准。

| 分辨率 / 窗口 | 适用场景 | 重点验证 |
|---|---|---|
| 1366x768 | 笔记本小屏或最低桌面分辨率 | 顶部导航、侧边菜单、筛选区、表格操作列、弹窗底部按钮不被遮挡 |
| 1440x900 | 常见办公显示器 | 列表、详情页、审批弹窗和电子签名弹窗布局稳定 |
| 1920x1080 | 主流桌面分辨率 | 工作台看板、图表区域、报告预览、数据分析配置区域展示完整 |
| 2560x1440 | 大屏显示器 | 表格列宽、图表拉伸、页面留白和固定操作区展示合理 |
| Chrome 125% / 150% 缩放 | 用户系统缩放或浏览器缩放 | 字段、按钮、状态标签、悬停提示和弹窗内容不重叠、不截断 |

## 操作系统覆盖要求

生成兼容性用例时，必须在 Chrome 浏览器前提下覆盖以下操作系统：

| 操作系统 | 覆盖要求 | 重点验证 |
|---|---|---|
| Windows 10 或 Windows 11 + Chrome | Windows 侧二选一必测 | 页面布局、文件上传下载、导入导出、滚动条、弹窗、表格操作列、固定按钮区、系统缩放和字体渲染稳定 |
| macOS + Chrome | 必测 | 字体、快捷键、文件选择器、滚动条、下载文件名、图表渲染和报告预览展示一致 |

## CPV Chrome 兼容场景

- Chrome 在 Windows 10 或 Windows 11、macOS 下，登录、工作台、计划、任务、方案、数据分析和报告页面功能可用。
- Chrome 不同屏幕分辨率和缩放比例下，列表、筛选区、详情页、审批弹窗、电子签名弹窗和图表区域展示。
- Chrome 在不同操作系统下，文件选择器、上传入口、下载文件名、滚动条、悬停提示和键盘输入表现符合预期。
- Chrome 在不同分辨率下，表格固定列、操作列、底部按钮区、弹窗底部按钮和图表区域无遮挡、错位或截断。
